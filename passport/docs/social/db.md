# Хранилище

Хранилище реализует API, с помощью которого можно управлять данными социальных профилей. Для каждого соцпрофиля могут храниться:

* Личные данные пользователя, предоставленные провайдером.
    
* Когда-либо полученные OAuth-токены для запросов API соцсети.
    
* Признак разрешения на авторизацию. Без этого признака с аккаунтом соцсети невозможно войти в учетную запись Яндекса, указанную в данном профиле.

    Признак автоматически выставляется при первой авторизации с аккаунтом соцсети, в дальнейшем пользователь может управлять им на странице [Социальные профили](https://social.yandex.ru).

* Признаки использования профиля на сервисах Яндекса, например, в [Поиске людей](http://help.yandex.ru/peoplesearch/about.xml). Пользователь может разрешать или запрещать сервисам использовать свои профили на странице [Социальные профили](https://social.yandex.ru).

Идентификатор, токены и личные данные в соцсети обновляются автоматически, когда пользователь авторизуется с социальным аккаунтом на Яндексе. Обновление можно запустить для уже авторизованного пользователя, [отправив его в Брокер](broker.md#data).

В этом же запросе можно указать недостающие права для доступа к соцсети. Если пользователь разрешит запрошенный доступ, Брокер отправит новый OAuth-токен в Хранилище вместе с обновленными данными.

Описание запросов к Хранилищу приведено на Вики: [https://wiki.yandex-team.ru/social/api](https://wiki.yandex-team.ru/social/api). Адреса хостов Хранилища для различных окружений перечислены в разделе [Хосты Социализма](location.md).
