# Обзор режимов Паспорта

Функциональные возможности Паспорта реализованы в виде режимов — методов, вызываемых по протоколу HTTP и HTTPS. С помощью режимов сервисы Яндекса могут регистрировать и авторизовывать пользователей, а также изменять данные учетных записей.

Некоторые режимы предоставляют пользовательский веб-интерфейс. Сервисы могут корректировать поведение таких режимов: например, пользователя можно вернуть на конкретную страницу сервиса после [авторизации](../reference/auth.md).

{% note info %}

Режимы Паспорта предназначены для изменения данных учетных записей. Если вашему сервису нужно только получать данные о пользователях, воспользуйтесь API Черного ящика (описан в документе [Программный интерфейс сервиса Яндекс.blackbox](https://doc.yandex-team.ru/blackbox/concepts/about.xml)).

{% endnote %}


## Классификация режимов

Режимы разделены на следующие группы:

- Внутренние. Могут вызываться только серверами, расположенными в сети Яндекса (см. раздел [Доступ к режимам Паспорта](location.md)).
- Публичные. Могут вызываться из внешних сетей — браузерами или другими приложениями.

В рамках документа режимы данных группы классифицированы по назначению: регистрация, авторизация и т. п. Структура документа соответствует итоговой классификации.
