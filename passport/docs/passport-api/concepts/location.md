# Доступ к API Паспорта

Пользоваться API могут только зарегистрированные в Паспорте потребители, которые запросили доступ к нужным методам или сценариям API.

Чтобы подключиться к API:

1. Придумайте идентификатор потребителя (короткое слово, соответствующее сервису, например «metrika» или «postoffice»).
1. Выберите методы или сценарии, для которых вам нужны гранты (из [приведенных на Вики](https://wiki.yandex-team.ru/passport/python/api)). Гранты — это разрешения, зафиксированные в ACL (Access Control List), для определенного множества IP-адресов. Для личных компьютеров разработчиков гранты не выдаются.
1. Составьте список подсетей или IP-адресов, с которых могут приходить запросы с указанным идентификатором потребителя.

Письмо с этими данными отсылайте на рассылку [passport-admin@](mailto:passport-admin@yandex-team.ru) — администраторы Паспорта занесут идентификатор в базу и выдадут нужные гранты для указанных адресов.


## Адреса для запросов {#address}

#### Тестовый хост

Находится по адресу [passport-test-internal.yandex.ru](http://passport-test-internal.yandex.ru).

Работает с тестовой базой данных и тестовым Черным ящиком ([pass-test.yandex.ru/blackbox](pass-test.yandex.ru/blackbox)). Авторизационные куки, выписанные тестовым API Паспорта, невалидны для боевого Паспорта.

#### Боевой хост

Находится по адресу [passport-internal.yandex.ru](passport-internal.yandex.ru).

Работает с боевым Черным ящиком ([blackbox.yandex.net/blackbox](blackbox.yandex.net/blackbox)). Предназначен только для вызовов с боевых серверов. Авторизационные куки, выписанные боевым Паспортом, невалидны для тестового Паспорта.

Раздел <https://doc.yandex-team.ru/blackbox/concepts/blackboxLocation.xml> документа [Программный интерфейс сервиса Яндекс.blackbox](https://doc.yandex-team.ru/blackbox/concepts/about.xml)
