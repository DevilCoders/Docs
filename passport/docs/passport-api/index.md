# Введение

**Документ содержит** описание базового API Яндекс Паспорта, предназначенного для реализации различных сценариев регистрации и авторизации на сервисах Яндекса. В документе дана общая информация о структуре и доступе к API, а также перечислены уже работающие сценарии.

**Целевая аудитория документа** — менеджеры сервисов Яндекса, которые разрабатывают собственные интерфейсы регистрации или авторизации.

**Документ не содержит** справочную информацию о методах API, особенностях их вызова и возможных ошибках. Эта информация приведена на Вики: [http://wiki.yandex-team.ru/passport/python/api](http://wiki.yandex-team.ru/passport/python/api).

Пожелания и замечания к документу направляйте в Jira: [https://jira.yandex-team.ru/browse/DOC-2226](https://jira.yandex-team.ru/browse/DOC-2226).

**Документ, рекомендуемый для предварительного ознакомления**:

- [Обзор системы авторизации](https://docs.yandex-team.ru/authdevguide/) — описание понятий и процессов, на которых основана работа системы авторизации для веб-сервисов.

## Структура документа

Документ состоит из следующих разделов:

1. [Сценарии в API Паспорта](concepts/requests.md): краткое введение в API Паспорта, перечень описанных в документе сценариев.
1. [Доступ к API Паспорта](concepts/location.md): приведены адреса тестового и боевого хостов API, а также процедура запроса доступа.
1. [Сценарии регистрации](concepts/registration.md): перечень сценариев API, предназначенных для регистрации пользователей.

О выпущенных версиях документа читайте в разделе [Релизы](concepts/releases.md).
