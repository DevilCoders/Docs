
Возможность смены паролей через интерфейс Яндекса: 0 — запрещено менять пароли, 1 — разрешено менять пароли.

{% note info %}

Запрет смены паролей применим, если владелец подключенного домена предоставляет третьим лицам адреса электронной почты в своем домене и при этом самостоятельно управляет паролями.

Такой сценарий могут реализовать, например, интернет-порталы, клиенты которых получают доступ к портальным сервисам и к почте по одним и тем же учетным данным. На портале ведется база учетных записей и реализован интерфейс смены паролей, но почтовые ящики клиентов расположены в Яндексе.

Когда клиент меняет пароль на портале, портал одновременно меняет пароль аккаунта в Яндексе, и клиент может пользоваться почтой по новому паролю. Чтобы избежать рассинхронизации паролей на портале и в Яндексе, клиентам запрещается менять пароли самостоятельно через интерфейс Яндекса.

{% endnote %}
