# Введение

API Яндекс ID позволяет разработчикам сайтов и веб-приложений однозначно идентифицировать пользователя Яндекса. Таким образом разработчик может персонализировать предоставляемый контент и интерфейс приложения.

Логин пользователя и его уникальный идентификатор на Яндексе предоставляется по любому валидному [OAuth-токену Яндекса](oauth/concepts/about.md). Использование API Яндекс ID для входа через Яндекс описано в виде [типичного сценария авторизации](api-id/tasks/algorithm.md).

Чтобы запрашивать личную информацию о пользователе, следует получить OAuth-токен с правами доступа к API Яндекс ID. Данные, которые можно запросить:

- фамилия и имя;
    
- никнейм;
    
- пол;
    
- дата рождения;
    
- адреса электронной почты;
    
- номер телефона;
    
- портрет пользователя.
    
Подробности — в разделах [{#T}](api-id/reference/request.md) и [{#T}](api-id/reference/response.md).
