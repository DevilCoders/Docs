# Отключение возможности получить токен по паролю

Приложения, зарегистрированные на {{ service }} после 25 мая 2016 года, не могут получить OAuth-токен в обмен на логин и пароль пользователя (запросы токенов с параметром `grant_type=password` будут отклоняться). Для остальных приложений этот способ получения токенов был отключен 25 июня 2016.

Это решение было принято потому, что в последнее время этот метод получения токенов показывает себя как не самый безопасный. В службу поддержки часто приходят сообщения о краже личной информации, данных банковских карт и других неприятных ситуациях.

Доступ к данным пользователя можно будет получать только через форму OAuth-авторизации Яндекса:

- [во всплывающем окне](../reference/web-client.md);
- [в новой вкладке браузера](../reference/auto-code-client.md);
- [в WebView](../reference/mobile-client.md) (для мобильных приложений).
