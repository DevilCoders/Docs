# Последовательность входа через Яндекс

Общая последовательность авторизации пользователя на вашем сайте с помощью Яндекса:

1. Пользователь Яндекса нажимает на сайте кнопку входа через Яндекс.

   Сайт, [подключенный к API Яндекс ID](../concepts/adoption.md), перенаправляет пользователя на [{{ service }}]{% if lang == "ru" %}(https://oauth.yandex.ru/){% endif %}{% if lang == "en" %}(https://oauth.yandex.com/){% endif %}, запрашивая доступ к определенным данным аккаунта на Яндексе.

1. Пользователь разрешает доступ к своим данным.
   
   Сайт [получает OAuth-токен](../../oauth/reference/web-client.md), дающий право запроса адресов e-mail пользователя.

1. Сайт отправляет [запрос к API Яндекс ID](../reference/request.md), указывая полученный токен.

   Сайт получает уникальный идентификатор пользователя и список электронных адресов. Формат ответа описан в разделе [{#T}](../reference/response.md).
   
   {% note info %}

   Для запроса к API Яндекс ID можно использовать OAuth-токен с правами доступа к любому из сервисов Яндекса (Фоткам, Диску и т. д.). Но расширенный доступ к данным пользователя дают только токены с правами доступа из секции API Яндекс ID.

   {% endnote %}

1. Сайт авторизует пользователя, используя настройки и контент, привязанные к полученному идентификатору.

Таким образом пользователь Яндекса может получить аккаунт на сайте и авторизовываться на нем, не заводя новую учетную запись.
