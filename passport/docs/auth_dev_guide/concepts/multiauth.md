# Мультиавторизация

Мультиавторизация — это механизм, позволяющий пользователям одновременно авторизоваться несколькими аккаунтами в одном браузере и переключаться между ними. Аккаунт, на который переключился пользователь, считается основным и используется для авторизации по умолчанию.

Яндекс.Паспорт реализует мультиавторизацию, разрешая указывать больше одного аккаунта в куке `Session_id` (до введения мультиавторизации каждая кука соответствовала только одной учетной записи). Новый формат куки обратно совместим, поэтому мультиавторизационная кука работает на всех сервисах.

Чтобы пользователи могли управлять мультиавторизацией, ее также должны поддержать сервисы, которые персонализируют интерфейс для авторизованных посетителей. Для этого нужно выполнить следующие условия:

- Пользователю должны быть доступны переключение аккаунта, добавление нового аккаунта в куку и выход из основного аккаунта. Элементы интерфейса для этих действий можно реализовать с помощью Лего или самостоятельно. О том, как реализовать эти функции, читайте в разделе [Интерфейс мультиавторизации](multiauth/choice.md).
    
- Пользователю необходимо позволить использовать разные аккаунты в одновременно открытых вкладках браузера. Каждая страница сервиса должна продолжать работать для аккаунта, с которым она была открыта, пока этот аккаунт авторизован или пока пользователь не переключился на другую учетную запись. О том, как поддержать эту возможность, читайте в разделе [Разные аккаунты в одном браузере](multiauth/concurrent-tabs.md).
