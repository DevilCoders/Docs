# API системы авторизации

Функции и данные системы авторизации предоставляются сервисам Яндекса посредством различных API. Программными интерфейсами обладают следующие подсистемы:

Ответственными за доступ ко всем API являются администраторы Паспорта. Все запросы доступа следует запрашивать в рассылке [passport-admin@](mailto:passport-admin@yandex-team.ru).


## Яндекс.Паспорт {#passport}

Режимы Паспорта — это HTTP-вызовы пользовательского и программного интерфейса Паспорта. Основное назначение API Паспорта состоит в изменении пользовательских данных.

Также Паспорт обрабатывает вызовы МДА и авторизационные запросов Яндекс.Бара.


## Черный ящик {#bb}

Черный ящик предоставляет API для проверки различных авторизационных данных (пар логин/пароль, авторизационных кук, OAuth-токенов) и для запроса информации о пользователях.


## YaSMS {#yasms}

YaSMS предоставляет возможность отправки SMS зарегистрированным пользователям Яндекса, указавшим номер телефона на странице личных данных в Паспорте. Сообщения отправляются с помощью SMSC (SMS-центров) операторов-партнеров.


## Паспорт для доменов {#pdd}

Паспорт для доменов — это режим Паспорта, предназначенный для управления пользователями Почты для доменов (ПДД). Пользователи ПДД не могут одновременно пользоваться портальными сервисами Яндекса, но проходят авторизацию в Паспорте.

Также система авторизации хранит информацию о зарегистрированных в ПДД доменах и почтовых базах.


## Капча-сервер {#captcha}

Сервис Яндекс.Капча обеспечивает проведение тестов на распознавание символов. Используя API капча-сервера, разработчики сервисов Яндекса избавляются от необходимости самостоятельно решать проблему отличения людей и роботов.
