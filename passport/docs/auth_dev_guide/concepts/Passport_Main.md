# Общие сведения о системе авторизации

Система авторизации — это интегрированный набор сервисов Яндекса, предназначенных для управления данными учетных записей (аккаунтов) и централизованной авторизации пользователей.

{% cut "Подробнее об авториазации пользователей" %}

Предоставление пользователю прав доступа к сервису.

Формально Паспорт предоставляет пользователям права только на использование функций Паспорта. Сервисы Яндекса должны проверять выставленные Паспортом куки и предоставлять права, соответствующие учетной записи.

{% endcut %}

Понятие учетной записи (аккаунта) в системе авторизации Яндекса, а также важные характеристики аккаунтов рассмотрены в разделе [Учетная запись](Account_About.md).

## Общее использование системы авторизации

Посредством API системы авторизации сервисы могут:

- запрашивать и изменять данные учетных записей;
- регистрировать и авторизовывать пользователей;
- выполнять служебные задачи — отсылать SMS, показывать капчу, подбирать логины для регистрации и т. д.

Все программные интерфейсы системы авторизации описаны в документах, перечисленных в разделе [API системы авторизации](api.md). Раздел [Часто используемые данные системы авторизации](DB_About.md) описывает наиболее важные данные о пользователях, хранимые системой.

## Авторизация в веб-интерфейсе

Сервисы с веб-интерфейсом должны придерживаться общих правил обработки авторизационных кук.

Правила и рекомендации по использованию системы авторизации в веб-сервисе собраны в разделе [Использование системы авторизации на веб-сервисе](usage.md): предложен алгоритм обработки авторизаций и требования к интерфейсу авторизации. Также описаны тонкости авторизации в разных доменах.

Виды авторизационных кук, используемых Паспортом, описаны в разделе [Куки системы авторизации](Auth_About.md). Раздел [Обновление авторизационной куки](authorization-policy-resign.md) содержит сведения о времени жизни и способах продления кук.
