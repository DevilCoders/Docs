# 4. Семантика

## Наименование сетевых элементов
Правила для формирования имен (hostname) для элементов:
```
[{Код_региона}{Очередь}-{Модуль}{Тип_устройства}{Порядковый_номер_устройства}]
```

Коды регионов:
| ID  | Code  | Type | Racktables Tag | Description   |
|:---:|:-----:|:----:|:--------------:|:-------------:|
| 70  | MAN   | DC   | Мянтсяля       | Мянтсяля      |
| 86  | IVA   | DC   | Ивантеевка     | ДЦ Ивантеевка |
| 87  | MYT   | DC   | Мытищи         | ДЦ Мытищи     |
| 104 | SAS   | DC   | Сасово-1       | ДЦ Сасово     |
| 116 | VLA   | DC   | Владимир       | ДЦ Владимир   |
| 117 | SAS2  | DC   | Сасово-2       | ДЦ Сасово-2   |
| ??? | VLA2  | DC   | Владимир       | ДЦ Владимир-2 |

[Ссылка на Racktables](https://racktables.yandex-team.ru/index.php?page=staffonly&tab=jsonedit&jsontab=DC+Codes)
[Wiki:Codes](https://wiki.yandex-team.ru/noc/Codes/)

Сеть внутри ДЦ строится из следующих устройств:
{{grid page="/NOC/hld/semantic" width="100%" }}

| Код   | Тип устройства                                                                                |
|:-----:|:----------------------------------------------------------------------------------------------|
| a     | aggregation - коммутатор аггрегации сервисных устройств (FW, SLB и т.д.)                      |
| b     | border - пограничный маршрутизатор                                                            |
| c     | core - коммутатор ядра сети                                                                   |
| d     | коммутатор аггрегации стоек (Spine1)                                                          |
| decap | декапсулятор                                                                                  |
| e     | edge-коммутатор                                                                               |
| ic    | incterconnect коммутатор, предназначенный для передачи между серверами по L2                  |
| fw    | firewall                                                                                      |
| ld    | балансировщик нагрузки                                                                        |
| nat64 | NAT64                                                                                         |
| rr    | BGP route reflector                                                                           |
| s     | стоечный коммутатор (ToR)                                                                     |
| tun64 | Туннелятор                                                                                    |
| x     | коммутатор фабрики коммутации (Spine2)                                                        |
| z     | DCI-маршрутизатор ("многодэшка" новой волны)                                                  |
| g     | Guppy - сервер/маршрутизатор предназначенный для подключения IPv4 оборудования к сети ДЦ      |


> К примеру:
> `vla1-1s101` - стоечный коммутатор в первом модуле VLA1, с порядковым номером 101

Полный список типов устройств и исключений:
