# Описание правил и политик infra-настроек

## Timezone

На всех устройствах сетей Датацентров используется единое значение timezone `UTC`, независимо от месторасположения устройства.

{% note info %}

Был выбор между `MSK` и `UTC`. От `MSK` отказались, так как существуюет вероятность возращения к смене времени с летнего на зимнее и наоборот.

{% endnote %}

## NETCONF

Протокол NETCONF используется для сбора части мониторинговой информации с сетевого оборудования (процесс миграции продолжается), а так же для сбора различной информации различными системами (например racktables). В перспективе, планируется реализовать единый сервис сбора информации с сетевого оборудования - glavnoc, который в основном будет оперировать только структурированными данными, а не парсингом CLI.

На всем парке сетевого оборудования, которое поддерживает протокол NETCONF, используется порт 22 вместо стандартного порта 830 с тем, что бы упростить вопрос доступа и унифицировать транспорт.

Тикет по включению NETCONF - [NOC-19313](https://st.yandex-team.ru/NOC-19313)

## SNMP trap
Для сбора истории и мониторингов SNMP trap есть сервис [snmptrapper](https://a.yandex-team.ru/arc/trunk/arcadia/noc/snmptrapper). Чтобы им воспользоваться, нужно настроить отсылку trap в noc-snmptrap.yandex.net. Что делать дальше с trap смотри в [доке](https://a.yandex-team.ru/arc/trunk/arcadia/noc/snmptrapper/Readme.md).
