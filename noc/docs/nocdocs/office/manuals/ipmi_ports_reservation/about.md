# Процедура резервирования портов в IPMI-коммутаторах

**Цель документа**

Документ описывает процедуру резервирования и подключения к сети управления IPMI. 

**Для кого этот документ**

Для инженеров сетевой инфраструктуры Яндекса, которым необходимо обеспечить управления сетевыми устройствами через выделенные IPMI-порты.

## Описание процедуры

Процедура состоит из нескольких основных действий:
- резервирование портов и трасс в ДЦ и офисах для подключения к оборудованию IPMI-сети
- конфигурация оборудования для получения сервиса

### Шаг 1

Необходимо завести тикет в очередь [NOC](https://st.yandex-team.ru/createTicket?queue=NOC) с указанием следующих параметров:
- Локация оборудоваия:
  - Датацентр/Офис
  - Стойка/Место
- Для какого оборудования требуются порты
- Количество портов
- Желаемые сроки

После этого тикет попадет в команду офисной сети. Сотрудник офисной сети определяет к какому коммутатору необходимо выделить трассы.

Если свободных портов нет, то сотрудник офисной сети:
- оценивает сроки расширения IPMI-сети
- сообщает инициатору, что выделение портов невозможно в настоящее время и планируемые сроки по предоставлению портов


### Шаг 2

Исполнитель из тикета очереди NOC из Шага 1 заводит тикет для резервирования или прокладки трассы до портов в коммутаторе IPMI сети в очереди [ITDC](https://st.yandex-team.ru/createTicket?queue=ITDC&_form=49117) следующим содержанием:
- Локация оборудоваия:
  - Датацентр/Офис
  - Стойка/Место
- Откуда и куда нужны трассы
- Количество трасс
- Желаемые сроки

После выделения и коммутации трасс тикет ITDC закрывается.

### Шаг 3

После коммутации оборудования:
- необходимый VLAN и настройки на IPMI-оборудовании уже есть
- инициатор производит настройку оборудования
- инициатор проверяет работоспособность

Если все успешно, то закрывается тикет NOC.
