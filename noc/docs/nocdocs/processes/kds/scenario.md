# В этом разделе описаны сценарии процесса КДС

## Обзор

1. Диффы прошлого (Past Diff)
2. Диффы настоящего (Present Diff)
3. Диффы будущего (Future Diff)

Мы выделяем одно из основных свойства строки конфигурации:
- safe - безопасное изменение (без прерывания сервиса)
- unsafe - опасное изменение (с прерыванием сервиса, или просто стремно его катить)

### Предложение по коммиту в Аннушку

Есть идея при MR в master Аннушки создавать тикет на автора MR по снулению диффов, которые он инициализировал.

## Present Diff

Пререквизитом Present Diff является условие, что предыдущий diff был равен 0. Например, diff mgmt был равен нулю,
и "вдруг" он стал отличным от нуля. Для таких ситуаций выделяем два разных процесса.

### Present Diff Safe

Процедура по снулению безопасного диффа:
1. Произошел MR в master Аннушки
2. Если diff > 0
3. Ждём какое-то время
4. Заводится тикет в ST
5. Как-то пинается INFRA
