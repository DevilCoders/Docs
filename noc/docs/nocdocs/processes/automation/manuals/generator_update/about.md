# Обновление генератора в Аннушке

Итак вы решили обновить генератор в Аннушке. Так как это повлечет за собой появление новых diff на сети, то хочется понимать причины внесения изменений. Поэтому, первым делом, стоит завести тикет в одной из очередей NOC и описать в нем что планируете поменять и для чего.

0. Завести тикет с описанием изменений

1. Обновить локальный репозиторий ann до актуального состояния. Например, сделав `git pull master`.
2. Создать новый бранч с любым именем. Например, `git checkout -b <branch name>`. Рекомендуем использовать следующий формат: `номер тикета` + `_` + `короткое описание`
3. Внести изменения 
4. Запустить локальные тесты tox
5. Протестировать изменения локально
6. Запушить изменения в бранч: `git push --set-upstream origin <branch name>`
7. Инициализировать Merge Request из бранча в master ветку проекта. При этом необходимо добавить в название Merge Request номер тикета в одном из форматов: 
  - `NOC-12345`
  - `[NOC-12345]`
8. Вмержить изменения

Далее сгенерируются diff на сети которые могут выкатиться двумя способами:
- автоматически по расписанию
- вручную

Если изменения должны будут выкатиться вручную, то вы будете назначены ответственными за их приименение на сети.
