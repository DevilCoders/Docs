## Как добавить новый файл

В зависимости от выбранного вами метода обновления конфигурации ([Web](https://docs.yandex-team.ru/nocdoc/processes/docs_update/howto/update_via_web/about) или [Arc](https://docs.yandex-team.ru/nocdoc/processes/docs_update/howto/update_via_arc/about))
рано или поздно потребуется создать файл.

**Как это сделать**

1. Определившись с темой будущего файла, перейдите в раздел в репозитории, допустим это будет `nocdoc\office\manuals`
2. Создайте **папку** (**директорию**) c понятным названием, например, `howto_choose_name_of_router`
3. В этой папке сделайте файлик `about.md`
4. Вносите изменения

**Почему так?**

Все потому, что есть высокая вероятность, что в будущем потребуется добавить что-то еще в этот раздел, например, изображения. Это удобнее делать, когда есть отдельная папка для документации.

## Как добавить картинку

Сложный этап разработки документации. Требует дополнительных усилий со стороны документируюзего

**Как это сделать**

1. Перейдите в папку/директорию с документацией, например, в `nocdoc\office\manuals\howto_choose_name_of_router`
2. Сделайте папку `_images`, если её еще нет. Должно получиться так: `nocdoc\office\manuals\howto_choose_name_of_router\_images`
3. Вставьте изображение. Лучше в `png`.
4. В файле с документацией добавьте: `[<всопогательный текст>](./_images/<имя_файла.png>)`.
5. commit - pull request
