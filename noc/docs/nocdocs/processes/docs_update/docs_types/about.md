# Типы документации NOC

Глобально документация в NOC можно разделить на:
- Документацию для инженеров (HLD, LLD, и пр.)
- Документацию для разработчиков (Developer Guide, Stackoverflow)
- Документацию для дежурных (инструкции, user guide и пр.)
- Документацию, описывающая процессы (регламенты, стандарты, рабочие процедуры)
- Обучающая документация (onboarding, различные материалы NOC, вендорские курсы и пр.)

![img.png](_images/img.png)

Так как мы преследуем цель создания единого источника документации для Отдела сетевой инфраструктуры, и понимаем, что сейчас много информации находится в кластерах Wiki, то было решено следовать следующим принципам:
- Регламенты и процессы взаимодействия между отделами и группами выносим в `docs:processes`
- Существующие кластера в Wiki оставляем для внутренней документации, но с последующим переносом их в docs в соответствущие разделы проекта:
  - Сети датацентров и магистрали --> `docs:design`
  - Офисные сети --> `docs:office`
  - IP-телефония и видеосвязь --> `docs:iptel`
  - Traffic Team --> `docs:trafficmgmt`
  - NOCDEV --> `docs:services`
