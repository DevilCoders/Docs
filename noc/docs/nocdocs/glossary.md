# Глоссарий NOC

## А
- Аннушка - система хранения и управления конфигурацией на оборудовании сети.

## Г
- Гидра - патчкорд, позволяющий преобразовать порт большей емкости в кратное число портов меньшей ёмкости (40GE в 4 порта по 10GE, 100GE в 10 портов по 10GE)

## Д
- Датацентр - совокупность инженерных, серверных и сетевых систем. Может состоять из одного здания, а может и из нескольких.

## М
- Макрос, сетевой макрос - это способ объединения сетевых сущностей в группы. Делается это путем сопоставления некоторого строкового идентификатора и списка сетевых сущностей.

- Модуль - часть здания внутри очереди, независимо функционирующая единица вводимой в эксплуатацию части ДЦ. ДЦ расширяются модулями, по 130-160 стоек

## О

- Очередь - здание внутри ДЦ, принимает значения от 1 до N

## П

- Переподписка - соотношение между суммарной полосой в сторону нижестоящих узлов и суммарной полосой в сторону вышестоящих

## С
- Свитч -
- СКС – структурированная кабельная система

## Т
- Точка Доступа (ТД) - WiFi-роутер внутри офиса

## У

Уровень - в Fat-tree и Clos-топологиях, набор одноранговых сетевых элементов , где нижний уровень (Leaf) имеет индекс 0. Обычно, связи между элементами одного уровня отсутствуют

## Ф

Фабрика коммутации (или просто фабрика) - набор сетевого оборудования, необходимого для создания связности внутри ДЦ, очереди и модуля

## A
- Argo - более продвинутая версия Глоссария
- Aggregation:
  - Коммутатор аггрегации сервисных устройств (FW, SLB и т.д.)
  - Объединение маршрутов по какому-либо признаку
- Annushka (Ann) - см. [Аннушка](#a)

## B
- Backbone - магистральная сеть, передающая трафик между DC
- Border - пограничный маршрутизатор сети

## C
- Command Line Interface (CLI) - см. [CLI](https://ru.wikipedia.org/wiki/Интерфейс_командной_строки)

## D
- Datacenter - см. [Датацентр](#d)
- DC - см. [Датацентр](#d)
- Decap - декапсулятор
- Diff - это информация о различии "правильной" конфигурации устройства и реальной конфигурацией устройства.
- dom0, domain 0 - хостовая часть, на которой поднимаются контейнеры.

## E
- Equal-Cost Multi-Path - концепция создания множества равнозначных путей между двумя произвольными точками с целью балансировки трафика по этим путям
- **Emergency change** - экстренное изменение, связанное с необходимостью решения инцидента.

## F
- Firewall (FW) - устройства фильтрации L4 трафика
- Fastbone (FB) - магистральная сеть для bulk-трафика

## G
- Global Routing Table (GRT) - глобальная таблица маршрутизации

## H
- Host-Based Firewall (HBF) - файрвол, расположенный на хостах и предназначенный для фильтрации межпроектного трафика
- HEAT - система визуализации алертов 

## L
- Label Switched Path (LSP) - путь коммутации по меткам
- Leaf - коммутатор, к которому подключаются сервера

## M
- Macros - см. [Макрос](#m)
- **Major change** - изменение, которое проводится более чем в одной локации или может потенциально привести к перерыву сервиса (критическое изменение).
- Multi-Tenancy Network (MTN) - концепция жизни всех сервисов в одной сети с разделением их по IP-адресации с помощью определенных правил
- **Minor change** - изменение, которое проводится не более чем в одной локации и к перерыву в сервисе не приводит.

## O

## P
- PoD (Point of Delivery) - независимый вертикальный срез в Fat-tree или Clos-топологиях, состоящий только из узлов 0-го и 1-го уровня. Связность между различными PoD-ами создается через ToF

## R
- Racktables (RT) - система инвентори сетевой инфраструктуры Яндекс.
- Radix - количество портов в коммутаторе
- Router - это сетевое устройство маршрутизирующее сетевые пакеты, работает на уровне L3 модели OSI.

## S
- Server - сервер
- Spine - коммутатор агрегации, не принадлежащий уровню 0 ([Leaf](#l)). В фабрике возможно наличие нескольких уровней Spine
- Switch - сетевое устройство (коммутатор), коммутирующий сетевые пакеты между логическими сущностями и физическими портами.
- Server Load Balancer (SLB) - сервер-балансирощик трафика

## T
- Top of Fabric (ToF) - набор узлов, которые обеспечивают связь между модулями/PoD-ами и не имеют вышестоящих соседей, т.е. находятся в самом верху фабрики. Не входит в состав какого-то модуля
- ToR (Top of Rack) - стоечный коммутатор, необходимый для подключения серверов, не имеющих нижестоящих соседей. Синоним [Leaf](#l)
- Tun64 - туннелятор трафика IPv6 в IPv4

## U
- Uplink - порт устройства, который подключен к оборудованию более высокого уровня.

  {% cut "Примеры..." %}

  - Uplink ToR - это порт в сторону Spine1
  - Uplink Spine1 - это порт в сторону Spine2

  {% endcut %}

## V
- Virtual Routing and Forwarding (VRF) - механизм изоляции IP-сетей внутри одного сетевого маршрутизатора
