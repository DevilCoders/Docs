## Авторизация
Пока поддерживается единственный метод обмен ssh ключа на OAuth токен

## Памятка по командам
```bash
# интерактивный режим, который открывает vim или то что определено в env DNSCLIENT_EDITOR
# zoneUpdate/--zone-update - пока единственная функция
# --interactive --zone "tt.yandex.net." обязательные опции для интерактивного режима
# -d/--debug показывает подробно что происходит
# -a/--ask планируется как опция для подтверждения действий(пока не реализована)
# --oldapi опция, которая говорит использовать старый xml api, без неё вызовется замоканная зона для эксперементов
./dns-client2 zoneUpdate --interactive --zone "tt.yandex.net." -d -a --oldapi

# режим эксперта, в который передаётся сформированная RR
# --expert - говорит, что в --expression мы ожидаем сформированную RR
# --add/--remove - формирует объект для добавления или удаления
./dns-client2 zoneUpdate --expression "test2.tt.yandex.net. 600 IN CNAME test1.tt.yandex.net." -d -a --expert --add --oldapi

# упрощённый режим в командной строке
# add-txt: делится на add и remove и через дефис указывается тип записи, может быть любой поддерживаемый
./dns-client2 zoneUpdate --expression "add-txt test1.tt.yandex.net. _globalsign-domain-verification=blablabla" -d -a
```

## Структура кода
- всё что лежит в pkg должно быть изолировано от всего остального кода
- всё что лежит в pkg должно стараться минимизировать использование других библиотек
- всё что лежит в pkg должно быть покрыто тестами
- надо рассчитывать, что pkg может быть подключен как библиотека к любому коду в аркадии, но не наоборот
