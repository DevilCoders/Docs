# Общие сведения

L3 manager — это сервис для управления конфигурациями L3 балансеров. L3 балансеры проверяют доступность хостов вашего сервиса и перераспределяют трафик пользователей к серверам, на которых работают ваши приложения.

Основная сущность в L3 manager — _сервис_. Сервис — это ваше приложение, работающее на серверах Яндекса и принимающее пользовательский трафик. Каждый сервис в L3 manager относится к одному из сервисов в [ABC](https://abc.yandex-team.ru/).

Сервисы в L3 manager имеют _конфигурации_ — списки виртуальных серверов, обеспечивающих подключение пользователей к вашим приложениям. У сервиса может быть только одна [активная](statuses.md) конфигурация.

Виртуальный сервер — это группа настроек, определяющая соответствие между:
- IP-адресом, портом и протоколом подключения к вашему приложению.
- L3 балансерами, распределяющими пользовательский трафик.
- _Рилами_ вашего приложения.

Рилы — это сервера в одном из ДЦ Яндекса, на которых запущены копии вашего приложения. Рилы могут быть указаны адресами хостов, группами Кондуктора, Gencfg или RT.

Настройки, определяющие работу L3 manager с вашим приложением, описаны в разделе [Настройка виртуального сервера](config-settings.md).

L3 manager проверяет активность ваших рилов при помощи [проверок доступности](checks.md). На основе проверок доступности вы можете организовать [мониторинги](logs.md) вашего приложения.
