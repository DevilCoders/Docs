# Введение
ЦК (Централизованная Катка) - это сервис для выполнения любого рода работ на сетевых устройствах в сети Яндекса. Данный сервис концептуально представляет собой более высокоуровневую надстройку над существующим тулсетом автоматизации в виде [ann](https://noc-gitlab.yandex-team.ru/nocdev/annushka/) и [Racktables](https://racktables.yandex-team.ru). Помимо этого он интегрирован с инфраструктурной автоматикой остального Яндекса.

Довольно долго основным и единственным нашим инструментом для управления сетевыми устройствами оставалась annushka. Любой инженер, которому требовалось что-то помимо изменения текущей конфигурации устройства или же проведения работ, которые аффектят связность, должен был сам готовить набор команд, которые бы перезагрузили/обновили софт на устройстве, вручную править объект в [Racktables](https://racktables.yandex-team.ru).

ЦК работает поверх существующего стека и становится высокоуровневой точкой входа инженеров, которым хочется автоматизировать любые взаимодействия с оборудованием. Он позволяет зашедулить выполнение однотипных операций, избавляя его от необходимости самостоятельно заходить на устройство и копипастить команды, либо же вручную запускать `ann deploy`
