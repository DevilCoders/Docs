# Сценарии/Kind
ЦК оперирует понятиям _сценария (scenario)_ - набора шагов _экшенов_, которые должны быть исполнены на устройстве. Ближайшая аналогия сценариев - это playbook в Ansible.

_Kind_ - это уникальное _имя/тип_ сценария.
{% note info %}
Понятия "scenario" и "kind" могут выступать синонимами в устной речи, но в контексте использования сервиса, предлагается воспринимать kind как свойство scenario или его экземпляров.
{% endnote %}

Сценарии описываются в виде YAML'a следующего вида

```yaml
name: scenario-name
actions:
- name: add maintenance tag
  racktables_tag:
    state: present
    name:
      - апгрейд или замена
        ...
```

Тело сценария состоит из экшонов, при этом все `actions` выполняются сверху вниз по очереди.  По умолчанию они это делают линейно до завершения работы последнего, но помимо этого экшены могут входить в другие экшены, возможно их условное выполнение, либо циклическое выполнение до достижения некоего условия.

Сценарии могут храниться в двух местах:
- В самом репозитории noc-ck: [app/declarative](https://noc-gitlab.yandex-team.ru/nocdev/noc-ck/-/tree/master/app/declarative)
- Непосредственно в базе ЦК

Как загрузить сценарий в базу можно увидеть в разделе Быстрый Старт.
