# Job
Джоба это инстанс конкретного сценария, запущенный для конкретного устройства. У него есть набор рантайм переменных среди которых основные основные это:

* **kind**: имя сценария который будет выполняться
* **ticket**: имя тикета в Startrek с которым связаны изменения на устройстве
* **state**: текущее состояние джобы, бывает created, scheduled, running, finished, failed
* **ticket**: имя тикета в Startrek с которым связаны изменения на устройстве
* **run after**: время когда джоба должна начать выполнение
* **run before**: время когда джоба должна быть зафейлена если она все еще не выполенена

Список текущих созданных джоб можно посмотреть через noc.yandex-team.ru на вкладке [Jobs](https://noc.yandex-team.ru/jobs?state=planned%2Cscheduled%2Crunning&page=1) либо через `noc-ck-cli list`

{% note info %}

_Scenario_ и _Job_ также нередко упоминаются как синонимы, в том числе в вызовах API (scenario_id <=> job_id).
Согласно определению выше, необходимо воспринимать job как экземпляр scenario, запланированный к выполнению на конкретном устройстве.

{% endnote %}
