# Alexandria

Сервис Alexandria создан для учёта и контроля версий программного обеспечения(далее ПО), устанавливаемого на сетеобразующее оборудование в Яндекс(далее "устройства").  

Основанием для разработки послужило желание решать следующие типы задач:
* Определение типа и версии ПО для нового сетевого устройства при введении его в эксплуатацию.
* Контроль версий ПО на эксплуатируемых сетевых устройствах.

Если коротко: _"какой софт ставить на новую коробку, и надо ли менять софт на действующей коробке"._

### Сервис
В виде API: https://alexandria.yandex-team.ru:443  
[Спецификация](https://alexandria.yandex-team.ru:443/swaggerui/)  

### Позиционирование в процессах эксплуатации и развития сети
Эксплуатация всегда хочет иметь детерменированный набор ПО на сети, так как это является одним из условий достижения желаемого состояния сети.  
Сетевые устройства с правильными настройками и правильным ПО в основе дают детерменированную картину для реализации дизайна сети.

![Network state](network_state.png "Желаемое состояние сети" =840x200)

### Процесс эксплуатации сервиса
**Вход:** Устройства c недетерминированным набором состояний ПО.  
**Выход:** Известно текущее состояние ПО устройств, известно желаемое состояние ПО устройств, зависимые системы по контролю, пусконаладке и обновлению ПО получают необходимые данные от сервиса.

Александрия и зависимые сервисы стремятся к тому, чтобы текущее состояние и желаемое состояние ПО устройств сети совпадали или степень совпадения увеличивалась. 

**Основные компоненты процесса**
* Устройства сети разбиваются на группы согласно списку [Matchers](./hld/match.md)
* Группы указывают на желаемые/допустимые/опасные экземпляры ПО [Soft](./hld/softs.md)
* Модули [признаваторов](./hld/recognizers.md) помогают определять принадлежность состояния известным экземплярам ПО или создавать новые.
