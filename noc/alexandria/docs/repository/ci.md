## Alexandria PR testing
Для облегчения контроля за изменениями, в процесс релиза Александрии введена полуавтоматическая проверка изменений.  
Проверка предполагает: 
* параллельную сборку trunk и PR ревизий
* запуск собранных ревизий Александрии в режиме отчёта, с данными из RT staging
* сравнение трёх видов отчётов
* ручное подтверждение инженера

Виды отчётов для Diff:
* _Matchers_: Оценить изменения матчинга (разбиения устройств на группы)
* _Softs_: Оценить изменения в составе Softs
* _Objects_: Оценить принадлежность устройств к матчерам, с детализацией объектам RT

Ссылка на pipeline находится в карточке Merge Checks вашего PR.
