# Свойства RFC

RFC имеет следующие свойства:

- `name` - Название изменения;
- `type` - Тип изменения;
- `linked-incident` - Связанный инцидент;
- `responsible` - Исполнитель;
- `addtocal` - флаг добавления события в календарь исполнителя;
- `add-approvers` - Флаг наличия дополнительных согласующих;
- `add-approverN` - список логинов N-ной группы согласующих, где N от 1 до 10 включительно;
- `infra` - Подсистема;
- `dc` - Область применения изменения;
- `devices-expression` - Устройства, на которых выполняется изменение;
- `experience` - Предыдущий опыт/успех изменения;
- `chg-auto` - Флаг автоматизированности изменения;
- `chg-test` - Тип проведённого тестирования изменения;
- `service-impact`  - Влияние на сервис;
- `services` - Потенциальное влияние на сервис(ы) (ABC);
- `influence-rollback`
- `date` - Дата начала изменения;
- `date-end` - Дата окончания (заполняется, если отличается от даты начала);
- `time-begin` - Время (начало изменения);
- `time-end` - Время (завершение изменения);
- `description` - Описание изменения;
- `preparation` - План подготовки к изменению;
- `execution` - План выполнения изменения;
- `rollback` - План отката измененения;
- `set-juggler-mute` - Флаг установки мьюта в NOC Heatmap
- `juggler-mute-mondata-help-keys` - Набор mondata help keys для мьюта в NOC Heatmap
