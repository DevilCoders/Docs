# Соглашения

## Краткое изложение идей папки command

- В данной корневой папке предполагается написание скриптов, которые также указываются в package.json
- Данные скрипты должны содержать только последовательные вызовы уже существующих команд из папки utils и общих tools
- Тут можно прокидывать необходимые переменные окружения
- Скрипты в этой директории нацелены на унификацию конкретного проекта, в данном случае website
- Список корневых скриптов в папке command должен повторять (по смыслу и названию) другие проекты, чтобы соответствовать [общей инструкции по работе с e2e тестами](https://wiki.yandex-team.ru/lavka/dev/front/e2e)
