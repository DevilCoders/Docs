# Соглашения api

## О поддержке нескольких проектов

- Чтобы сделать логику mock-server более лёгкой и читаемой, определим свои routers для каждого проекта, который должен поддерживается для mock-server
- Подразумевается, что все controllers и services имеют одну и ту же логику, различаются только маршруты
- Следование этим соглашениям мотивирует оставить одну и ту логику обработки запросов между всеми проектами
