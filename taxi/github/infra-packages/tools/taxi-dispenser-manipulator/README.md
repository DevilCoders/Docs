# О чем это
Штука умеет выдать нужные квоты в диспенсере для баз в MDB. Принимает slug ABC-сервиса с этой базой и три ресурса - cpu, ram, ssd.
Если ни один из ресурсов не указан или желаемые значения не отличаются от текущих, то просто покажет размер квоты и выйдет.
Если отличия есть, то покажет что как меняется и спросит, хочешь ли ты эти изменения накатить.

Cpu измеряются в штуках ядер, ram и ssd - в гигабайтах. Можно указать как абсолютное значение (типа `--cpu 200`), так и относительные (типа `-- cpu +10` или `--cpu -5`).

# Посмотреть размер квоты
`./dispense.py --abc taxi_tmp_quota_migrator` или любой другой интересующий тебя slug ABC-сервиса.

# Изменить размер квоты
1. Открой нужную базу в yc.yandex-team.ru
2. Вытащи оттуда slug ABC-сервиса этой базы (его еще можно найти в админке)
3. Попробуй применить нужные изменения в ресурсах. Это не получится, но в ошибке будет видно, сколько и каких квот нехватает.
4. Запусти команду с полученными параметрами, примерно так: `./dispense.py --abc taxi_tmp_quota_migrator --cpu +3 --ram +12`
5. Следуй инструкциям на экране и помни, что в yc квота прорастает долго

# Перенос квоты между деревьями
1. Проверь, что https://abc.yandex-team.ru/services/taxi_tmp_quota_migrator находится в ветке-доноре.
2. Выдай нужную квоту:  `./dispense.py --abc taxi_tmp_quota_migrator --cpu 30 --ram 120 --ssd 2000`
3. Перенеси в ABC taxi_tmp_quota_migrator в нужное дерево
4. Слей квоту из временного сервиса: `./dispense.py --abc taxi_tmp_quota_migrator --cpu 0 --ram 0 --ssd 0`
5. Верни taxi_tmp_quota_migrator в изначальное дерево
