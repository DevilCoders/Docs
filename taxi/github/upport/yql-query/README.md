Для хранения запросов из Нирваны(и не только). 
---
---
Кубик для получения текста запроса из github
https://nirvana.yandex-team.ru/operation/b5e4a16d-3702-404a-acf0-f23af8317523 

Желательно в названии файла указывать цифры задачи, для которой создан запрос.

Несколько замечаний по запросам из гит:
1. В каждом запросе указываем прагму(даже если он запускается на личном токене).

2. Название файла запроса = названию файла скрипта = названию квери из прагмы
только к названию прагмы добавляется преффикс **st_** если запуск из скрипта
**bot_** если из нашего телеграмм бота (пока тут таких нет, они хранятся в другом месте)

Пример:
* скрипт - **test.py**
* квери  -  **test**
* прагма - 
```
$name_query = 'st_test_9998';
$uuid_query = CAST(RandomUuid($name_query) as string);
$pr =
'{"annotations" = {"taxidmp_task" = "'
|| $name_query
|| '"; "taxidmp_run_id" = "'
|| $uuid_query
|| '"; "backend_type" = "yql"}}';
pragma yt.OperationSpec = $pr;
```

3. Структура файлов запросов, как и скриптов, вроде
startrek/scripts/startrek/edutariff

4. Создавать/править запросы можно прямо в гитхабе, не отправляя пр
