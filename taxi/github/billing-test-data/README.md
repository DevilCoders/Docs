# Набор тестовых данных для billing-testing

Тестовые данные для проекта billing-testing.

## Быстрый старт

* Расположить каталог с тестами на одном уровне с billing-testing
* Запустить тестирование при помощи billing-testing 
   ```shell
   ./run.sh -s billing-accounts 0min
   ```
   Подробнее об опциях в документации проекта billing-testing

## Добавление новых тестовых наборов

Название тестового набора формируется из имени проекта и продолжительности тестов.
```
billing-accounts-10s
```
* Проект: **billing-accounts**
* Продолжительность: **10s**

Аналогично для любого другого сервиса.

При запуске без опций billing-testing будет использовать продолжительность тестов по-умолчанию и все известные ему проекты для поиска доступных сетов.

Например, если поддерживаются billing-docs и billing-accounts, то будет выполняться поиск по шаблонам `billing-accounts-<выбранная продолжительность>` и `billing-docs-<выбранная продолжительность>`.

Стоит отметить, что под продолжительностью тестов понимается совокупность запросов к серверам за выбранный интервал времени, а не время работы самого теста (в текущей реализации время работы часового сета на TaxiVMs составляет ~7 часов).
