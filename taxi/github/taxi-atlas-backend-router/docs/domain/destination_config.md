Структура конфига конечной таблицы для поставки данных:
* cluster_name: str; имя кластера
* table: Раздел конфига, где описана метаинформация о таблицах с данными
    * database: str; имя бд
    * table_name: str; имя таблицы
    * engine: str; имя движка
    * partition: str; партиция
    * index_granularity: str; гранулярность. По умолчанию 8192
    * index_columns: list; индексные колонки(список). Порядок ВАЖЕН. 
    Должна быть как минимум одна
    * sign: str; для Collapsing имя колонки со знаком.
    * version: str; для Replacing имя колонки, по которой идёт логика замещения.
    По умолчанию не прописывается
        
* distributed_table: Раздел конфига, 
где описана метаинформация о distributed таблицах
* columns: список колонок в конечной таблице
    * 'orders_quadkey': имя колонки
        * type: str; тип данных колонки
        * is_null: str; является ли Nullable. Если отсутствует, то false
