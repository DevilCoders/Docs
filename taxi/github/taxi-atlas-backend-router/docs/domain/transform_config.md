Структура трансформационного конфига для поставки данных:
* table_names: далее перечисляются имена таблиц, 
из которых мы будем брать поля
    * 'id': имя исходного поля
        * alias: алиас для названия поля в конечной таблице
* calculated_fields: список полей, которые вычисляются
    * 'orders_quadkey': имя вычисляемого поля в конечной таблице
        * function: str; str - это путь до функции, которая вычисляет значение
        * parameters: содержит список параметров. 
        Может отсутствовать, значит что у функции нет параметров
            * 'lat': имя параметра, который передается в функцию
                * column_name: str; указывается, 
                из какого столбца в исходной таблице брать значение.
                Может отсутствовать
                * value: str; указывается, какое конкретно значение использовать. 
                column_name приоритетнее.
                
* filter_fields: список таблиц и полей в них(в исходном источнике),
 по которым надо фильтровать поля(дату). Если отсутствует, то нет фильтрации
    * 'orders': название таблиц источника
        * '-created_at': списком перечисляются поля для фильтрации 
        в этой таблице
