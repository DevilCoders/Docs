# Работа с контентом <!-- omit in toc -->

В приложении присутствует описание следующих сущностей, связанных между собой:

| Название сущности | Описание                                                                |
| ----------------- | ----------------------------------------------------------------------- |
| Person            | Человек, сотрудник Яндекс Go                                            |
| Project           | Проект или продукт, над которым работают в компании                     |
| Post              | Статья или пост сотрудника Яндекс Go на внешнем ресурсе (Habr, VC, etc) |
| Quote             | Цитата сотрудника Яндекс Go                                             |
| Vacancy           | Вакансия в Яндекс Go                                                    |
| Interview         | Текст о подготовке к интервью по одному из направлений                  |

## Сущности

### Person

Сущности людей добавляются и редактируются путём изменения файла [/content/persons/persons.json](persons/persons.json).
Для добавления сущности следует добавить JSON-объект с [параметрами](#параметры-сущности-person) в произвольное место списка.

#### Параметры сущности Person <!-- omit in toc -->

| Параметр           | Тип       | Обязательный | Описание                                                                              |
| ------------------ | --------- | ------------ | ------------------------------------------------------------------------------------- |
| `id`               | `string`  | да           | Уникальный идентификатор человека. Ожидается, что будет совпадать с логином на Стаффе |
| `first_name`       | `string`  | да           | Имя человека                                                                          |
| `last_name`        | `string`  | да           | Фамилия человека                                                                      |
| `position`         | `string`  | да           | Должность человека в Яндекс Go                                                        |
| `is_current`       | `boolean` | да           | Является ли человек нынешним или бывшим сотрудником                                   |
| `is_top`           | `boolean` | да           | Является ли человек топ сотрудником, чей аватар будет в карусели                      |
| `avatar_url`       | `string`  | да           | Ссылка на изображение-аватар                                                          |
| `contacts`         | `object`  | нет          | Объект со ссылками на контакты человека                                               |
| `contacts.twitter` | `string`  | нет          | Ссылка на twitter                                                                     |

> NOTE: аватарки складываем в директорию [/public/avatars](../public/avatars).
> Лучше всего, чтобы имя файла с аватаркой совпадало с `id` сущности [Person](#person)

### Project

Сущности проектов создаются путём добавления [.mdx][mdx] файлов в директорию [/content/projects/](projects).
Имя файла одновременно является уникальным идентификатором проекта.

#### Параметры сущности Project <!-- omit in toc -->

Параметры проекта указываются в начале файла между разделтелями `---` в формате [YAML][yaml]. Например

```md
---
order: "1"
name: "Яндекс Еда"
---
```

| Параметр | Тип      | Обязательный | Описание                                               |
| -------- | -------- | ------------ | ------------------------------------------------------ |
| `order`  | `string` | да           | Порядковое число, используется для сортировки проектов |
| `name`   | `string` | да           | Человекочитаемое название проекта                      |

#### Подробное описание сущности Project <!-- omit in toc -->

Полный текст проекта указывается после параметров, в [формате Markdown](#формат-markdown).

### Post

Сущности постов добавляются и редактируются путём изменения файла [/content/posts/posts.json](posts/posts.json).
Для добавления сущности следует добавить JSON-объект с [параметрами](#параметры-сущности-post) в произвольное место списка.

#### Параметры сущности Post <!-- omit in toc -->

| Параметр | Тип              | Обязательный | Описание                                                                             |
| -------- | ---------------- | ------------ | ------------------------------------------------------------------------------------ |
| `id`     | `string`         | да           | Уникальный идентификатор статьи. Будет использоваться в внутреннем URL поста         |
| `url`    | `string`         | да           | Ссылка на внешний ресурс, на котором опубликован пост                                |
| `tags`   | `Array<string>`  | да           | Список тематических тегов для поста, e.g. ['b2b', 'backend']                         |
| `title`  | `string`         | да           | Заголовок поста                                                                      |
| `source` | `'habr' \| 'vc'` | да           | Внешний ресурс, на котором опубликован пост                                          |
| `author` | `string`         | да           | Идентификатор автора поста. Должен совпадать с полем id в сущности [Person](#person) |
| `date`   | `string`         | да           | Дата написания поста в формате [ISO8601][iso8601]                                    |

### Quote

Сущности цитат добавляются и редактируются путём добавления подкаталогов, названия которых должны совпадать с id автора цитаты, в каталог /content/quotes и последующего добавления в эти подкаталоги файлов с расширением .mdx. Имена этих файлоа должны совпадать с логической сутью цитат. В итоге возникает структура вида: /content/quotes/id автора цитаты/название*самой*цитаты.mdx.

#### Параметры сущности Quote <!-- omit in toc -->

| Параметр | Тип      | Обязательный | Описание                                                                                                                                                  |
| -------- | -------- | ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `id`     | `string` | да           | Уникальный идентификатор цитаты.<br />TODO: подумать над форматом, пока это просто порядковые числа. Хорошо бы придумать более содержательный формат      |
| `author` | `string` | да           | Идентификатор автора цитаты. Должен совпадать с полем id в сущности [Person](#person).<br />Возможно, должно быть `string \| object`, как в [Post](#post) |
| `quoute` | `string` | да           | Текст цитаты                                                                                                                                              |

### Vacancy

Сущности вакансий создаются путём добавления [.mdx][mdx] файлов в директорию [/content/vacancies/](vacancies).
Имя файла одновременно является уникальным идентификатором вакансии и используется в URL на вакансию. Например, при добавлении файла `frontend_dev.mdx` в директорию, создаётся URL `/vacancies/frontend_dev`. На странице `vacancies` также появляется карточка этой вакансии.

#### Параметры сущности Vacancy <!-- omit in toc -->

Параметры вакансии указываются в начале файла между разделтелями `---` в формате [YAML][yaml]. Например

```md
---
title: "Frontend разработчик в Яндекс Go"
description: "Привет. Я ищу к себе в команду Яндекс Go Frontend разработчиков!"
date: "2021-06-21T21:00:00+03:00"
url: "https://yandex.ru/vacancies/frontend-dev"
hiring_manager: "tsarev"
cities:
  - spb
  - moscow
professions:
  - mobile
  - frontend
---
```

| Параметр         | Тип             | Обязательный | Описание                                                                          |
| ---------------- | --------------- | ------------ | --------------------------------------------------------------------------------- |
| `title`          | `string`        | да           | Название вакансии                                                                 |
| `description`    | `string`        | да           | Краткое описание вакансии                                                         |
| `url`            | `string`        | да           | URL на внешний ресурс с описанием вакансии                                        |
| `hiring_manager` | `string`        | нет          | Идентификатор сущности [Person](#person), к которой привязана эта вакансия        |
| `cities`         | `Array<string>` | да           | Массив с [идентификаторами городов](#города), в которых ведется набор на вакансию |
| `professions`    | `Array<string>` | да           | Массив с [идентификаторами профессий](#професии)                                  |
| `date`           | `string`        | да           | Дата публикации вакансии в формате [ISO8601][iso8601]                             |

#### Подробное описание сущности Vacancy <!-- omit in toc -->

Полный текст вакансии указывается после параметров, в [формате Markdown](#формат-markdown).

# Формат Markdown

Из-за технических особенностей `next` ссылки прописываем так:

- ссылка, открывающаяся в текущей вкладке `<a href="https://www.yandex.ru" rel="noreferrer">`
- ссылка, открывающаяся в новой вкладке `<a href="https://www.yandex.ru" rel="noreferrer" target="_blank">`

В остальном придерживаемся формата [Markdown][md-cheatsheet].

# Content style

В названиях проектов и отделов используем латинское название (например, `Яндекс Pro`, а не `Яндекс Про`).
Слова `iOS-` и `android-` прописываем латиницей и через дефис (например, `iOS-разработчик` и `android-разработчик`).
Между короткими предлогами или короткими словами в названиях стараемся использовать неразрывный пробел (например, `Яндекс&nbsp;Pro` и `в&nbsp;Яндекс`)
