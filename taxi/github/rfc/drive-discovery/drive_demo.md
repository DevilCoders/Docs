## Flow


#### Главный экран

В эксперимент drive_discovery добавляем флаг `demo_mode_enabled`. 

Если выполнено хотя бы одно из условий:
1. Пользователь неавторизован в Такси (даже фонишем)
2. Пользователь незалогинен порталом
3. Пользователь незареган в Драйве

и при этом `drive_discovery.demo_mode_enabled == true`, то при нажатии на 
турбо-кнопку Драйва необходимо открыть экран дискавери драйва в демо-режиме.

#### Экран discovery Драйва в демо-режиме
Экран discovery Драйва в демо-режиме похож на обычный экран discovery Драйва.
Отличия:
1. В демо-режиме нет заголовка экрана (иконка + "Драйв")
2. В демо-режиме открывается карточка SDK Драйва (см. дизайн)
3. В демо-режиме в запросе в layers указывается флажок `state.drive.demo_mode=true`.

Пример запроса:
```json
{
  "state": {
    ...,
    "mode": "drive",
    "screen": "discovery",
    "drive": {
      "demo_mode": true
    },
    ...
  }
}
```
На экране должен быть поддержан `optimal_view`, но в отличие от обычного 
 режима Драйва не нужно отображать ошибки
`no_objects_message/no_objects_subtitle` (т.к. снизу экрана будет карточка SDK).

Если layers не отвечает, либо 500-тит, то ничего делать не нужно. 
Если пользователь сам начал двигать карту, то нужно снова сходить в layers.

Машинки приходят с тем же экшеном, что и в обычном режиме дискавери. SDK
сам понимает, что нужно открыть карточку замоканной машинки.
