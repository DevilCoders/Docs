**Название сервиса** 

signal-device-external-api

**Какую продуктовую проблему решает сервис?**

Добавление внешнего API для интеграций

**Почему нельзя решить эту задачу без разработки нового сервиса существующими решениями?**

Логику интеграции с сервисами внешних клиентов нужно выделить в отдельный failure-домен

**Как именно сервис будет решать поставленные перед ним задачи?**

1. Клиент сервиса авторизуется по API-ключу. Для этого используется uapi-keys
2. После авторизации клиент может получать нужную ему информацию с помощью ручек signal-device-external-api, которые ему доступны

**Разрабатывается один сервис или система?**

Один сервис

**С кем взаимодействует сервис?**

1. С signal-device-api-admin - для получения событий и возвращения наружу [в первой итерации]
2. С uapi-keys - для авторизации клиентов по API-ключу

**Какая нагрузка ожидается?**

5 RPS на старте с линейным ростом по количеству SignalQ парков

**Какие фолбеки предусмотрены на сам этот сервис?**

Если сервис недоступен, не не отдаём сервисам внешних партнеров данные SignalQ

**Какие фолбеки предусмотрены внутри этого сервиса на взаимодействие с другими сервисами?**

Если недоступен signal-device-api-admin или uapi-keys, не отдаём внешнему миру данные SignalQ

**Какие возможности масштабируемости закладываются?**

Увеличение ram/cpu на инстансе, увеличение количества инстансов сервиса. Могут появиться кэши с наиболее часто запрашиваемыми данными

**Какие точки отказа есть в сервисе?**

- Отказ signal-device-api-admin
- Отказ uapi-keys

**Укажите ключевые продуктовые метрики сервиса, за которыми планируете следить**

- Количество партнёров, которые пользуются сервисом

**Укажите технические метрики**

- Количество запросов
- Максимальное и среднее время ответа на запрос

**Какая функциональность ожидается в сервисе в будущем?**

По мере надобности будут добавляться новые ручки для взаимодействия с клиентами

**Какое изменение нагрузки планируется?**

Прямо пропорционально количеству партнеров и количеству камер

**Активно ли будет изменяться сервис?**

Будем расширять функционал под конкретные кейсы интеграции
