# 4.0/payments/v1/list-payment-methods



## location_info

Для оплаты заказов xPay-методами клиентам необходимо предоставить region_id, country_code и currency_code.

В некоторых сервисах (например, Еда/Лавка) данная информация возвращается в цикле заказа (`/preorder`), однако в некоторых сервисах (Самокаты) эта информация в цикле заказа отсутствует.

Предлагается в корень 4.0/lpm добавить информацию зависяющую от локации:

```json5
...
    "location_info": {
        "region_id": 255,
        "country_code": "RU",
        "currency_code": "RUB"
    }
...
```

Поле `location_info` опционально. Поля `region_id`, `country_code`и `currency_code` — обязательные

Плюсы:
* ничего не надо писать на бекенде
* на клиентах пользуемся готовыми механизмами

Минусы:
* +1 ручка для цикла заказа Самокатов
* Зависимость от таксишных зон в Самокатах

Отказались т.к. завязка на таксишные зоны

Плюсы:
* Не расширяем протокол
* Избегаем +1 ручки

Минусы:
* В какой-то момент клоз может не хватить для описания логики

Отказались т.к. кажется что можно сразу в протокол положить
