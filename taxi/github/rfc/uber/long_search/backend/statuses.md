# Статусы поиска(эксперимент)

Суть: необходимо показывать нотификационные пузыри на клиенте если поиск затягивается на 1 минуту или больше от среднего

## Откуда
Эксперимент из `totw`

## Что
Эксп `uber_long_search_notfications`
```json
{
  "notifications": [
    {"promotion": "promotion_id1", "delay": 60}, 
    {"promotion": "promotion_id2", "delay": 120}
  ]
}
```

## Структура нотификации
При реализации с [promotions](https://tariff-editor.taxi.yandex-team.ru/images) (можно фильтрануть по notifications для примера) - всю необходимую информацию похоже можно разместить в promotion(доступ по promotion_id)

## Где хранится картинка
Yandex Go Communications(сервис `promotions`, [админка](https://tariff-editor.taxi.yandex-team.ru/promotions)) и там же похоже можно хранить текст, ключи танкера, даже что-то вычислять

## Next step
Есть предложение реализовать цепочку нотификаций сменяющих друг друга как один promotion-notification с набором слайдов и тайминги брать из таймингов слайдов. В таком случае как будто бы можно даже лучше организовать нотификации(в т.ч. если потребуется сделать четко связанную цепочку)
