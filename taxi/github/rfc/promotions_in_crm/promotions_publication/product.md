## Продуктовая проблема

Сейчас маркетологи не имеют простого инструмента для настройки аудитории публикации. 
Поэтому они могут или раскатывать на всех, или просить помочь менеджеров или разработчиков настроить эксперимент 
или опубликовать по YQL. 
Самый частый вариант раскатки сейчас - публикация по YQL, он неудобен.

Ещё одна проблема кроется в том, что публикация по эксперименту подходит всё меньше, потому что Такси уже очень большой бизнес. 
Надо уметь выбирать аудиторию более точечно, чем это обычно делается через эксперимент (пример: выбрать тех, 
у кого за последние 3 дня была поездка в Ультиме в Москве).

## Решение

Мы попытаемся сделать настройку аудитории возможной силами маркетологов. 
Для этого:

1. Перенесём используемые сейчас настройки аудитории в интерфейс CRM.
2. Сделаем публикацию по списку пользователей из CRM, заменяющую сценарий публикации по YQL.

Движемся к такому: самые частые сценарии можно настроить через CRM, для сложных случаев можно оставить редактирование 
эксперимента и YQL в админке коммуникаций. Возможно, если решение для CRM будет учитывать все текущие потребности CRM, 
редактирование в админке останется только для локальных продуктовых коммуникаций, которые запускают менеджеры продукта. 

Переносить настройки будем пока только для фуллскринов. 
Выбрали фуллскрины, потому что их уже можно настраивать через CRM, 
и это самый часто используемый и важный тип коммуникации, который через неё настраивается.
