## Задача:
https://st.yandex-team.ru/TAXIPROJECTS-2757

## Проблема:
В некоторых городах работы в РФ на самокатах не установлены замки => не нужно отображать кнопку "Открыть замок" на карточке поездки. В Израиле нужно отображать кнопку "Открыть шлем" и адаптировать цикл заказа под wind.

## API
### Предложение:
В ответе ``/sessions/current`` ввести новое поле ``cars[i].features``, пример его содержания: 
```(json)
{
    "sessions": [...]
    "cars": [
        {
            "number": "123",
            "location": {...},
            "telematics": {...},
            "features": { // новое поле
                "cable_lock": {
                    "available": true
                },
                "helmet": {
                    "available": false,
                    "required": false
                },
                ...
            }
        }
    ]
}
```
В зависимости от того, что лежит внутри ``features``, клиенты будут понимать, какие кнопки отображать на карточке (пример: "открыть замок"). 

### Какие еще могут быть способы:
1) В эксперименте [scooters](https://tariff-editor.taxi.yandex-team.ru/experiments3/experiments/show/scooters/current) добавить флажки, которые будет принимать разные значения в разных зонах. Минус: вообще говоря, наличие замка — это свойство самоката, а не города, такая штука может быть плохо расширяема.
2) Унести конфигурацию кнопок на экране заказа с клиента на бэкенд. Бэкенд будет сам в зависимости от каких-либо условий возвращать список кнопок, а клиенты будут его только отображать. Плюсы: можно будет конфигурировать наборы кнопок без разработки на клиентах (не уверен, что очень нужно), дешевле добавлять новые кнопки. Минусы: потребует много клиентской разработки в начале.

## Логика наличия замка на самокате:
1) Верить телематике, честно получать факт наличия замка от нее. Минус: не знаем, насколько можно верить телематике.
2) Привязать логику замков к "городским" тэгам. Минус: нужна ручная разметка при перевозе самоката в другой город (но она уже есть для тарифов).

В реализации оба варианта занимают примерно пару дней, и оба имеют проблемы. Можно написать периодик, который будет сравнивать оба значения и алертить расхождения. В итоге использовать тот, что лучше работает.
