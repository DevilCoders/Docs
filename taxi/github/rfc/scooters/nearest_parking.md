# Поиск ближайшей парковки при завершении поездки
[ITAXI-16198](https://st.yandex-team.ru/ITAXI-16198)

## Логика
При попытке завершения поездки вне парковки в диалоге ошибки будет кнопка "Найти ближайшую парковку". По нажатию на которую будет находиться ближайшая к локации пользователя парковка и до нее построится маршрут.
Если пользователь находился на экране мульизаказа, то мы при нажатии на кнопку сначала переходим в детальный экран, после уже отправляем запрос в `/v2/objects`

## Изменения в бэке в слоях
Есть идея дополнить ручку `/v2/objects`
Мы находимся на экране `totw` при возникновении ошибки и нам нужно получить id ближайшей парковки, отобразить ее и построирть до нее маршрут.

По тапу на кнопку "Найти ближайшую парковку":
- в `v2/objects` передается контекст
{
    //...
    "screen": "totw"
    //...
    "scooters": {
        "destination": [/*lon*/, /*lat*/]
    }
}
- в ответ отправляется парковка, у которой будет экшен `build_route_navigation`, а также поле `selected_object_id`, чтобы клиент программно тапнул на эту парковку
{
      // ...
      "selected_object_id": "<some_object_id>",
      "features": [
          {
              // ...
              "id": "<some_object_id>",
              "properties": {
                  // ...
                  "options": [
                      {
                          // ...
                          "actions": [
                              // ...
                              {
                                  "type": "build_route_navigation"
                              }
                          ]
                      }
                  ]
              }
          }
      ]
  }
  
## Изменения на клиенте
- При завершении поездки вне парковки (если в `/evolve` приходят ошибки `invalidParkingZone` или `parkingForbidden`) в диалог ошибки добавится кнопка "Найти ближайшую парковку".
- По нажатию на эту кнопку будет отправлен запрос в `/v2/objects` с указанием локации пользователя. Если были на экране мультизаказа, то перед отправкой надо перейти в детальный экран.
- В ответ получим id ближайшей парковки (`selected_object_id`) с экшеном `build_route_navigation`.
- Добавить обработку нового экшена: построение маршрута.
- При отмене маршрута убрать `destination` из `scooters` и перезапросить `v2/objects`.
