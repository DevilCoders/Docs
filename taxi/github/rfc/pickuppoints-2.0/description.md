# Pickuppoints 2.0.

## Какие есть проблемы?

* Неясная структура ответа ручки `/zones` (матчинг choices по имени это плохо);
* Невозможность адресоваться к конкретной зоне/пойнту извне сервиса;
* При переходе на persuggest продолбали коллекцию `geoobjareas` -- список вокзалов/аэропортов.

## Как решим?

* Добавим URI для зон/точек и будем их отдавать из zones, points и т.д.;
* Переделаем ответ zones на более иерархичный в `/zones/v2`;
* Новую ручку для получения списка зон как suggest item-ов -- замена geoobjareas (надо продумать);
* Новую ручку для получения локализации пойнта по URI (возможно, слить с предыдущим пунктом) localizeaddress;
* . . .

## Какие есть вопросы?

* Как генерить uid зон, точек? Uuid?
* Как генерить uri зон, точек? По uid-у + так, чтобы можно было по точке понять зону/мб чойс.
* Какой префикс uri? ytpp -- Yandex Taxi Pickup Points -- ok?
* Какая правильная схема uri ручек? `/zones/v2` -- ok?
