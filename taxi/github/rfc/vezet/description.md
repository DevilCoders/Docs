как я вижу процесс интеграции:
1) притаскиваем данные в каком-то формате
2) заливаем их в YT
3) натравливаем аналитиков с нирваной на данные - проверить связи заказ-пассажир-водитель-машина, проверить данные на валидность,
добавить плейсхолдеры там где не хватает, дропнуть мусорные записи
4) добавляем данные в наши таблицы для тестинга в YT
5) проверяем что оно работает
6) добавляем данные в наши таблицы в проде


Из сложностей:
1) userplaces нужно добавлять в оперативные базы (заказы можно добавлять прямо в YT)
2) нужно создать phone_obj для всех пользователей из выгрузки перед заливанием заказов/избранных
3) историю нужно зареплицировать в phone_history
