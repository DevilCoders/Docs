# Изменения ручки 4.0/lpm

## Проблема
- Для рекуррентных платежей не подходят токинезированые платежи, из-за их коротного срока жизни (WIND)

## Решение
- В `4.0/lpm` Добавляется новый query параметр `feature`, который решает проблему разных методов оплаты для сервиса

Возможные варианты `fature` для сервиса `WIND`:
- `passes`
- `topup`
- `order`

## Как это работает на бэкенде
- Влияет на список доступных методов оплаты (запрос в available/payment/types)
- Влияет на последний метод оплаты (будет возвращаться именно для данного `feature`)

## Что это значит для клиентов
- На бэкенде уже есть логика, что если не передан никакой `feature`, автоматически срабатывает фолбек на `order`
- Поэтому, сейчас на клиентах используем данный query параметр только для экрана с покупкой `free/super пассов` со значение `passes`

Пример url-а: `POST /4.0/payments/v1/list-payment-methods?service=grocery&feature=passes`
