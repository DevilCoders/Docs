Хотим объединить разные требования Детских кресел в одно, чтобы клиенты могли сохранять выбор между категориями (в том числе в Детской вертикали)

## Как сейчас (пример конфигурации)
В Москве есть 2 требования double_childchair_moscow в К+ и Бизнесе и childchair_for_child_tariff в Детском и Минивэне. Отличаются тем, что в последнем можно заказать 2 кресла (в первом есть только одно)
Из-за того что требования в категориях разные не сохраняется выбор кресла между переключениями
Определенная разница есть в случаях переключения на категории без кресел:
Если переходим с Детского/Минивэна то игнорируем пожелание, если с K+/Бизнеса -- пишем "сбросить НП"
разница в tariff_specific флаге.

## Что предлагается
Делать одно детское требование childchair_v2 с возможностью разных параметров в 
```
max_tariffs[].supported_requirements["childchair_v2"].tariff_specific
max_tariffs[].supported_requirements["childchair_v2"].max_weight
max_tariffs[].supported_requirements["childchair_v2"].options[].weight/max_count
```
А также новое поле для приоритетов при сбросе опций
```
max_tariffs[].supported_requirements["childchair_v2"].options_drop_sequence: ["booster", "chair", "infant"]
```

А именно, по наличию одноименного supported флага в запросе zi и наличию экспа бекенд может заменить старые требования на новое childchair_v2 которое имеет разные лимиты в разных категориях.

## Логика клиента
При каждом переключении категории выбор multiselect требований проходит валидацию и если она не проходит, то либо
1. (в новом флоу requirements_v2) По определенному приоритету сбрасывается часть опций пока валидация не пройдет. Сброшенные опции отображаются как бабблы НП и используется соответствующий числу НП попап. 
2. (в старом флоу) Плашка "Сбросить НП" после которой сброшены будут только лишние опции.
   ВАЖНО Чтобы правильно работал старый флоу в routestats, надо будет добавить соотв строки в TARIFF_SPECIFIC_OVERRIDES

## Эксперимент childchair_v2
В zi уже есть возможность экспом отключать показ требований (старые кресла), но в нашем случае *в том же экспе* нужно также иметь возможность включать другое (новое кресло).
ВАЖНО чтобы по умолчанию новое требование не было показано (даже если оно есть в настройках зоны),  эксп должен иметь default содержащий это требование:
```
{
  "enabled": false,
  "requirement": "childchair_v2"
}
```

Пример значения эксперимента childchair_v2:
```
{
  "enabled": true,
  "requirement": "childchair_v2",
  "replace": ["double_childchair_moscow", "childchair_for_child_tariff"],
  "overrides": {
    // поддерживается __default__ если во всех категориях в зоне общие ограничения 
    "comfortplus": {
      "max_weight": 3,
      "tariff_specific": false,
      "options": {
        "infant": {"weight": 2, "max_count": 1},
        "chair": {"weight": 2, "max_count": 1},
        "booster": {"weight": 1, "max_count": 2},
      },
      "options_drop_sequence": ["booster", "chair", "infant"]
    }
    ... //  другие категории
  }
}
```
