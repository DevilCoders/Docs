## Бизнесовая задача

### Описание

Хотим давать скидку на заказы по тарифу "Курьер"/"Доставка" для заказов из постаматов boxberry.
В будущем может добавиться pickuppoint и другие партнеры.
Сейчас эти заказы совершаются через бэкенд cargo.

### Гипотеза

Сейчас это промотивирует пользователей использовать тарифы "Курьер"/"Доставка".
Так же впоследствии, это позволит нам гибче настраивать скидки (сейчас нет никакого явного признака для таких заказов).


### Продуктовые ограничения

Для добавления новых "лейблов" могут потребоваться доработки на стороне клиентов.
В данной задаче (CARGODEV-706) это будет бэкенд cargo.


## Решение

Добавлять в отдельное поле явный "лейбл" для поездки/заказа пользователя.

### API

Ручки `/v1/orders/estimate`, `/v1/orders/draft`:

В запрос добавляем поле `partner_tag` со следующим типом
```
partner_tag:
  description: |
    тег партнерской поездки. Например, boxberry или pickuppoint
    можно указывать кинотеатры/театры (для будущих интеграций)
  type: string
```


### Детали реализации

Данное поле будет прокидываться из запроса в сервис discounts через pricing-data-preparer.
В сервисе discounts будет производиться проверка настроек скидки на указанные в запросе ride_properties.
