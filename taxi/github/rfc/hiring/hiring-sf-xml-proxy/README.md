#### Название сервиса
hiring-sf-xml-proxy

#### Задачи сервиса
##### Какую продуктовую проблему решает сервис?
Отправка данных из sf в yt с малой задержкой и без траты api calls

##### Почему нельзя решить эту задачу без разработки нового сервиса существующими решениями?
Сейчас нет сервиса который умел бы парсить xml от SF. Забирать данныые из SF поего апи будет траатить api calls.

##### Как именно сервис будет решать поставленные перед ним задачи?
Сервис на лету будет парсить данные отправленные из salesforce и передавать их в репликацию.
Будет конфиг на основе которого будет преобразовывать персональные данные в id, также на лету. В репликацию будет лить уже id.
