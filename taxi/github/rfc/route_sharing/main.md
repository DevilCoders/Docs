### Изменения в шаринге маршрутов

#### Проблемы
- после завершения заказа нельзя посмотреть маршрут

#### Текущий алгоритм работы страницы шарилки
запросы в sharedroute завершаются по получению финального статуса
запросы в sharedroutetrack ретраются до тех пор, пока не завершатся ошибкой

#### Предлагаемое решение
на финальный статус sharedroute сделать единоразовый поход в sharedroutetrack -
с существующим полем "use_full_route":true

существующее поле use_history можно выпиилить, так как оно не нужно

При отправке use_full_route в ответе sharedroutetrack должен вернуться ответ следующего вида:
```(json)
{
  "completed_route": [
        [
            37.642030896700348,
            55.733634362409958
        ],
        [
            37.642017364501953,
            55.733627319335938
        ],
    ...
  ]
}
```
То есть основная идея в том, чтобы не было блока driver, а был просто совершенный трек
