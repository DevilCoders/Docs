#### Формирование version

Важное замечание: мобильным клиентам не важно содержимое поля version
Предлагается формировать поле version из следующих значений: <revision>:<position>:<locale>, где
* _revision_ - ревизия эксперимента
* _position_ - позиция клоза, в который попал пользователь на основе кваргов
* _locale_ - локаль пользователя, так как от этого может зависеть l10n структура

После реализации https://st.yandex-team.ru/TAXIPROJECTS-1405 потребуется небольшое переосмысление формата version

#### Выкатка на престейбл
В случае если клиент прислал _client_revision_ > _backend_revision_ - отправляем на клиент статус _not_modified_

Ключевым условием выполнения такой логики - является условие, что откат эксперимента с престейбла требует повышения ревизии эксперимента
