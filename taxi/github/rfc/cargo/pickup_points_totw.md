## Задача

Заказы из ПВЗ (см ./pickup_points.md) прорастают в приложение пассажиру
При этом он видит стандартный набор кнопок: "Уже иду", "Добавить точку", "Сменить способ оплаты" и тд.
Большинство этих действий должно быть недоступно для таких заказов.
Основная проблема в кнопке "Отмена". После того, как водитель забрал груз, мы не должны
давать пользователю возможность отменить заказ (на самом деле уже не даем, валидация в /orders/cancel).
Хочется иметь возможность самим управлять показом этих кнопок для наших заказов


### Решение

1. В сервисе cargo-claims пишем ручку, которая по id нашей заявки (order.request.cargo_ref_id)
   и списку всех возможных allowed_changes вернет информацию о том, какие из них допустимы. 
   Помимо этого, вернет флаг cancel_disabled, говорящий о возможности отмены.

2. В totw начинаем ходить в эту ручку параллельно со стандартной логикой. Поле cancel_disabled в ответе
   totw перетираем полем, полученным от нас. Поле allowed_changes в ответе totw фильтруем по тем, которые
   разрешены (из ответа ручки)
  
3. Фоллбек на дефолтное поведение

4. Производительность: поход в ручку только для наших заказов с cargo_ref_id. 
   Ожидаемый 98-pr нашей ручки - 80мс (ручка еще не написана, не могу предоставить более точных и
   обоснованных данных)
