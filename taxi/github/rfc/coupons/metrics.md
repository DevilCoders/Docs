# Метрики сервиса coupons

Описание метрик, фиксируемых в сервисе купонов.

## coupon_finish

* Фиксирование количества успешно и неуспешно завершенных заказов с использованием промокода. ```(time)```

## generate

* Фиксирование случаев, когда промокод не может быть сгенерирован, т.к. длина серии превышает или равна максимально возможной длине промокода. ```(crit.)```
* Фиксирование случаев, когда промокод не может быть сгенерирован, т.к. количество разрешенных перегенераций превысило лимит (промокод перегенерируется в случае коллизии и в случае 2 перегенераций отправится нотификация). ```(crit.)```
* Фиксирование количества сгенерированных промокодов с разделением по сервисам, запросившим генерацию. ```(time)```

## referral_complete

* Фиксирование количества успешно и неуспешно завершенных поездок по реферральным промокодам с разделением по странам. ```(time)```

## coupons_activate & coupon_check & coupons_list

* Фиксирование количества успешно проверенных промокодов с разделением по виду (multi/single/referral). ```(time)```
* Фиксирование количества ошибок при проверке промокодов с разделением по виду (multi/single/referral). ```(time)```

## referral

* Фиксирование количества сгенерированных реферральных промокодов с разделением по странам. ```(time)```

```
(time)- измерение метрики происходит во времени (некоторое кол-о фиксаций в промежуток времени).
(crit.) - при фиксации метрика требует индивидуального рассмотрения. Необходимо настроить уведомления для быстрого реагирования на такие случаи.
```
