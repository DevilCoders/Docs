## Бизнесовая задача

Сервис отвечат за мультизаказ.
Пока в нем только информационные ручки, но возможно, будет что еще из протокола в него вытащить.

## Архитектура

### API

Ручка `GET /v1/order_info`:

```json
{
    "order_id": "some_order_id"
}
```

```json
{
  "order_name": "Второй заказ",
  "can_make_more_orders":"not_modified",
  "no_more_orders_reason": "",
}
```

### v1/order_info
Если мультизаказ, возвращает имя заказа. И информацию, можно ли сделать еще один заказ.
####  Fallback
Можно не показывать. 

### Технические детали и ограничения
1. Информация о том, мультизаказ ли, лежит в order_proc.
