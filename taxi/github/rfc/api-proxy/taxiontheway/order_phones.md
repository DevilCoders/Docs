## Бизнесовая задача

План в том, чтобы сервис хранил контактную информацию, связанную с заказом.

Сюда же можно забрать и информацию о софтсвитче для телефона пассажира и бэк таксометра будет ходить сюда за ним.
Вероятно, понадобится своя база, которая заберет себе forwarding из прока.
В начале можно просто иметь одну ручку, которая будет пользоваться проком.

Если решим, что сервис с таким функционалом нам не нужен, то проще объединиться с сервисом performer_info. Потому что отдельный сервис, который просто берет две строчки из прока это явный оверхед.

## Архитектура

### API

Ручка `GET /v1/driver_phone`:

```json
{
    "order_id": "some_order_id",
    "driver_id": "some_driver_id"
}
```

```json
{
    "forwarding": {
      "ext": "9972",
      "phone": "+79671503270"
    },
    "phone": "+79671503270,,9972",
}
```

### v1/driver_phone
Фактически, забирает данные из order_proc.candidates.forwarding
####  Fallback
Можно не показывать. Но вообще учитывая простоту, тут ломаться нечему.
