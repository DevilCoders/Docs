### Гибкий экран деталей поездки

Сейчас экран полностью захардкожен на полях из ответа totw. Каждая секция и каждая строка строится или не строится в зависимости от того, есть определённое поле в ответе или нет. Предлагается сделать более гибкую схему построения экрана для того, чтоб добавление новой информации не требовало доработок со стороны клиентов.

#### MVP

В MVP делаем только одно изменение: конструктор простых строк в секции. "Простая строка" в данном контексте - это строка без экшенов (копирования в буфер обмена, показа информации о водителе, переход на экран набора телефона). Прилинкованность строки к секции ("Водитель", "Партнёр", "Перевозчик", ...) ___пока___ всё также захардкожена. В будущем, при необходимости, можно будет проработать механизм, аналогичный тому, что используется в `/4.0/mlutp/v1/products` для построения сетки шорткатов.
