### zone_name
Понятно, что берем. Но может не быть, поэтому необязательна.

### point
Может придти вместо zone_name. Естественно, берем.

### options
Включает отображение карточки тарифа. Берем, необязательна.

### size_hint
обязательна для картинок, берем. Прям обязательна, картинок без нее не будет. Берем, необязательна.

### skin_version
нужна для картинок. Необязательна, но нужна. Берем, необязательна.

### supported
Нужна только для еул и zone_mode. Не берем.

### capabilities
Передаются отдельными элементами.

Красивого общего решения нет:
 - делать его через additionalProperties: type: bool, нарушая стайл гайд в новом сервисе не хочется
 - перегонять флаги в массив в api-proxy тоже нехорошо

Забираем только те, что нужны. общее решение не делаем. 
Дальнейшее появление подобных флагов баним и заставляем переезжать в supported.
Берем:
 - kSupportsHideableTariffs - берем supports_hideable_tariffs
 - kSupportsRequirementTitle - берется на основе версии, по идее есть в pa_auth_context от api-4.0
 - kSupportsMultipleBrandings - тоже на основе версии

### supported_vertical_types
Очевидно, нужна только для вертикалей. Не берем.
