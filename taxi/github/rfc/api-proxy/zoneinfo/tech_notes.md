## Ответы на вопросы
### Уносим zoneinfo за api-proxy
Тотв проверяет авторизацию через db.users и это не менялось void Data::CheckAuth(). То есть, я не нашел, чтобы в тотв что-то меняли ради api-proxy.
В zoneinfo мы тоже просто берем юзера из db.users и даже не проверяем ничего.

### requirements
С разговора: "мы выкидываем из ответа все требования, которые выкидываются под каким-то условием. Возможно, будут исключения.
что-то себе, что-то в апи прокси, надо по каждому требованию отдельно смотреть"

Отключение требований экспериментом делали в этой задаче
https://st.yandex-team.ru/TAXIBACKEND-25744
Есть всего одно требование такого типа и эксперимент выключен. То есть оно не отфильтруется.

hourly_rental сейчас включен, но мне в Москве не вернулся. Не стал разбирать почему.

### images
Со встречи: "Без картинок будет некрасиво, весь саммари сломается." Настя скинула скриншот, на котором не сломалось: клиент грузил из зашитых в приложение.
Не знаю, как там с новыми тарифами. Но новые тарифы не требуют нового клиента, я проверил на тестинге старым приложением. Так что вероятно будет правда некрасиво. Но не факт.
Поля кстати помечены как required.

### experiments
Первы експерименты видно в сурже, еулах, Плюсе - nо есть в тех вещах, что мы не берем в фолбек. Подход такой - если оказывается, что в логике для фолбека всплывут эксперименты - или выкидываем эту проверку или перетаскиваем на 3и.

### discounts
Нужны только для брендингов, мы брендинги в фолбек не берем.

### feedback_choices
Нужны для supported_feedback_choices, мы их в фолбек не берем.
