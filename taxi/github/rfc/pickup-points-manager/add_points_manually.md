## Задача

* Уметь добавлять POI (points of interest, популярные точки) по координатам из админки (не с помощью загрузки geojson)
1. с уже готовым танкерным ключом с названием
2. с произвольным названием (без танкерного ключа, без локализации, название просто текстом)

* Уметь выгружать для нужд ML информацию о POI внутри некоторого bbox, содержащую в том числе и это текстовое название


## Как работает сейчас

* Админка: https://tariff-editor.taxi.tst.yandex-team.ru/geo-areas2/pickup-points

* POST /v1/admin/points/ - добавление пикап-точки происходит только с помощью загрузки geojson, который имеет сложную структуру, парсятся из него только coordinates, icon_caption, title_tanker_key для точек

* GET /v1/admin/points/ (админка) отдает id точки (порядковый), coordinates, icon_caption пикап-точек

* DELETE /v1/admin/points/ - удаляет пикап-точку по id

* GET /v1/points/ (ручка для umlaas-geo) отдает id, coordinates, icon_caption, title_tanker_key, title (локализованный текст, который виден пользователю) пикап-точек внутри bbox

* Отредактировать точку сейчас нельзя, можно только удалить и создать заново


## Что нужно сделать

* Пикап-точки и POI различать по point_tags=['pickup'], point_tags=['poi'] (добавить в ручки дополнительным аргументом с default=['pickup']). Будет легко добавлять новые типы точек. Одна точка может иметь несколько тегов, например, быть точкой отправления и точкой прибытия.

* Добавить в базу:
1. опциональное текстовое поле title_fallback, где будет указан произвольный текст, который будет виден пользователю при отсутствии title
2. опциональное текстовое поле uri - uri точки (для ml)
3. опциональное целочисленное поле score - число, по которому будут ранжироваться точки при выдаче в GET /v1/points/

* Новая ручка добавления точек из админки POST /v1/admin/points/manual - добавление точки не через geojson, а обычным жсоном

* Добавить новые поля (title_fallback, uri, score) в ручку GET /v1/admin/points/ (также можно добавить title_tanker_key, чтобы из админки смотреть на ключи для точек)

* Добавить новые поля (title_fallback, uri, score) в ручку GET /v1/points/

* Новая ручка редактирования точек из админки POST /v1/admin/points/edit, которая будет апдейтить title_tanker_key, title_fallback, icon_caption, uri, score. Не будет возможности редактировать координаты. Если хочется отредактировать координаты, нужно создать новую точку, а старую удалить.

* Новая ручка поиска точки по uri GET /v1/admin/points/find (для ml).


## Как будет работать

* Новые точки (pickup и poi) в админке можно будет добавить двумя споcобами: несколько точек с одним набором тегов с помощью geojson и по одной с помощью /v1/admin/points/manual

* Можно будет отредактировать точку из админки (не перезаливать geojson с той же координатой, а именно редактировать по id), в том числе проставить точек новые теги

* У точки будет поле с текстом, на которое можно будет фоллбечиться при отсутствии локализованного через Танкер названия
