#### Требования и пожелания продукта к сервису

Возможности, которые продукт хочет от сервиса:
   - Логирование всех действий пользователя в сервисе
   - Аудит, драфты (***TODO**: чем аудит отличается
   от логирования?*)
   - Частичная блокировка (водителя можно не только полностью
   отключить от сервиса такси, но и от его отдельных частей,
   например, от брендинга)
   - Массовая разблокировка водителей
   - Возможность блокировать по различным идентификаторам (*номер
   паспорта, номер ВУ, номер ТС, номер телефона, VIN,
   unique_driver_id* и т.д.)
   - Единый процесс разблокировки через chatterbox (это
   значит, что все заявки на разблокировку идут через
   саппорт. Сейчас часть идёт через админку асессоров)
   - Добавить метку "процесса" в рамках которого была
    сделана блокировка (ДКК, ДКВУ, Маяк, антифрод),
     чтобы в дальнейшем можно было аккуратнее
     сортировать запросы на разблокировку.
   - Функционал "Заблокировать на N часов" (сейчас есть
   только функционал "Заблокировать до определённой даты)
   - Поддержка license_pd_id.
   - Гибкие фильтры вида `phone_id == 
   «584087e7250dd4071d74bbe4» AND NOT 
   park_id == «мой любимый парк»`
    
При блокировке сервис позволяет указать:
   - Срок блокировки
   - Причину блокировки для показа водителю
   - Номер тикета, в рамках которого блокируется сущность
   - Фотографии, на которых выявлены нарушения
   - Тип блокировки (ДКК, ДКВУ, Маяк, антифрод)
