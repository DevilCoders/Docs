### Архитектурное описание сервиса - fleet-payouts-web

### Название сервиса
fleet-payouts-web

### Какую продуктовую проблему решает сервис?
Сервис предоставит АПИ для фронта.
Предоставить максимально прозрачный механизм вывода средств, показывать отчетность.

### Почему нельзя решить эту задачу без разработки нового сервиса существующими решениями?
Существует несколько компонентов, которые будут собраны в один сервис + есть маппинг и локализация данных для фронта.

### Как именно сервис будет решать поставленные перед ним задачи?
Показывать детализацию 
- в разрезе платежек
- в разрезе транзакций

Выгрузка данных в файл

Выплата по кнопке

Информация об остатках на договорах
Информация по лимитам парка

### Разрабатывается один сервис или система?
Один сервис

### С кем взаимодействует сервис?
billing-bank-orders
fleet-payouts
fleet-internal-api
park-replica
billing-replication
drivers-profiles
fleet-parks

### Какие базы использует?
Не использует

### Какие периодические процессы?
Нет

### Планируется обработка персональных данных
Нет

### Какая нагрузка ожидается?
10rps

### Какие фолбеки предусмотрены на сам этот сервис?
Частичная деградация отдельный компонентов

### Какие фолбеки предусмотрены внутри этого сервиса на взаимодействие с другими сервисами?
Конфиги с настройками (таймауты, попытки, лимиты, чанки) для походов в сервисы за данными.

### Какие возможности масштабируемости закладываются?
Горизонтальное масштабирование, увеличение количества машинок

### Какие точки отказа есть в сервисе?
Недоступность отдельных сервисов из которых будем получать данные

### Укажите ключевые продуктовые метрики сервиса, за которыми планируете следить
Цели в Яндекс.Метрика

### Укажите технические метрики
Тайминги, rps, ресурсы

### Какая функциональность ожидается в сервисе в будущем?
Шедулер, для управления расписанием выплат
Поиск водителей не только в разрезе текущего парка в детализации

### Какое изменение нагрузки планируется?
Пропорционально количеству парков и пользователей

### Активно ли будет изменяться сервис?
Только на этапе разработки
