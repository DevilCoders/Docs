## Что это
Репозиторий для описаний новых архитектур, сервисов и изменений в api.

### Что такое RFC
RFC - request for comments - просьба посмотреть и откомментировать, проревьюить. В данном случае архитектуру.

### Чем RFC не является

RFC не является документацией, это лишь пропозал на изменение. Использовать репозиторий как хранилище документации не стоит. Но можно использовать его для понимания того, почему первые версии архитектур были именно такими, как они менялись.

## Зачем
1. На GitHub значительно удобнее ревьюить архитектуру, чем на wiki или в трекере.
2. Желающие смогут следить за появлением новых сервисов и важных изменений в Такси (кнопка Watch сверху), это поможет избежать дублирования усилий.
3. Ревью может проходить качественнее, если оно более публично.

## Делает ли так кто-то еще
Да, например, [Uber](https://blog.pragmaticengineer.com/scaling-engineering-teams-via-writing-things-down-rfcs/) и [Google](https://www.industrialempathy.com/posts/design-docs-at-google/).
У нас идею подали и началось все с [репозитория первой команды](https://github.yandex-team.ru/jolfzverb/docs).

## Как с этим работать
### Я предлагаю изменение
1. Делаете пул-реквест с изменениями архитектуры с отдельной папкой про ваш сервис
2. В пул-реквесте в дире прикладываете все нужные файлы - описание API в yaml, markdown описание архитектуры, схемы в виде картинок и тп.
3. В пул-реквест призываете ревьюеров - как и раньше, кого считаете нужным (тимлид и тп). Если есть изменения API - как минимум @twirl.
4. После апрува нужными ревьюерами мержите пул-реквест.

### Я ревьюю изменение
1. Смотреть markdown файлы в диффе удобно отрендеренными: над файлом диффа есть кнопка `Display the rich diff`
2. Если вы ревьюите что-то из интереса, то там, где вас не добавляли в ревьюеры, пожалуйста, будьте аккуратны с комментариями: если все будут ревьюить всех, то есть риск сильного затягивания ревью.

### Картинки
В репозитории схемы описываются в формате `.puml`. Для просмотра и редактирования таких схем используется плагин PlantUML

Можно класть прямо в репозиторий отрендеренные файлы, рендерить их можно утилитой `plantuml`, она есть в репозиториях пакетных менеджеров.
