В проекте есть TeamCity (https://teamcity.taxi.yandex-team.ru/viewType.html?buildTypeId=YandexTaxiProjects_Frontend_Atlas_CustomTesting), у которого есть 3 сборки

# Сборки
## AutoRelease
1. Мержит master в стейбл
2. Собирает stable и пушит в docker-registry (stable)
3. Делает тикет в Taxirel
4. Добавляет коммиты в этот тикет (смотрит дифф по времени коммитов) и добавляет в связанные нужных людей
5. Уведомляет в TAXIREL-тикет о сборке

## Release
1. Мержит master в стейбл
2. Собирает stable и пушит в docker-registry (stable)

## Testing
1. Мержит в мастер пр с меткой
2. Собирает и пушит в docker-registry с регистри (testing)

# Версии
По умолчанию при сборке ставится версия как номер билда + префикс с нужными major и minor. 

Если хочется обновить минорную версию или мажорную в окружениях **Release** и **AutoRelease**, надо в **Parameters** указать переменной VERSION нужную версию, чтобы с этой версии скрипт тулзов начать апать патч

Например, была версия 1.5.99, мы обновили минор. При деплое в Release указываем перед RUN переменной _VERSION_ значение **1.6.0**. Во всех следующих релизах будет браться уже значение из памяти тимсити
