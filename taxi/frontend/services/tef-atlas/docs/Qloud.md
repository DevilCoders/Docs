Нам можно воспринимать qloud как крутилку докер-контейнеров а облаке Я.

Подробное описание проекта можно найти [здесь](https://wiki.yandex-team.ru/qloud/).

Атлас живёт [тут](https://platform.yandex-team.ru/projects/taxi-tools/atlas)

![](https://jing.yandex-team.ru/files/eugenest/taxi-toolsatlas_2018-11-08_16-00-44-keh2i.png)

Тестовый стенд выглядит так (на боевом всё так же, только ресурсов чуть больше и образы другие): 

![](https://jing.yandex-team.ru/files/eugenest/taxi-toolsatlastesting_2018-11-08_16-19-57-s9do1.png)

Сейчас на стендах по 2 контейнера, для нашей статики, и для форвардинга в бэкенды, т.е. там просто крутится nginx.
Ноду пока совсем открутил, т.е. сейчас папка ./server/ нужна только для работы на дев-машинке. При необходимости её можно добавить отдельным контейнером.

Модалка обновления контейнера выглядит так:

![](https://jing.yandex-team.ru/files/eugenest/taxi-toolsatlastesting_2018-11-08_16-23-41-64vsp.png)

Обычно, после выкатки докер-образа в регистри с тэгом **latest**, достаточно просто нажать **Update**, и начнётся обновление. Обычно оно занимает пару минут. В этот момент статусы контейнеров будут меняться с ISS CRIT до ISS OK.

Возможно, позже прикручу версионирование по номерам билдов.
