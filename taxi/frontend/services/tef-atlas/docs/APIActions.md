В коде, в api/* файлах можно встретить много обёрток над экшнами, которые не добавляют никакой новой функциональности, а являются просто проксями к экшнам.

Пока здесь есть только 1 плюс — прозрачные интерфейсы, автокомплит, навигация в ide. 

Для того, чтобы не дублировать код, есть вариант добавлять экшны как статические методы класса API через `utils/mixins/actions`:

```javascript
class EditPolygonsAndStatsApi {
    static someMethod = pair => dispatch => {
       
    };
}

export default ActionsMixin(actions)(EditPolygonsAndStatsApi);
```

Главное требование в этом случае — явное описание параметров / шэйпа actions.payload в редьюсере.
