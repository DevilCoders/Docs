# Описание тулзы:
Тулза разбивает область на полигоны, границы которых проходят по дорогам опрееленной категории (или выше)

# Пайплайн работы:
Запуск алгоритма:

`taxi-regions-poly-cover [путь до конфига]`

На выходе будет 2 файла: логи (можно забить) и описание полигонов.

Полигоны описываются в формате easyview. Соответственно, их можно нарисовать и посмотреть, что получилось, а можно распарсить питоном и заняться постобработкой.

# Описание конфига:
1. Путь до директории графа (по умолчанию -- тестовый граф из аркадии)
2. Категория дорог, которые будут испольщоваться (по умолчанию -- 6 - `EC_LOCAL_ROADS`)
3. Путь до выходного файла с полигонами (в формате easyview)
4. Путь до выходного файла с логами
5. Читать ли ребра из файла или нет (0/1)

Если запускать на одном и том же наборе ребер несколько раз, то имеет смысл один раз записать ребра 
в файл, а потом читать их оттуда (сильно ускорит работу алгоритма, особенно на большом графе)

6. Путь до файла с ребрами (туда будут либо писаться, либо оттуда будут читаться ребра)
7. Минимальная и максимальная долгота
8. Минимальная и максимальная широта
9. Путь до файла со всеми ребрами, по которым будет работать алгоритм

Ребра рисуются как прямые от начальной точки до конечной (если вдруг кривая дорога превращается в прямую, это норм). Так можно проверить, что обсчитались именно те ребра, которые нужны.

Описывают границы бокса, по которому обрезуются ребра, если ребра не читаются из файла
