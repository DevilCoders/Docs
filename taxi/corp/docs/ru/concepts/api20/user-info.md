# Детальная информация о пользователе

Запрос позволяет получить подробную информацию о пользователе клиента.

## Синтаксис запроса

```
GET https://b2b-api.go.yandex.ru/integration/2.0/users/{идентификатор пользователя}
```

{% include [user-create-headers-title](../../_includes/concepts/api20/user-create/id-user-create/headers-title.md) %}


#### Authorization
OAuth-токен. Процесс получения токена описан в разделе [Начало работы](../quickstart.md).

## Описание полей ответа

В ответе могут содержаться следующие поля:

Поле | Описание | Формат
----- | ----- | -----
`fullname` | Полное имя пользователя. | Строка
`nickname` | Краткое имя пользователя. | Строка
`is_active` | Признак активности пользователя. Неактивный пользователь не имеет возможности самостоятельного заказа и на его имя нельзя заказать поездку. | Логическое
`is_deleted` | Признак, были ли пользователь удален. | Логическое
`phone` | Телефонный номер пользователя. | Строка
`email` | Адрес электронной почты пользователя. | Строка
`cost_centers_id` | id набора [настроек центров затрат](../cost-center-settings-list.md) (при наличии у клиента новых центров затрат). Необязательное поле. | Строка
`cost_center` | Название [кост-центра клиента](../cost-center-create-old.md). | Строка
`limits` | Ограничения на сумму, которую пользователь может потратить на сервисы за календарный месяц. | Массив элементов, содержит отдельный элемент для каждого сервиса.


Структура элемента массива `limits`:

Поле | Описание | Формат
----- | ----- | -----
`limit_id` | Ограничение на сумму, которую пользователь может потратить на определенный сервис за календарный месяц. | Строка
`service` | Название сервиса. Возможные значения: `taxi`, `eats2`, `drive`. | Строка


## Пример запроса

```
GET https://b2b-api.go.yandex.ru/integration/2.0/users/f65...c57d
...
Authorization: Bearer <OAuth-токен>
```

## Пример ответа

Пример ответа на данный запрос выглядит следующим образом:

```no-highlight
{
  "fullname": "Иванов Илья",
  "nickname": "ИИлья",
  "is_active": true,
  "is_deleted": false,
  "phone": "+75551234567",
  "email": "",
  "cost_centers_id": "123...fef",
  "cost_center": "some cost center"",
  "limits":[
     { 
       "limit_id": "abcdef_taxi",
       "service": "taxi" 
     },
     {
       "limit_id":"abcdef_eats",
       "service":"eats2"
     },
     {
       "limit_id":"abcdef_drive",
       "service":"drive"
     }
  ]   
}
```

## Возможные коды ответа

Ответ на данный запрос может содержать следующие стандартные HTTP-коды:

- `200` — запрос выполнен успешно.
- `401` — был передан неверный [OAuth-токен](../quickstart.md).
- `403` — у клиента не хватает прав на выполнение данного запроса.
- `404` — запрашиваемая запись не найдена.
