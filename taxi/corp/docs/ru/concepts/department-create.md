# Создание нового подразделения

Запрос позволяет создать запись о новом подразделении.

## Синтаксис запроса

```
POST https://business.taxi.yandex.ru/api/1.0/client/{идентификатор клиента}/department/
```

{% include [order-create-headers-p](../_includes/concepts/order-create/id-order-create/headers-p.md) %}


#### Authorization
OAuth-токен. Процесс получения токена описан в разделе [Начало работы](quickstart.md).

Данные о новом подразделении передаются в теле запроса, в формате JSON:

Поле | Описание | Формат
----- | ----- | -----
`parent_id` | id родительского подразделения (`null` — для корневого подразделения). | Строка
`name` | Название подразделения. | Строка


## Описание полей ответа

В ответе могут содержаться следующие поля:

Поле | Описание | Формат
----- | ----- | -----
`_id` | Идентификационный номер подразделения. | Строка


## Пример запроса

```
POST https://business.taxi.yandex.ru/api/1.0/client/233e725b0511459da7b38cb24f2d8fd7/department/
...
Authorization: <OAuth-токен>


{
       "parent_id": "233e725b0511459da7b38cb24f2d8fd7",
       "name": "Some department"
}
```

## Пример ответа

Пример ответа на данный запрос выглядит следующим образом:

```no-highlight
{
        "_id": "2f07049ecedb43b0b0be506071f34c53"
}
```

## Возможные коды ответа

Ответ на данный запрос может содержать следующие стандартные HTTP-коды:

- `200` — запрос выполнен успешно.
- `400` — в запросе был передан неизвестный параметр или параметр с недопустимым значением.
- `401` — был передан неверный [OAuth-токен](quickstart.md).
- `403` — у клиента не хватает прав на выполнение данного запроса.
- `406` — запись с указанными параметрами уже существует.
