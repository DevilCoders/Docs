# Изменение района

Запрос позволяет изменить существующий район поездок клиента.

## Синтаксис запроса

```
PUT https://business.taxi.yandex.ru/api/1.0/client/{идентификатор клиента}/geo_restrictions/{идентификатор района}
```

#### Authorization
OAuth-токен. Процесс получения токена описан в разделе [Начало работы](quickstart.md).

Новые данные о районе поездок передаются в теле запроса в формате JSON:

Поле | Описание | Формат
----- | ----- | -----
`name` | Имя. | Строка
`geo_type` | Тип гео ограничения. Сейчас поддерживается только «circle». | Строка
`geo` | Описание гео ограничения. Содержит следующие поля:<br/>- `center` — координаты центра;<br/>- `radius` — расстояние от центра (в метрах). | Объект


## Описание полей ответа

В ответе могут содержаться следующие поля:

Поле | Описание | Формат
----- | ----- | -----
`_id` | Идентификационный номер района поездок. | Строка


## Пример запроса

```
PUT https://business.taxi.yandex.ru/api/1.0/client/7498c21b09a04172bd41472f6a7c0af3/geo_restrictions/b45efcaa04014fff997c2683ef91f0de
  {
    "geo": {
      "center": [
        37.642639,
        55.734894
      ],
      "radius": 200
    },
    "geo_type": "circle",
    "name": "Офис 2"
  }
```

## Пример ответа

Пример ответа на данный запрос выглядит следующим образом:

```no-highlight
  {
      "_id": "b45efcaa04014fff997c2683ef91f0de"
  }
```

## Возможные коды ответа

Ответ на данный запрос может содержать следующие стандартные HTTP-коды:

- `200` — запрос выполнен успешно.
- `400` — в запросе был передан неизвестный параметр или параметр с недопустимым значением.
- `401` — был передан неверный [OAuth-токен](quickstart.md).
- `403` — у клиента не хватает прав на выполнение данного запроса.
- `404` — указанная запись не найдена.
