# Обработка заказа

Запрос позволяет передать на обработку заказ, созданный с помощью запроса [Создание черновика нового заказа](order-create.md).

## Синтаксис запроса

```
POST /client/{идентификатор клиента}/order/{идентификатор заказа}/processing
```

{% include [order-create-headers-p](../_includes/concepts/order-create/id-order-create/headers-p.md) %}


#### Authorization
OAuth-токен. Процесс получения токена описан в разделе [Начало работы](quickstart.md).

## Описание полей ответа

В ответе могут содержаться следующие поля:

Поле | Описание | Формат
----- | ----- | -----
`_id` | Идентификационный номер заказа. | Строка
`status` | Блок статуса заказа. | Объект
`simple` | [Упрощенный статус](order-list.md#simple-desc) заказа. | Строка
`full` | [Подробный статус](order-list.md#full-desc) заказа. | Строка
`description` | [Описание состояния](order-list.md#description-desc) заказа. | Строка


## Пример запроса

```
POST https://business.taxi.yandex.ru/api/1.0/client/a2...d09/order/3c...b05e/processing
...
Authorization: <OAuth-токен>
```

## Пример ответа

Пример ответа на данный запрос выглядит следующим образом:

```no-highlight
{
    "_id": "3caa3587675b49deb62e3286b753b05e",
    "status": {
        "simple": "active",
        "full": "search",
        "description": "Такси приедет через N мин."
    }
}
```

## Возможные коды ответа

Ответ на данный запрос может содержать следующие стандартные HTTP-коды:

- `200` — запрос выполнен успешно.
- `401` — был передан неверный [OAuth-токен](quickstart.md).
- `403` — у клиента не хватает прав на выполнение данного запроса.
- `404` — указанная запись не найдена.
- `406` — цена поездки изменилась с момента получения [оффера](route-stats.md). Повторно получите оффер, передайте номер оффера в запросе [Создание черновика нового заказа](order-create.md) и передайте заказ на обработку.
