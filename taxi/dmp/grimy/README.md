## Как разрабатывать локально:
```
docker-compose build
docker-compose up
```
После успешного старта докера, нужно открыть адрес `http://localhost`


Поднимет докер с postgresql + веб приложение в режиме livereload.
Синтетические данные хранятся в директории postgresql. Даты в insert задаются
как некоторое смещение от sysdate. Это позволяет при старте веб приложения
сразу показывать страницу с данным (подефолту отображается текущий день).
В случае харкода дат пришлось бы искать данные для тестирования в ui.

Если изменился файл Dockerfile, то нужно пресоздать контейнер.
Изменение кода обычно не требует перезапуск контейнера.

После изменений в зависимостях (deb/py пакетов) или Dockerfile, нужно пересоздать
контейнер с веб приложением (правки кода не требуют пересоздание контейнера):
```shell
docker-compose rm web
docker-compose build --no-cache web
```

После внесения правок в sql файлы в директории пересоздать контейнер с бд:
```shell
docker-compose rm db
docker-compose build --no-cache db
```

Для запуска контейнеров должны быть доступны порты 80 и 5432

## Выкатка
1. скопипасить изменения в /opt/grimy
2. найти текущий процесс `ps aux | grep -i grimy` и убить его
3. запустить `sudo /opt/grimy/run_grimy.sh &`
