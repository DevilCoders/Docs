# Взаимодействие с бекэндом

В целом - аналогично фильтрам:

- бек отдает в ручке menu/goods возможные сортировки товаров
- клиент их отображает. Если какая-то сортировка была применена - можно ее
  отобразить в UI пред-выбранной.
- юзер выбирает сортировку - фронт передает на бек выбранный slug в запросе
- бек возвращает отсортированный ответ - по идее, дальше работает как и сейчас

# Доработки на фронте
- перейти на POST-ручку menu/goods, где еще нет (iOs, Android). Заодно можно перести с catalog
на menu, чтобы убрать лишний редирект
- принимать сортировки и отображать их (возможно, под фронтовым экспериментом)
- передавать их на бекэнд

Остальное должно уже работать.

Сохранение сортировки делаем аналогично фильтрам - при проваливании в подкатегорию - 
сохраняем, иначе - сбрасываются. 


# Доработки на бекэнде
- [0.5д] добавить эксперимент отдачи сортировок (применение можно без эксперимента)
  В нем задавать, какие сортировки включены и их локализованные названия.

- [2д] отдавать их в ответе
  Отдавать все сортировки во всех категориях.

- [3-5д] принимать от фронта сортировку и сортировать - см. ниже как работает

# Как работают сортировки
Предлагаю не использовать название "Доверюсь вам" - звучит довольно странно
  - по популярности: работает как сейчас
  - По возрастанию цены - сортируем по акционной цене. Если цена одинаковая -
    сортируем как проще (например по алфавиту)
  - По убыванию цены - аналогично
