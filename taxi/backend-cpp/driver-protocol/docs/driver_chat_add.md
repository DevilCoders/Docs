## /driver/chat/add

Ручка для добавления водительского сообщения в чат. Может быть как GET, так и POST запросом

## Параметры

- `db` - идентификатор парка
- `session` - водительская сессия для проверки авторизации
- `channel` - канал, в который добавляется сообщение
- `msg` - текст добавляемого сообщения
- `tag` - тэг сообщения (например, SOS)
- `language_hint` - (опционально) UTL-encoded json объект с перечнем кодов языков. Использется как подсказка
при определении языка сообщения (языки клавиатуры, приложения, системы):
```json
"language_hint": {
  "keyboard_languages": ["de"],
  "app_language": "en",
  "system_languages": ["de", "en"],
}
``` 

Параметры db и session всегда передаются в query string

При GET-запросе в query string передаются channel и msg

При POST-запросе все параметры кроме db и session передаются в теле

## Пример запроса

`curl X POST -d 'msg=Это тестовое сообщение&channel=PRIVATE&tag=SOS'
"driver-protocol.taxi.dev.yandex.net/driver/chat/add?db=c52e88b724674ef7917ee0f8fa4627de&session=abcd18cfb6ad465ab6f47a4d73f7d42c" -v`

## Коды ответа

200 - сообщение было успешно добавлено

400 - невалидный запрос, чат отключен или заблокирован
      или невалидный ответ от сервиса для внешнего чата

401 - авторизация не удалась, невалидные парк или сессия

404 - не найден водитель, или парк при не валидном канале,
      или получен 404 от сервиса внешнего чата

500 - не найдена страна при валидном парке,
      непредвиденная ошибка при хождении в чат клиент-водитель

В случае успешного добавления в ответе ожидается пустая строка
