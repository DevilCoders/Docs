## /service/chat/messages

Ручка для просмотра сообщений в канале чата. POST запрос.

## Параметры

### Параметры в query string
- `db` - идентификатор парка

### Параметры в теле запроса
- `channel` - канал, из которого извлекаются сообщения
- `md5` - хэш, соответствующий каналу. Если совпадает с тем, что в базе, то новых сообщений нет.

## Пример запроса

`curl X POST -d '{"md5": "907084d42d6a44e4cbf7e70ee31568b5", "channel":
"PRIVATE:7ad36bc7560449998acbe2c57a75c293:de93261d567541d8b7baec7738fc7bd4"}'
"driver-protocol.taxi.tst.yandex.net/service/chat/messages?db=7ad36bc7560449998acbe2c57a75c293" -v`

## Коды ответа

200 - успешно получены сообщения или ответ с информацией о выключенном чате в парке

400 - невалидный запрос или отсутствуют канал и md5 одновременно

500 - не найдена страна при валидном парке и невалидном канале
