### /service/chat/dispatcher/remove

Удалить сообщение по идентификатору из диспетчерской парка

### Пример запроса

`curl GET "driver-protocol.taxi.tst.yandex.net/service/chat/dispatcher/remove?db=7ad36bc7560449998acbe2c57a75c293&id=aa61de52db4c4aa0be981b865ab00ca0" -v`

### Параметры

- `db` - идентификатор парка, обязательный
- `id` - идентификатор сообщения в соответствующей хэш-таблице в redis

Все сообщения диспетчерской для данного парка можно найти в redis, например,
по ключу Chat:Dispather:7ad36bc7560449998acbe2c57a75c293

### Пример ответа

400 - невалидный запрос (не удалось извлечь параметры, кто-то из них пустой и т.д.)

    {
      "error": {
        "text": "Missing or empty parameter db"
      }
    }

500 - не удалось либо удалить сообщение (оно не было найдено),
      либо обновить md5 для диспетчерской парка

200 - Сообщение было удалено
