## Задача
Нужно добавить красную точку для таба "Магазины", для новичков и, возможно, по другим признакам (список юзеров).
Тикет: https://st.yandex-team.ru/RETAILSPECS-181

## Полноценный вариант vs MVP
Есть полноценная задача: https://st.yandex-team.ru/EDAPRODUCT-3192, но она будет сделана к концу мая 2022.

Мы же хотим сделать быстрее (середина мая), потому что кажется что ожидаемый профит от новых клиентов будет оправдывать костыльность разработки.

## Предлагаемое решение
1. В ответ Layout constructor, где приходит таббар, в объект `Tab`, добавляем новое опциональное поле (рядом с `action_payload`):
```json
{
    "id": "shops",
    // ...,
    "notifications": {
        "red_notification_dot": {
            "id": "some_id_1",  // Id нотификации
        }
    }
}
```
2. Это поле бек заполняет из эксперимента - где можно будет таргетировать по списку юзеров.
Новичков будем определять по стандартной логике (надо найти в LC где признак новичка уже есть, либо ходить в eats-order-stats / куда-то еще), и прокидывать в кварг эксперимента.
3. Клиент (фронт), если получил поле `"red_notification_dot": ...`, отображает красную точку, если факт отображения еще не был запомнен на этом девайсе (см. далее)
4. При клике на таб, клиент запоминает `id` таба и id нотификации (из поля `red_notification_dot`) в локальном хранилище, и больше не показывает эту точку никогда для таба с таким `id` И таким же id нотификации. Тем самым, с бека можно вызвать повторный показ красной точки, изменив значение id нотификации.
5. Клиенты должны корректно обрабатывать отсутствие поля `notifications`, потому что его очень вероятно что уберем с бека после реализации полноценного решения.

Наверное где-то тут надо добавить клиентские события "показали красную точку".

### Альтернативный вариант
- на клиентах запоминать дату последнего показа точки, а с бека передавать дельту в секундах, после которой надо точку показывать. 
Так можно будет показать точку еще раз через Х дней, но пока не ясно, нужно ли это, и это немного усложнит разработку. Поэтому не делаем.
