## Варианты реализации

#### 1. Через кастомные категории
- блоггеры присылают нам рецепты 
- контент вручную заводит их как кастомные категории
- фронт ходит в обычную ручку menu/goods и набирает там нужные данные (картинка + название)
- [1d] делаем доработку в eats-products по скрытию этих категорий, если пришли без category_id
- [1d] для целей кеширования (чтобы не сложить eats-products при просмотре рецепта) например, делаем в api-proxy отдельную ручку С КЕШИРОВАНИЕМ, и ведем траффик через нее. И еще ставим на нее же жесткий рейт-лимитер
- [1d] делаем выдачу отсутствующих товаров в этой кастомной категории

### 2. Через отдельный сервис `retail-products-lists`
#### Флоу
- [фронты] добавляем в корзину флоу по добавлению всех товаров в список
- все товары из корзины новый сервис (по-товарно например) добавляет в новый список
- по ссылке на список фронт ходит в новый сервис, а тот отдает картинку и название сохраненных товаров прямо из своей базы, без походов в eats-products - чтобы снизить нагрузку и защититься от отсутствующего ассортимента
- при переходе в Еду для заказа товаров, фронт ходит в `retail-products-lists`, тот ходит в `eats-products` в get-items, получает все данные для отображения и отдает. Если товара в номенклатуре нет, то отдаем картинку и название из своей БД, и пишем что раскупили

#### Доработки
- [0.5d] новый сервис `retail-products-lists` + arch review (видимо, лайтовое из-за хакатаона)
- [2.5d] создать список товаров (сразу передавая набор товаров, входящих в этот список) (НИЖЕ ПРИОРИТЕТОМ, ПОТОМУ ЧТО МОЖНО СОЗДАТЬ В БАЗЕ РУКАМИ)
    - на вход: 
        - brand_slug
        - название списка
        - товары:
            - core_id товаров (который в корзине называется item_id)
            (остальное не передаем чтобы защититься от "фрода")
    - на выход
        - id списка товаров (является ключом в url) условно uuid v4 hex
    - запоминаем картинку и название в БД сервиса
- [1d] отдавать список товаров для блога (данные из БД)
    - на вход:
        - id списка товаров
    - на выход:
        - brand_slug
        - массив товаров:
            - картинка из БД сервиса в подобной структуре: https://a.yandex-team.ru/arc/trunk/arcadia/taxi/uservices/services/eats-cart/docs/yaml/definitions.yaml?rev=r8782838#L181
            - название
    - рейт-лимитер условно на 5к рпс
- [2-3d] отдавать список товаров для каталога (полные данные)
    - на вход:
        - id списка товаров
        - place_slug
    - на выход:
        - название списка
        - массив товаров как в menu/goods
    - ?? если товар не в ассортименте, отдать картинку и название из БД и написать "Разобрали" 
- [1d] настройка среды, алертов, рейт-лимитеров, api-proxy
