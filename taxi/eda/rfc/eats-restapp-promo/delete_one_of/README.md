# Убираем из апи создания акций и конфига oneOf

Тикет: [https://st.yandex-team.ru/EDAPARTNERS-1118](https://st.yandex-team.ru/EDAPARTNERS-1118)

В проекте из-за наличия oneOf большое число вариантов, что сильно затрудняет сборку проекта. Все типы акций похожи друг на друга, тип акции определял только обязательность полей и их ограничения. Решение: сделать тип акции, в котором все эи поля опциональны, а проверку обязательности полей и их ограничений вынести в конфиг. Новые апи и конфииг совместимы со старыми версиями. В итоговый json конфига добавятся несколько полей с ограничениями.

Новые api.yaml и eats_restapp_promo_settings.yaml описаны в директории.
