### Описание
В рамках задачи по продаже алкоголя на самовывоз в алкомаркетах требуется показывать измененный боттомшит магазинов.

### Ожидаемые изменения

Для реализации задачи потребуется изменить формат ответа ручки `/eats-catalog/v1/slug/{slug}`.

#### Заголовок, время сборки заказа, время ожидания заказа
  Данные для отображения этого блока передаются в foundPlace.place.messages

  В messages добавить еще один тип сообщения
  ```
  pickup:
      $ref: "#/components/schemas/Message"
  ```

  Объект Message расширить дополнительным полем
  ```
  message:
    type: string
    description: Строка, содержащая текст, который должен расположиться под текстом из title и над текстом из footer
    examle: Соберем заказ от 30 минут
  ```
  
  В случае, если тип сообщения pickup, объект Message должен быть заполнен следующим образом:
   1. Свойство title содержит строку, используемую в качестве заголовка боттомшита. Для типа доставки pickup у магазинов это свойство должно иметь значение "Самовывоз". Текст взять из Танкера по ключу из конфиг.
   2. Свойство message содержит строку, которую нужно нужно будет расположить под текстом из title и над текстом из footer. Эта строка должна содержать информацию о времени сборки заказа, например "Соберем заказ от 30 минут". Текст взять из Танкера по ключу из конфига. Время сборки есть в админке в поле "Время приготовления блюда". Эта информация уже должна быть проброшена в eats-catalog.
   3. Свойство footer содержит строку, которую нужно нужно будет расположить под текстом из message. Эта строка должна содержать информацию о времени хранения собранного заказа, например "Заказ будет храниться 4 часа". Текст взять из Танкера по ключу из конфига. Время хранения заказа взять из конфига, схема которого приложена ниже, поле storage_time. 
   *Дополнительно в prd описывается логика, которую делать дольше, поэтому стоит уточнить при реализации: в случае если количество часов хранения закончится в ближайшее нерабочее время магазина, то использовать текст "Заказ будет ждать вас в магазине до закрытия" (точный текст уточнить при реализации).

  Строки для title, message, footer, BottomBanner.name взять из Танкера по ключам из конфига. Конфиг не должен быть связан с продажей алкоголя в магазинах, семантика конфига должна быть связана со способами доставки (при реализации разобраться с тем, какие конфиги уже есть, если нет подходящего, то создать).

#### Дополнительная информация в футере
  В самом низу раскрытого боттомшита требуется указать дополнительную информацию. Это правила заказа в магазине (такой же текст, что и в диалоге подтверждения возраста) и текст об адресах магазинов и лицензиях. 
  
  На бэкенде брать эти два текста из Танкера по ключам из конфига (поля rules и  licenses) и склеивать их разделив через пробел. Тексты правил заказа и лицензий могут отличаться между брендами. Предлагаемую схему прилагаю ниже.

  Итоговый составленный текст будет отправляться в существующем поле footerDescription.
  На фронтах для алкомаркетов интерпретировать содержимое footerDescription как текст с html тегами для корректного отображения ссылок на страницы с лицензиями.


### Конфиг

Предлагаю название: EATS_RETAIL_ALCOHOL_SHOPS

Схема:
```
description: Конфиг содержащий побрендовую информацию алкомаркетов которую требуется отображать пользователям.
tags: [notfallback]
schema:
    type: object
    description: Ключ - числовой id бренда (завернутый в строку), значение - данные для магазинов этого бренда.
    additionalProperties:
        $ref: '#/definitions/AlcoholBrand'
    definitions:
        AlcoholBrand:
            type: object
            required:
              - rules
              - licenses
              - rules_with_storage_info_short
              - rules_with_storage_info_full
              - storage_time
            properties:
                rules:
                    type: string
                    description: Ключ танкера, по которому хранится текст, содержащий условия заказа товаров в магазине. Текст, среди прочего может содержать ссылки, завернутые в html теги.
                licenses:
                    type: string
                    description: Ключ танкера, по которому хранится текст, ссылающийся на лицензии магазина. Текст, среди прочего может содержать ссылки, завернутые в html теги. 
                warning:
                    type: string
                    description: Ключ танкера, по которому хранится текст, предупреждающий о вреде черезмерного употребления алкоголя
                    default: <todo: тут указать дефолтное значение ключа танкера для предупреждения>
                rules_with_storage_info:
                    type: object
                    additionalProperties: false
                    required:
                      - full
                    properties:
                        short:
                            description: Ключ танкера, по которому хранится сокращенный вариант текста, содержащий условия заказа товаров в магазине совместно с упоминанием времени хранения собранного заказа. Текст, среди прочего может содержать ссылки, завернутые в html теги.
                            type: string
                            default: <todo: тут указать дефолтное значение ключа танкера для сокращенного варианта текста>
                        full:
                            type: object
                            additionalProperties: false
                            description: Полные правила покупки
                            required:
                            properties:
                                title:
                                    description: Ключ танкера, по которому хранится заголовок вьюшки, содержащей полные правила покупки
                                    type: string
                                    default: <тут указать дефолтное значение ключа танкера для заголовка>
                                text:
                                    description: Ключ танкера, по которому хранится полный вариант текста, содержащий условия заказа товаров в магазине совместно с упоминанием времени хранения собранного заказа. Текст, среди прочего может содержать ссылки, завернутые в html теги.
                                    type: string
                                    default: <todo: тут указать дефолтное значение ключа танкера для полного варианта текста>
                storage_time:
                    type: integer
                    description: время хранения собранного заказа в часах
                    minimum: 1
            additionalProperties: false
```
