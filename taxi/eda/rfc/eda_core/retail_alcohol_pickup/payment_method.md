### Описание
  При покупках в алкомаркетах на экране оплаты требуется отображать время сборки заказа, плашку самовывоза и единственный способ оплаты "на кассе".
  Изменения предлагается сделать в два этапа - сначала сделать быстро, затем правильно и универсально.

### Версия 1

#### Концептуально
  На бэкенде информация о самовывозе и времени сборки заказа уже должна пробрасываться без изменений текущей логики. Для отображения единственного способа оплаты требуется в сервисе eats-cart для товаров из алкомаркетов устанавливать флажок "is_alcohol" и пробрасывать его в ручку коры go-checkout, при получении такого флажка для хотя бы одного товара в магазине, ручка go-checkout вернет единственный способ оплаты.

#### Подробно
Изменения в eats-cart: 
  в ручке `/internal/eats-cart/v1/related_data`, объект `CartRelatedDataItem` расширить полем
  ```
    is_alcohol:
        description: Признак того, что товар является алкоголем.
        type: boolean
  ```

  Логика заполнения в сервисе eats-cart: в случае если запрос пришел на магазин и у магазина включен самовывоз, то считать его алкомаркетом и проставлять всем его товарам флаг `is_alcohol`.

Изменения в go-checkout:
  В случае если eats-cart вернул хотя бы для одного товара флажок `is_alcohol` со значением `true`, то ручка go-checkout должна вернуть единственный способ оплаты.
  В ответе ручки go-checkout для магазинов с самовывозом в объекте PossiblePaymentBase в поле `title` отправлять текст "Оплата при получении" (требуется уточнить при реализации с @aadavidov)

Судя по всему, изменений на фронтах не требуется.


### Версия 2

#### Концептуально

Механизм переключения на единственный способ оплаты "Оплата при получении" должен включаться в двух ситуациях:
  1. В случае если это специальное заведение, которое поддерживает единственный способ оплаты. Например, алкомаркет, пользователь может набрать в нем только закуски, но при этом способ оплаты должен все равно быть "Оплата при получении".
  2. В случае если пользователь добавил алкоголь в корзину. Такое поведение должно работать и для алкомаркетов и для обычных магазинов (этой функциональности у обычных магазинов сейчас нет, но в будущем может появиться).

Таким образом нам нужна возможность проставлять признаки и на уровне заведения и на уровне товара.

#### Подробно

#### Проброс признака алкогольности для товара

Признак алкогольности товара есть в сервисе eats-nomenclature, его нужно пробросить до ручки go-checkout.

Последовательность передачи данных в этом случае такая:
eats-nomenclature -> eats-products -> eats-cart -> go-checkout

Изменения в eats-products:
	в ответе ручки v2/menu/get-items для каждого товара отдавать флаг `is_alcohol`, который заполняется данными из ответа ручки сервиса eats-nomenclature `v1/products/info` (там уже есть соответствующий флаг).

Изменения в eats-cart:
	получать из ответа ручки `v2/menu/get-items` признак алкогольности товара и пробрасывать его в ответе ручки `/internal/eats-cart/v1/related_data` в поле `is_alcohol`, которое было указано в версии 1 (в первой версии проставляли этот флажок для всех магазинов с самовывозом, теперь будем пробрасывать реальные данные).

Изменения в go-checkout:
  никаких дополнительных изменений относительно версии 1 нет (изменений из версии 1 достаточно).

#### Проброс признака оплаты на кассе для заведения

На данный момент, единственный вид заведений, в которых должен быть доступен только вид оплаты "Оплата при получении" - алкомаркеты.
Магазин считается алкомаркетом когда его бренд указан в конфиге алкомаркетов.

Последовательность передачи данных в этом случае такая:
eats-catalog -> eats-cart -> go-checkout

Изменения в eats-catalog:
  в ответ ручки `/internal/v1/place` в поле `constraints` добавить флаг `is_cash_only` который будет выставляться в `true` в случае, когда в заведении доступен единственный способ оплаты "Оплата при получении". Для заполнения этого поля нужно сходить в конфиг `EATS_RETAIL_ALCOHOL_SHOPS` и, если бренд магазина есть в этом конфиге, то выставить флаг в `true`

Изменения в eats-cart:
  в ручке `/internal/eats-cart/v1/related_data`, объект `CartRelatedData` расширить полем
  ```
    place:
      description: Дополнительная информация о магазине
      $ref: '#/components/schemas/PlaceRelatedData'
    components:
        schemas:
            PlaceRelatedData:
              type: object
              additionalProperties: false
              properties:
                  is_cash_only:
                      description: Признак того, что в заведении доступен единственный способ оплаты "Оплата при получении".
                      type: boolean
  ```
  Выставлять `place.is_cash_only` в `true` тогда, когда из каталога получили `constraints.is_cash_only` со значением `true`.

Изменения в go-checkout:
	получать из ответа ручки `/internal/eats-cart/v1/related_data` параметр `place.is_cash_only` и, если этот флаг выставлен в `true`, то ручка go-checkout вернет единственный способ оплаты.
