# Адрес в чекауте
## Проблемы:
Фронт начинает работать с новыми адресами из сервиса persaggest. Не все адреса из этого сервиса имеют ID, и многие имеют дополнительные поля.
При этом ручка в коре по созданию заказа не умеет работать с такими адресами.

## Решение:
1. Меняем схему АПИ позволяя передавать в ручку адрес без ID
2. При создании заказа если ID адреса нет, не создаем не создаем новый

## Изменение в API:
1. сервис: eda_core, ручка: POST api/v1/orders, добавим в объект запроса возможность указать аддрес без ID и отдельно необязательный ID для избранных адресов
