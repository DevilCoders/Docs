# Изменение go-checkout
## Проблемы:
1. Нет возможности создавать заказы не используюя ID адреса

## Решение:
Модифицируем ручку /go-checkout сервиса eda_core. Добавляем в него новый провайдер, делаем возвращаемый объект адреса не обязательным


Модифицируем ручку POST api/v1/orders сервиса eda_core. 
1. Делаем ID адреса не обязательным. При его наличии он пишется в заказ. 

Новая схема ручки https://a.yandex-team.ru/arcadia/taxi/schemas/schemas/services/eda-checkout/api/api.yaml
Новая схема GoCheckoutRequest https://a.yandex-team.ru/review/2436862/files/15

## Изменение в API:
сервис: eda_core, ручка: api/v2/cart/go-checkout
1. Добавляем новый провайдер адреса. Вместо идентификатора адреса ожидаем полный объект адреса со всеми полями
2. Так же на вход для провадера taxi начинаем ожидать location(координаты) в обязательном порядке 
3. Если на вход пришел объект адреса, в ответе userAddress возвращаться не будет
4. Если на вход пришел объект адреса, в ответе опционально может быть возвращен объект address, в том случае если бэку понадобилось его подкорректировать
