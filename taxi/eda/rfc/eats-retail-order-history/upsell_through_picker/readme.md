# Пуш-уведомления

Необходимо отправлять 2 пуш-уведомления:

1. "Забыли товар? Позвони нам, мы добавим его в корзину"
   **Условия**: event.event_type == STATUS_CHANGE && event.order_status == "new"
2. "Забыли самое нужное? Просто позвоните сборщику и попросите добавить товар в заказ"
   **Условия**:
   Сборщик начал собирать заказ и для него уже сгенерирован номер телефона ИЛИ Сборщик уже собирает заказ и его номер телефона только был сгенерирован
   (event.event_type == STATUS_CHANGE && event.order_status == "picking" && customer_picker_phone_forwarging && (!picker_id || picker_id != event.picker_id))
   || (order_status >= "picking" && order_status < "paid" && !customer_picker_phone_forwarging && event.event_type == PICKER_PHONE_FORWARDING_READY)
   Таким образом, пользователь получит пуш-уведомление, когда уже будет доступна кнопка "Позвонить сборщику" на трекинге, и уведомление будет отправлено только один раз даже при переназначении сборщика.

Оба пуш-уведомления должны вести по диплинку на экран трекинга этого заказа.

Отправку этих уведомлений лучше конфигурировать экспериментами.

Если поменялся сборщик, то отправлять пуш-уведомление заново не нужно.
