### Описание
  В рамках задачи по продаже алкоголя на самовывоз в алкомаркетах требуется изменить отображение экрана корзины, а именно: 
    - изменить отображение блока с информацией о способе доставки
    - добавить отображение доп информации внизу экрана

### Ожидаемые изменения

#### Отображение блока с информацией о способе получения

  Объект place.available_shipping_types из ответа ручек eats-cart расширить дополнительным полем
  ```
  text:
    type: string
    description: Строка, содержащая человекочитаемое название способа доставки
    examle: Самовывоз
  ```

  Текст взять из Танкера по ключу из конфига, который не должен быть связан с продажей алкоголя в магазинах, семантика конфига должна быть связана со способами доставки (при реализации разобраться с тем, какие конфиги уже есть, если нет подходящего, то создать).

  Свойство text должно содержать строку, которую нужно будет использовать как заголовок блока, отображающего способ доставки. Для типа доставки pickup у магазинов это свойство должно иметь значение "Самовывоз". 
  Дополнительно, чтоб было понятнее про какой блок тут идет речь: это блок который содержит текст со способом получения и адресом магазина (адрес магазина для ресторанов уже пробрасывается на фронтам, поэтому расчитываю, что на бэке изменения не потребуются).

#### Отображение доп информации внизу экрана

  Ответы ручек eats-cart расширить объектом rules. Новое свойство должно разместиться в поле place.

    ```
      rules:
          type: object
          additionalProperties: false
          required:
            - short
            - full
          properties:
              short:
                  description: Сокращенные правила покупки. Строка может содержать ссылки, завернутые в html теги.
                  type: string
              full:
                  type: object
                  additionalProperties: false
                  description: Полные правила покупки
                  required:
                    - title
                    - text
                  properties:
                      title:
                          description: Заголовок вьюшки, содержащей полные правила покупки
                          type: string
                      text:
                          description: Текст, содержащий полные правила покупки. Строка может содержать ссылки, завернутые в html теги.
                          type: string
  ```

  Шаблоны значений для полей short, full title и full text требуется взять из танкера по ключу из конфига, структура которого описана в RETAILSPECS-161, если, на момент выполнения, конфига еще не будет существовать, то реализовать его. Эти шаблоны параметризовать числом обозначающим длительность хранения собранного заказа в магазине это число тоже взять из конфига, поле storage_time. 
  Перед реализацией уточнить, то ли просто подставить число, то ли нужно вычислить дату и время до какого хранится заказ и подставить в текст её (тогда надо как-то закладываться на таймзону юзера/магазина).

  На фронтах требуется обрабатывать поле rules и отображать данные из него внизу экрана.
