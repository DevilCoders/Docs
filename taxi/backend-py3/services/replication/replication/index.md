# Yandex TechPlatform Replication
Сервис **Replication** предоставляет высокоуровневые декларативные инструменты для описания потоков
данных из источников через промежуточную очередь к конечным потребителям.

## Быстрый старт { #start }
Сначала [ознакомьтесь](docs/arcadia/concepts.md) c коцепциями и требованиями к источникам и о способах репликации.
Затем приступайте к [созданию правил](docs/arcadia/how_to_start.md).

## Базовые возможности сервиса { #basic }
  - Единый, надежный и проверенный подход к СУБД для чтения
  - Единый формат очереди сырых событий сразу в 2 имплементациях из коробки: стриминг и **YT**
  - Декларативный **yaml** формат без кода для решений типовой задачи по поставке данных

## Примеры задач, решаемых сервисом { #examples }
  - Нужна реплика СУБД, такой как **MongoDB, PostgreSQL, MySQL, Oracle** и т.д. в **YT** или **YDB**
    как бекап, но нет времени на написание кода.
    Для этого достаточно декларативно в **yaml** настроить вашу репликацию
  - База бекенда (например, PostgreSQL) сильно "раздулась" из нее нужно начинать удалять данные.
    Тогда [можно использовать](https://wiki.yandex-team.ru/users/desire/data/arxitektura-pg-yt-kak-gorjachee-xolodnoe-xranilishhe/)
    **YT RAW** таблицы как "холодное" хранилище на чтение
  - Аналитикам нужно делать сложные запросы, для этого нужны данные в аналитическом хранилище
    в определенном виде. Для этого с нами напрямую работает
    [**DMP**](https://wiki.yandex-team.ru/dmp/tools) -
    это масса инструментов для построения таблиц в **YT** и **Greenplum** и отчетов в **Tableau**
