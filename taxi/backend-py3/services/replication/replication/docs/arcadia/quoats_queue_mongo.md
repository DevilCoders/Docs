# Очередь репликации Queue Mongo
Стандартный способ репликации - через промежуточную очередь, в качестве которой на данный момент выступают кластера **MDB Mongo**.
- При небольшом потоке данных **~1-2 MB/s**, можно использовать стандартные настройки, не думать о новой очереди.
- При большем потоке можно считать что 1 кластер монги **10 CPU, 80 GB RAM, 300 Gb SSD** выдержит поток до **25 MB/s** с необходимым запасом ~20-30% по нагрузке.
- Более точные расчеты будут произведены и расписаны детальнее позже.

## Заказ ресурсов
На данный момент очереди поделены по направлениям, в каждом направлении несколько кластеров.
Заказ ресурсов и подключение к сервису производится через обмен квоты, на данный момент стоит лично договариваться с мейнтейнерами сервиса.

## Флейворы кластеров
На данный момент используются два флейвора: s3.xlarge (16 CPU и 64Gb RAM) и m3.3xlarge (10 CPU и 80Gb RAM).
