## EXT target

Ext target - это "читатель" сырой очереди репликации. Сейчас используется **dwh** для построения **ODS** YT таблиц в реалтайме

Ссылка на чат поддержки dwh [здесь](https://nda.ya.ru/t/i10qZF184qPa92).

Напротив таргетов можно найти ссылки replication cron. У ext-target'ов ссылка будет вести на логи в Кибане
(в рамках [тикета](https://st.yandex-team.ru/DMPDEV-3075) её починят).

### Включение ext-таргетов
- Перед включением нужно убедиться, что походы в read_queue уже включены (в проде это верно по умолчанию)
- Если источник с большим объёмом данных (то есть и очередь), например таблица **заказов такси**, то стоит ограничить очередь датой инициализации
- Если такие таргеты вовремя не удалить (**remove**) в случае проблем, то очередь будет [переполняться](diagnostics/queue_mongo_quota_exceeded.md) и
в логах появятся ошибки вида: `Quota exceeded for {rule_name}: occupied space=6000.73 Mb > quota=6000 Mb`

### Пример создания ext-таргета:
```yaml
example_target_ext:
    type: ext
    target: {}
```
