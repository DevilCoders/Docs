Инструмент (крон-таска) позволяет по заданному конфигу чистить старые ноды на YT.

Для корректной работы инструмента в **продакшне**:

1. Необходимо запросить роли для робота [robot-taxi-yt-clean]({{ link_to_yt_cleanup_robot }})
в интерфейсе YT. Для этого нужно перейти в веб-интерфейсе в директорию, которую хочется
чистить от старых нод и выбрать вкладку "ACL", нажать кнопку "request permissions",
ввести имя робота в поле subjects и выбрать роли "read/write/remove" и "mount". К примеру, для
правила чистки ноды **//home/taxi/testing/tmp/** страничка ACL выглядит [так]({{ link_to_acl_example }}).
После этого надо попросить ответственных окнуть появившийся запрос в IDM.
2. Прописать свое правило чистки нод в [конфиге]({{ link_to_yt_cleanup_config }}).
Правила настройки чистки указаны в описании конфига. Оки просить в [чатике]({{ link_to_replication_telegram_chat }}).
3. Проверить, что ноды в нужной директории чистятся корректно по [логу]({{ link_to_yt_cleanup_cron }})
крон-таски. Можно искать нужные логи по full_path своего правила, уровень логов INFO.

Для корректной работы инструмента в **тестинге**:

Те же шаги, что и выше, только доступы в ACL, скорее всего, запрашивать не надо, так как
используется robot-taxi-test (но лучше проверить по логам крон-таски в тестинге).
