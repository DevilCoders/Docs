Ниже по тексту под термином `redux-экшены` понимается action creator, т.е. функция возвращающая plain-object, передаваемый в редьюссеры.

Все действия, влияющие на состояние приложения, вы делаете через redux-экшены, потому важно их удобно организовать, чтобы у разработчика была возможность быстро понять что он может вообще сделать с состоянием приложения в рамках уже написанного кода.

В Админке есть следующие источники redux-экшенов:
* Файлы редьюссеров, у которых есть `export const actions`
* Отдельные файлы с экшенами, сгруппированными логически, например файлы в папке `packages/*/src/actions` (DEPRECATED)
* Экшены сгенерированные из инстансов АПИ через хелпер `createActions` (DEPRECATED)
* Экшены сгенерированные из [сервисов](/arcadia/taxi/tariff-editor/docs/5.4-sagas-services.md) через хелпер `createService`

На данный момент создание новых редьюссеров и экшенов для них запрещено. Все экшены генерируются автоматически из сервисов, все чтения данных происходят из операций.

В легаси разделах можно встретить рутовый файл `actions.ts` который аггрегирует экшены бандла, такие разделы надо рефакторить, удалять оттуда редьюссеры и заменять их операциями, в итоге остануться только сервисы для которых файл `actions.ts` не нужен и его можно удалить
