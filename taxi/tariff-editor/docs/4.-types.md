В репозитории есть две ключевых папки, в которых могут быть описания общих типов - `node_modules/@types` и `packages/types/src`.

Папка `packages/types/src/typings` служит хранилищем для файлов деклараций типов (`*.d.ts`) и подключается как рутовый источник типов наравне с `node_modules/@types`, в ней могут хранится как описания модулей которых нет в `node_modules/@types`, так и (до)переопределния того что есть.

В каждом пакете может быть папка `src/types` содержащая общие для пакета типы.

Также типы могут располагаться в следующих местах:
* В любых ts файлах при условии, что описание типа занимает в районе 10 строчек кода, любые объемные описания выносятся в отдельный файл
* Отдельные файлы с типами всегда именуются `types.ts`, если файлы требуется разбить логически, то они хранятся в подпапке `types`. Основными местами, где встречаются такие файлы, являются корень бандла, папка react-компонента, папка `api`

Требования к описанию типов:
* Типы должны соблюдать правила линтера
* Описания не должны иметь неймспейсов без очень очень очень веской причины, поскольку вложенность типов раздувает код, усложняет чтение, и плохо работает с автоимпортом и подсказками по типам (во всяком случае в VS Code). Единственное место, где сейчас есть неймспейсы - это легаси тайпинги АПИ

* Стейт бандла описывается следующим образом

bundle/types.ts
```typescript
import {ExtendedState} from '_infrastructure/store';

export type BundleState = ExtendedState<{
   ...
}>;
```

* Именование енумов должно быть таким какое [принято](https://www.typescriptlang.org/docs/handbook/enums.html) в TypeScript

types.ts
```typescript
enum Direction {
    Up = 1,
    Down,
    Left,
    Right,
}

const enum Response {
    No = 'NO',
    Yes = 'YES',
}
```
* Переопределение свойств осуществляется с помощью хелпера `Assign`, который рекурсивно переопределяет свойства первого типа на свойства второго, включая их обязательность (знак `?`)

```typescript
type StoryView = Assign<Story, {
    cid?: string;
    languages: StoryLocale[];
    experiment?: {
        range?: {
            from?: string | number;
            to?: string | number;
        }
    };
}>
```
