# Файловая структура к которой мы стремимся (Это пока черновик)

Мы хотим, чтобы во всех проектах была одинаковая структура, это позволит тратить меньше мыслетоплива на поиск файлов и на выбор подходящего места для нового файла.
Плюс, это позволит добавить больше кодогенерации, например, создание новой страницы.

```
...
+-- realty-core // общее между несколькими сервисами
|   +-- app // TODO серверный код
|   +-- view // верстка
|   |   +-- components // общие компоненты
|   |   |   +-- common // TODO тут не понятно как лучше назвать. была идея, что здесь адаптивные компоненты
|   |   |   +-- deskpad // компоненты для декстопов
|   |   |   |   +-- MyComponent
|   |   |   |   |   +-- index.tsx
|   |   |   |   |   +-- styles.module.css
|   |   |   +-- touch-phone // компоенты для тачей
|   |   +-- libs // общие либы
|   |   +-- enhancers // общие хоки
|   |   +-- modules // тут храним бизнес-логику, сгруппированную по фичам и не храним компоненты
|   |   |   +-- profiles // фича профили
|   |   |   |   +-- libs // её либы
|   |   |   |   +-- enhancers // её хоки
|   |   |   |   +-- selectors/index.ts // её селекторы, в одном файле или нескольких
|   |   |   |   +-- reducers/index.ts // её ридьюсеры, может быть один, а может быть несколько и combineReducers
|   |   |   |   +-- actions/index.ts // её экшены, в одном файле или нескольких
  types // ts типы
...    
+-- realty-mobile
|   +-- app
|   +-- view
|   |   +-- components // компоненты
|   |   |   +-- MySuperTitle
|   |   |   |   +-- MySuperTitleBadge // может быть сабкомпонент, но он не должен использоваться за пределами MySuperTitle и это максимальная вложенность компонентов
|   |   |   |   |   +-- index.tsx
|   |   |   |   |   +-- styles.module.css
|   |   |   |   +-- index.tsx
|   |   |   |   +-- styles.module.css
|   |   +-- enhancers // хоки
|   |   +-- libs // либы
|   |   +-- pages // входные точки страниц
|   |   |   +-- SearchPage
|   |   |   |   +-- index.tsx
|   |   |   |   +-- styles.module.css
|   |   +-- modules // аналогично как в realty-core
|   |   +-- entries // входные точки бандлов
|   |   |   +-- common // бандл common
|   |   |   |   +-- reducer.ts // его корневой ридьюсер
|   |   |   |   +-- routes.ts // его роуты
|   |   |   |   +-- server.ts // его entry файл для серверного бандла
|   |   |   |   +-- client.ts // его entry файл для клиентского бандла
|   |   +-- vars/vars.css // css переменные
 ...
 ```
