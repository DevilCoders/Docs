FBAN — инструмент, который позволяет автоматически закрывать (открывать) отстающие (актуальные) реплики на поиск.
Применяется в том случае, когда для сервиса важно поддерживать актуальность документов.
Каждый сервис может иметь свои собственные требования свежести данных,
поэтому FBAN конфигурируется посервисно. Настройки хранятся в ардкадии,
см. [settings.yaml](https://a.yandex-team.ru/arc_vcs/saas/tools/devops/fban/settings.yaml?rev=93d872be24).

Описание возможных опций:
```yaml
nanny_name: <str>     # название няня-сервиса
service: <str>        # название saas-сервиса
ctype: <str>          # ctype saas-сервиса
cnt: <int>            # максимальное количество реплик, которые можно забанить в рамках одного шарда, нельзя использовать вместе с "perc"
perc: <int>           # то же, что и "cnt", но в процентах, нельзя использовать вместе с "cnt"
itype: <str>          # itype saas-сервиса, возможные значения: rtyserver или fusion (для свежести)
ban_minutes: <int>    # отставание реплики в минутах, начиная с которого ее следует закрывать на поиск (включительно)
unban_minutes: <int>  # отставание реплики в минутах, до которого она считается актуальной и может быть разбанена (не включительно)
check_interval: <int> # интервал проверки актуальности реплики

# Прочее:
# Обязательно нужно прописать либо "perc", либо "cnt".
# По умолчанию ban_minutes=35, unban_minutes = 25.
# Необходимо соблюдать условие: unban_minutes <= ban_minutes.
# Неверно сконфигурированные сервисы будут проигнорированы.
```
