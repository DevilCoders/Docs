# Конфигурация и переменные окружения

В качестве конфигов, для приложения используются файлы с предустановленными переменными окружения.
Все файлы конфигурации находятся в директории [/configs](../configs).
Название файла формируется из префикса `.env.` и названия окружения, в котором будет запускаться приложение.
Под окружением подразумевается значение переменной `APP_ENV`.

`APP_ENV` — основная переменная для передачи названия окружения, в котором работает приложение.

`NODE_ENV` — системная переменная от значения которой зависит сборка проекта.
Может принимать следующие значения:
- production
- development
- test

Значение переменной `NODE_ENV` крайне редко используется в коде проекта.
Пока что единственный пример — инициализация next.js.

Если значение переменной `APP_ENV` не установлено, оно будет равно значению `NODE_ENV`.

Если значение переменной `NODE_ENV` не установлено, оно будет равно `production`.
