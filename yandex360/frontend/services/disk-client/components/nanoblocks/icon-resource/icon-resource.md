# Наноблок `nb-icon-resource`

## API

```javascript
params = {
    'size': 's' | 'm' | 'l' | 'xl'
    'type': 'icon' | 'image' | 'document' | 'video'
    'icon': 'jpg' | 'png' | 'doc' | ...
    'preview': '...'
    'shared': true()
    'icon-video': true()

    'class': ...
    'attrs': {
        ...
    }
}
```

Есть специальные атрибуты `class` и `attrs` — работает точно так же как в наноостровах: добавляет еще один класс или какие-нибудь атрибуты для корневой ноды блока.

### size

Определяет размеры:

* `xl` - 240x240
* `l` — 145x145
* `m` - 80x80
* `s` - 30x30

### type

Определяет, какой тип отображения будет выбран:

* `dir` — иконка папки
* `icon` — иконка ресурса (по умолчанию)
* `image` — квадратное обрезанное превью
* `document` — превью документа, с сохранением пропорций
* `video` — квадратное обрезанное превью с иконкой проигрывания
* `file` — иконка файла без превью

### icon

Определяет, какой тип иконки будет использован для ресурса

* `root` — корневая папки (с НЛО)
* `trash` — корзина
* `dir` — папка
* `photosream` — папка фотокамеры
* `shared` — шареная папка
* `screenshots` — папка скриншотов
* `jpg` | `pdf` ... — иконки для различных файлов

### icon-video

Определяет, нужно ли рисовать иконку start-play

### preview

Определяет урл до картинки, которую нужно показать в качестве превью.

Сейчас нет джаваскриптовой функциональности прелоадера, в листинге это реализовано как часть вьюшки ресурса.
А в слайдере прелоадер скрыт в ЦСС. Нужно не забыть это убрать, когда js-часть блока будет реализована.

### published

Показывать или нет иконку публичности. По умолчанию она скрыта.

### pointer

Определяет указатель мыши на иконке

### static

Используется в паре с полем preview. Означает, что превьюшку сразу считаем загруженной.
И не надо рисовать иконку вместо превью, а затем вызывать в js загрузку.
