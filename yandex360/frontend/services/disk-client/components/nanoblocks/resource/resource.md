## Наноблок `nb-resource`

Блок состоит из трех элементов: `nb-icon-resource` или `nb-icon-group`, чекбокса и имени ресурса. Кроме того, блок инкапусулирует в себе некоторое поведение: у него есть состояния.

## API

```
{
    // внешний вид
    'kind': 'tile' | 'frame' | 'line' | 'stripe' | 'photo'
    'type': 'icon' | 'image' | 'document' | 'video' | 'audio'
    'icon': 'jpg' | 'png' | 'doc' | ...
    'size': 's' | 'm' | 'l' | 'xl' ...

    // конфигурация
    'selectable': 'single' | 'multi'
    'hoverable': true()
    'hover-shaded': false()
    'button-delete': false()

    // состояния
    'selected': true()
    'disabled': true()
    'published': false()

    // данные
    'name': ...
    'preview': ...
    'count': ...
    'space': ...
    'date': ...
}
```

Часть параметров повторяет `nb-icon-resource` и `nb-icon-group`, потому что эти блоки используются здесь.

### kind

* `tile` — иконка или превью, чекбокс внутри, имя снаружи. Используется в листингах видах типа `tile` и `icons`
* `frame` — иконка или превью, чекбокс и имя внутри. Используется в таких же типах списка, но в фотокамере
* `line` — иконка или превью, чекбокс и имя снаружи. Используются в списочных видах, здесь же выводится метаинформация ресурса.
* `stripe` — иконка или превью, имя снаружи. Представляет собой "короткий" вариант `line` без метаинформации и чекбокса
* `photo` — квадратная превью, есть переключатель, имени нет. Большая превьюшка для фотосреза.

### type

Определяет, какой тип отображения будет выбран:

* `icon` — иконка ресурса (по умолчанию)
* `image` — квадратное обрезанное превью
* `document` — превью документа, с сохранением пропорций
* `video` — квадратное обрезанное превью с иконкой проигрывания
* `audio` - к стандартному виду добавляются контролы для плеера

### icon

Определяет, какой тип иконки будет использован для ресурса

* `root` — корневая папки (с НЛО)
* `trash` — корзина
* `dir` — папка
* `jpg` | `pdf` ... — иконки для различных файлов

### size

Определяет размеры:

* `xl` — 240x240
* `l` — 145x145
* `m` — 80x80
* `s` - 30x30

### preview

Определяет урл до картинки, которую нужно показать в качестве превью.

Сейчас нет джаваскриптовой функциональности прелоадера, в листинге это реализовано как часть вьюшки ресурса.
А в слайдере прелоадер скрыт в ЦСС. Нужно не забыть это убрать, когда js-часть блока будет реализована.

### selectable

Может быть в значениях `single` или `multi` (по умолчанию — первое). Влияет на состояние `selected`:

* `single` — если `selected: true()`, то добавим только модификатор;
* `multi` — если `selected: false()`, то добавим еще и чекбокс (и модификатор).

### hoverable

Отвечает за состояние ховера ресурса. По умолчанию всегда включено.

### hover-shaded

Отвечает за состояние наведеления на ресурс с приглушение (притемнением) превью. По умлочанию отключено.

### button-delete

Включает кнопку удаления для режима `kind: photo`. По умолчанию кнопка отключена.

### selected

Состояние выделенности: добавляется класс `nb-resource_selected`, который можно застилизовать, кроме того чекбокс (если есть) приводится в состояние `checked`

### published

Выводить или нет иконку публичности. По умолчанию ее нет.

### disabled

Задизейбленный ресурс. Используется, например, когда ресурс поставили на загрузка и она еще не завершилась.

### name

Имя ресурса

### preview

Если есть превью, то это урл до картинки

### count

Число, которое будет показано в `nb-icon-group`, используется при drag'n'drop

### space

Размер файла

### date

Дата создания

## JS API

```
var resource = nb.block(nodeResourse);

/**
 * Select resource
 */
resource.select();

/**
 * Deselect resource
 */
resource.deselect();

/**
 * Return resource selected state
 */
resource.isSelected();

/**
 * Toggle selected state
 */
resource.toggleSelected();
```

## События

* `nb-delete` - событие удаления ресурса. Возникает при клике по иконке удаления ресурса. Иконка доступна в режиме: `kind: photo`
