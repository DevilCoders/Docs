## Как попадать в эксперименты

### вариант 1: `test-id`
В рамках сессии: добавить в запрос query string параметр `test-id={id выборки 1}_{id выборки 2}...`

Можно захардкодить эксперименты на стенде, чтобы автоматически в них попадать. Для этого при деплое ресурса в Qloud
1. Перейти в **Variables**
2. В поле **Environment variables** добавить строку `TESTS={id выборки 1}_{id выборки 2}...`

На dev-стенде можно хардкодить эксперименты определением переменной окружения `TESTS` через `export` (например, в .bashrc), либо при запуске серверной части.

Если при работе на стенде с хардкодовыми экспериментами добавить в запрос qs параметр `test-id`, то попадем и в
хардкодовые и в переданные выборки.

### вариант 2: жук
- зайти на [морду Яндекса](https://yandex.ru)
- открыть жука в нижнем углу
- перейти на вкладку "Эксперименты"
- найти эксперимент по его id или названию флага
- нажать "Залипнуть"

⚠️ залипание через жука приведёт к отключению всех остальных экспериментов, в том числе после рефреша
и на других сервисах Яндекса. Для возвращения в нормальный режим экспериментов нужно будет отлипнуть обратно,
для этого нужно опять зайти во вкладку "Эксперименты" в жуке и нажать "Отлипнуть"
