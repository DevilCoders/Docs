# Быстрый старт

Отладка и разработка Front-end приложения ведется на "машине" `ufo.dev.yandex.ru`. Вход на неё можно осуществить по прикрепленному на staff ssh ключу или через доменный логин и пароль.

## Разворачивание приложения

1. Войти на `ufo.dev.yandex.ru`. Для этого нужно запустить в консоле команду:

    ```
    ssh ufo.dev.yandex.ru
    ```

2. Создать директорию `ufo/www` в пользовательской папке:

    ```
    mkdir -p ufo/www
    ```
3. Склонировать репозиторий Диска в `ufo/www` директорию:

    ```
    git clone git@github.yandex-team.ru:UFO/www.git ufo/www
    ```

    *Замечание:* клонирование по ssh требует создание и подключение ssh ключа к github репозиторию.
    Прочесть можно тут: [Generating ssh keys. Github.](https://help.github.com/articles/generating-ssh-keys/)

4. Обновить сабмодули Диска:

    ```
    cd ufo/www
    git submodule update --init
    ```

5. Запустить `descript`:

    ```
    cd ~/ufo/www
    make restart
    ```

6. Запустить приложение:

    ```
    npm start
    ```

    или

    ```
    make server-client
    ```

7. Запущенное приложение доступно по URL `<user-name>.ufo.dev.yandex.ru`

## Запуск дополнительных копий приложения

Все созданные папки в `ufo` директории могут быть открыты как новые поддомены `ufo.dev.yandex.ru`.
Например, можно было создать директорию: `ufo/www2` и применить шаги выше к ней. Тогда приложение будет открываться по URL `<directory-name>-<user-name>.ufo.dev.yandex.ru`

Получается, что все внутренние папки `ufo` (кроме `www`) требуют при написании домена вызова приложения дополнительного префикса имени папки с дефисом перед именем пользователя.
