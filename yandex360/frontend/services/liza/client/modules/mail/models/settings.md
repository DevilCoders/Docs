setSettingOn(name, \[callback\])
--------------------------------
Установить значение true<br>
Подразумеваем, что настройка имеет Boolean значение, но реально сохраняется значение "on".<br>
Значение "on" эквивалентно true.<br>
Функция обратного вызова принимает два аргумента:
* isSuccess - boolean признак успешного сохранения настройки
* errors - массив строковых кодов ошибок


**Parameters**

**name**:  *String*,  название настройки

**[callback]**:  *Function*,  функция, выполняемая при завершении сохранения значения

setSettingOff(name, \[callback\])
---------------------------------
Установить значение false<br>
Подразумеваем, что настройка имеет Boolean значение, но реально сохраняется значение "".<br>
Значение "" эквивалентно false.<br>
Функция обратного вызова принимает два аргумента:
* isSuccess - boolean признак успешного сохранения настройки
* errors - массив строковых кодов ошибок


**Parameters**

**name**:  *String*,  название настройки

**[callback]**:  *Function*,  функция, выполняемая при завершении сохранения значения

setSettings(settings, \[callback\])
-----------------------------------
Установить значение<br>
Физически значения настроек могут быть сохранены только строкой или числом.<br>
Поэтому выполняются следующие преобразования типов:
* true -> "on"
* false -> ""
* undefined -> ""
* NULL -> ""
* object -> encode json string
* array -> encode json string
Функция обратного вызова принимает два аргумента:
* isSuccess - boolean признак успешного сохранения настройки
* errors - массив строковых кодов ошибок


**Parameters**

**settings**:  *Object*,  объект настроек

**[callback]**:  *Function*,  функция, выполняемая при завершении сохранения значения
