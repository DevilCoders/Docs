1. Создать пользовательскую ветку с происзвольным названием
2. Запустить команду для создания релизной ветки `npm run package x.y.z-beta1`. Команда создаст релизную ветку. запушит её и вернется на исходную.
3. Команда выпллюнет ссылки на ресурсы (само приложение и setttings)
4. Обновить конфиг сервера
    4.1 Запустить `make mailfront-config` в корне проекта
    4.2 Если команда сразу не выдала `Resource id:`, то узнать id собранного ресурса – https://sandbox.yandex-team.ru/resources?type=MAILFRONT_CONFIG&limit=3&attrs=%7B%22component%22%3A%22tavern%22%7D
    4.3 поменять id ресурса в шаблоне `tools/specs/front-app.j2`
5. Выкатить приложение на prestable (`cd tools/specs && make prestable put-prestable`)
6. Оповестить интересантов
7. Убедиться что все работает как ожидается:
    7.1 Шторка в мобильных приложениях
    7.2 Таверна в почте
    7.3 Таверна в диске
    7.4 Таверна в других сервисах
8. Если есть проблемы, то внести правки и повторить с шага 2. (`npm run package 1.2.3-beta2` -> `...beta3` ... `...beta4`)
9. Запустить команду для создания релизной ветки `npm run package 1.2.3`. Команда создаст релизную ветку. запушит её и вернется на исходную.
10. Выкатить приложение на prod (`cd tools/specs && make prod put-prod`)
11. Вмержить ветку.
