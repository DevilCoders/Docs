# Запуск регресса из релизного тикета


## Где работает автозапуск

**Автозапуск работает в следующих компонентах:**
* в клиенте;
* в заметках;
* в админке b2b;
* в паблике;
* в морде;
* в dv


## Что надо сделать

{% note info %}

Дежурный разработчик перед запуском регресса должен убедиться в следующем:

* все выкачено на номерной стенд (в случае dv еще и раскатан корп)
* версия пакета на стенде правильная
* стенд живой и с ним можно работать

{% endnote %}

**Порядок запуска в асессоров:**

1. переводим тикет руками в статус "Можно тестировать"
2. откроется форма для запуска асессоров. в ней нужно заполнить следующие поля:
    * **Бранч** - указывем, то, что записано в соотствествующем поле в ST (например, `web-rc-123`)
    * **Ссылка на стенд** - указываем стенд, на который выкатили релиз, формат наш общепринятый (например, `https://telemost2.dsp.yandex.ru`)
    * **Запустить на асессорах** - нужно выбрать `Обычный регресс`
    * **Agnitum fix build** - нужно указать версию пакета
3. нажать кнопку "Применить"
4. вы восхитительны

{% note info %}

Немного про поле **Запустить на асессорах**:

* `Обычный регресс` - запускается обычный регресс или бизнес-логика. Это то, что нужно выбирать в 100% случаев
* `Полный регресс` - запускаются вообще все кейсы, даже низкоприоритетные. Дорого и **запускать только после согласования с QA**

{% endnote %}


## Что нужно делать с тикетом дальше

**Ничего не нужно**

Тикет временно перейдет в статус "Готов к выкладке на асессоров", разметится и уйдет асессорам

Ожидаем запуска и завершения регресса
