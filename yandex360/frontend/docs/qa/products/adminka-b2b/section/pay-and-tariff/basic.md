# Базовое про структуру раздела


У пользователя может быть множество состояний и комбинаций в админке, например:
* может быть N+1 организаций, где пользователь плательщик
* может быть N+1 организаций, где пользователь не-плательщик
* множество организаций может находится на разных тарифных планах
* организации могут находится в разных флоу оплаты (постоплата/предоплата)

Такое многообразие требует адекватного и понятного отображения в интерфейсе
