# Balance


**Balance** - это программный комплекс, осуществляющий учет объема потребляемых услуг, расчет и списание денежных средств в соответствии с тарифами компании

Явлется нашим внутренним продуктом, который развивается поддерживается силами команды Биллинга


## Уровни доступа и стенды

### Доступы

Доступ тестированию 360 разрешен только до тестового контура

Это означает, что все мероприятия, которые проводит тестирвание (прямое ручное тестирование фичи, автотесты, асессоры), проходят с использованием тестовой DB Баланса

Доступ разрешен как к тестовым ручкам, так и к интерфейсам

#### пропал/хочу заказать доступ до тестового контура Баланса

До тестовых контуров админки, которые приведены ниже, нужно запросить персональный доступ

{% note info "Чтобы запросить доступ нужно:" %}

* взять свой `passport_uid` (проще всего перейти по ссылке админки и скопировать его в правом верхнему углу интерфейса)
* перейти по [форме Баланса](https://wiki.yandex-team.ru/balance/forms/)
* создать заявку на выдачу роли на сервис `Яндекс для Бизнеса - 671`, где `671` - это id сервиса
* ожидать решения в созданном тикете
* _p.s. в тикете лучше отдельно попросить добавить ваш уид в постпереналивочный скрипт (про переналивки читай ниже)_

Лучше всего попросить своего руководителя создать запрос роли сразу на группу

{% endnote %}

## Переналивки Баланса

**Переналивка** - это ежемесячное мероприятие Баланса, при котором команда Билоинга выполняет упражнения по перекладываю снэпшота с продовой база на тестовую в целях поддержки актуальности контура

Это усложняет жизнь тестированию 360 тем, что **все тестовые данные, которые лежат в Балансе, стираются!**

{% note info "Как узнать о переналивках" %}

* подписаться на рассылку `balance-qa-alert`, где ребята заблаговременно предупреждают о переналивках
* помнить, что все переналивки, как правильно, приходятся на 15 число каждого месяца

{% endnote %}
