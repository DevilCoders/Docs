# Billing 360

**Billing 360** - это критически важный сервис Персональных сервисов, который обеспечивает продажи всех продуктов Яндекс 360 как для B2C, так и для B2B клиентов.

_Основной функционал сервиса_ - продать подписку пользователю или организации, и проследить чтобы все оплаченные услуги были выданы


## Работа с админкой

### Сет b2b продуктов

Админка работает с сетом b2b продуктов:

* `postpaid for b2b 360` - линейка продуктов с постоплатой
* `prepaid for b2b 360` - линейка продуктов с предоплатой
* `edu prepaid for b2b 360` - линейка для образовательных учреждений

Это сет продуктов, внутри которых есть отдельные продукты!

**Пример продукта:**
```ini
mail_pro_b2c_premium50000_v20210813

в него входят фичи:
* дополнительное место
* красивый адрес
* умная подборка писем для просмотра в первую очередь
* резервная копия важных писем
* безлимит для видео с телефона
* автосохранение файлов в программе для Windows
* неограниченное скачивание с публичных ссылок
* расширенная история изменений
* приоритетная поддержка
* без рекламы
```

### Типы продуктов

#### User product

Тим продукта определяется полем `billing_type`, который бывает:

* **TRUST** - продукт, который продается через веб. Списыванием денег занимается Траст. Биллинг 360 управляет подписками - выдачей/апгрейдом/отменой
* **FREE** - бесплатный продукто
* **GROUP** - групповой продукт

#### Group product

Продукты различаются по `payment_type`:

* **prepaid** - продукты на предоплате
* **postpaid** - продукты на постоплате

Полный набор знаний о биллинге 360 можно получить на их [страничке на Вики](https://wiki.yandex-team.ru/disk/billing360/)
