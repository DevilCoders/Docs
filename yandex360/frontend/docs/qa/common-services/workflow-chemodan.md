# Воркфлоу тикетов в очереди CHEMODAN


## Вводная

Так как практически все задачи на общие компоненты проходят через очередь **CHEMODAN**, то они подчиняются установленному воркфлоу данной очереди
Ниже описаны статусы, через которые проходят тикеты

{% note info %}

Более подробно со схемой ворфлоу можно ознакомиться тут: https://wiki.yandex-team.ru/users/totora/instrukcija-po-zapusku-asessorov-razrabotkojj/

{% endnote %}

## Статусы

Статусы описаны по порядку от зарождения задачи и до ее выкатки в прод


|       Статус        | Описание                                                                                                                                                   |
|:-------------------:|:-----------------------------------------------------------------------------------------------------------------------------------------------------------|
|        Open         | Задача не взята в работу или она вернулась на доработку                                                                                                    |
|     In Progress     | Задача находится в работе                                                                                                                                  |
|      In review      | Задача находится на стадии код-ревью. Тестировать такую задачу нельзя                                                                                      |
|  Review completed   | Код получил, хотя бы, один аппрув от ревьюера                                                                                                              |
|       Resolve       | Задача прошла стадию код-ревью и готовится к передаче в тестироваие                                                                                        |
|   Ready for test    | Задача готова к тестирванию и не находится в стадии активного тестирования                                                                                 |
|       Testing       | Задача, на данный момент, находится в стадии тестирования                                                                                                  |
|       Tested        | Задача получила аппрув от тестирования на влитие в trunk                                                                                                   |
|     Tested dev      | Может перевести разработчик, минуя тестирование. Такой статус задаче ставится только после согласования с тестированием и только там, где оно не требуется |
|    Ready for dev    | Задача находится на стадии влития в trunk                                                                                                                  |
| Awaiting deployment | Задача находится в trunk'е и ждет релиза                                                                                                                   |
|       Closed        | Задача выкачена в продакшен                                                                                                                                |
