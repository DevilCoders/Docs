### Компонент выбора красивого почтового адреса

Используется в десктопной веб-почте (liza) и на странице подписки (tuning).

Содержит сам визуальный компонент, а также API для работы с красивым адресом - саджест,
регистрация, статус регистрации и отмена адреса.

Поставляется NPM-пакетом в виде исходников на TypeScript, компилируется и собирается на стороне проектов.

#### Запуск страницы с примерами

```$bash
npm ci
npm run storybook
```

#### Основная структура

```
__tests__ – unit-тесты на логику
locales – файлы с переводами ключей, используемых в компоненте
src
├─ components – отдельные части компонента выбра красивого адреса
│  ├─ ConfirmDialog – конфёрм подключения красивого адреса с информацией, что повторное подключение возможно не ранее, чем через 6 месяцев
│  ├─ DomainStatus – статус введённого пользователем адреса (недоступен / свободен) + попап с ошибкой валидации
│  ├─ EmailInput – "составное" поле ввода красивого адреса из 2 частей (логин и домен) с валидацией
│  ├─ FetchError – сообщение об ошибке при подборе красивого адреса
│  ├─ Fetching – крутилка, показываемая на время выполнения саджеста красивого адреса
│  ├─ NavBar – панелька с ссылками "Узнать больше" и (опционально) "Все тарифы"
│  ├─ NavLink – стилизованная ссылка для панельки. Также используется в конечных проектах для единого вида подобных ссылок
│  └─ SuggestList – список кнопок с адресами, полученными из саджеста. Клик по такой == выбор адреса
├─ helpers
├─ lego-components – обёртки для лего-компонентов с нужными темами и модификаторами
├─ api.ts – API для вызова моделей красивого адреса
├─ domenator-types.d.ts – тайпинги для ответов моделей доменатора
├─ index.tsx – верхнеуровневый компонент для выбора красивого адреса
└─ with-i18n.tsx – обёртка для автоматической локализации и сборки по языкам (не работает в Почте)
stories – примеры компонента для storybook-а
index.ts – входная точка, экспортирующая компонент и обёртку для его локализации
```
