* `_id`
    * Уникальный идентификатор записи в базе, должен иметь парный `Customization._id`
    * `string`
    * default отсутствует
* `app_system_flag`
    * Правило применится, если приложение является системным
    * `string`
    * default отсутствует
        ```json
        "app_system_flag" : "true",
        ```
    * Возможные значения
        * `"true"` (строка, не бул)
        * `"false"` (строка, не бул)
* `applications`
    * Правило применится, если запрос пришёл от любого приложения из списка
    * `array[{name, {version{from, to}}}]`
    * default `[]`
        ```json
        "applications" : [
            {
                "name" : "launcher",
                "version" : {
                    "from" : "1",
                    "to" : "2"
                }
            },
            {
                "name" : "afisha",
                "version" : {
                    "from" : "3.1.1",
                    "to" : "7.2"
                }
            }
        ],
        ```
    * `name`
        * Значения регистрозависимые, берутся как индексы `_id` коллекции `localization._Yandex_apps`
        * `string`
        * default `""`
        * Возможных значений слишком много, например,
            * `"Auto.ru", "afisha", "561b6ba629f3e21c9d541dcb", "5c62c57715b05400667209ee", ...`
    * `version`
        * Может отсутствовать полностью, может отсутствовать любая из границ
        * `{from, to}`
        * default `{}`
        * `from`, `to`
            * Строка в виде списка цифр, разделённых точками. Длина списка не ограничена.
                * `"4.2"`
                * `"4.2.2"`
                * `"4.2.2.5"`
            * `string`
            * default `""`
* `audience`
    * Если параметр задан, условие выполняется для процента пользователей. Процент берётся по `uuid` - равномерно распределённая величина, описана в `uuids`.
    * `float [0.0, 1.0]`
    * default `1.0`
        ```json
        "audience" : 0.3,
        ```
* `audience_offset`
    * Используется совместно с `audience`. Условие выполняется для части аудитории `[audience; audience + audience_offset]`
    * `float [0.0, 1.0]`
    * default `0.0`
    * Если аудитория разбита на три равные части, что последняя часть может быть описана как
        ```json
        "audience" : 0.34,
        "audience_offset" : 0.66,
        ```
* `clids_set`
    * Правило применится, если у приложения выставлены клиды. Список клидов имеет две вложенности. Внешний список `clids_set` обрабатывается по ИЛИ.
    * `array[{clids : array[{number, value}]}]`
    * default `[]`
        ```json
        "clids_set" : [
            {
                "clids" : [
                    {
                        "number" : 1,
                        "value" : 12345
                    },
                    {
                        "number" : 2,
                        "value" : 67890
                    }
                ]
            },
            {
                "clids" : [
                    {
                        "number" : 3,
                        "value" : 45678
                    }
                ]
            }
        ],
        ```
    * `clids`
        * Внутренний список, обрабатывается по И, `number` должны быть уникальными
        * `array[{number, value}]`
        * default `[]`
        * `number`, `value`
            * `int`
            * default отсутствует
* `detectOperatorByIp`
    * Определять ли оператора связи по ip-адресу пользователя. Работает в паре с `operators`.
    * `boolean`
    * default `false`
        ```json
        "detectOperatorByIp" : true,
        ```
* `deviceTypes`
    * Если поле задано, то девайс пользователя должен соответствовать любому из перечисленных значений. Список значений фиксированный.
    * `array["phone", "phablet", "tablet", "desktop"]`
    * default `[]`
        ```json
        "deviceTypes" : [
            "phone",
            "tablet"
        ],
        ```
* `enabled`
    * Применять ли правило клидирования в целом
    * `boolean`
    * default `true`
        ```json
        "enabled" : true,
        ```
* `extendedParams`
    * Список дополнительных параметров. Можно указать только часть из них. Все параметры имеют тип `string`, регистрозависимые, default отсутствует. Правило применится, если все параметры совпадут.
        ```json
        "extendedParams" : {
            "countryCode" : "250",
            "clid" : "1234",
            "installReferrer" : "onepr16052016",
            "referrerPattern" : "^(utm_source=google-play&utm_medium=organic)?$",
            "brandId" : "olympic2018"
        },
        ```
    * `countryCode`
        * Идентификатор страны оператора связи (MCC), например, `"250"` для России
    * `clid`
        * Уникальный идентификатор партнёра, deprecated
    * `installReferrer`
        * Referrer от Google Play Market, проверяется на точное соответствие
    * `referrerPattern`
        * Регулярное выражения для referrer от Google Play Market, более гибкий вариант, чем `installReferrer`
    * `brandId`
        * Идентификатор бренда
* `geoCodes`
    * Список гео-кодов территорий, на которых должен находиться пользователь для срабатывания условия. Пустой список игнорирует проверку гео-кода.
    * `array[uint64]`
    * default `[]`
        ```json
        "geoCodes" : [225, 203],
        ```
* `geoCodesBlacklist`
    * Список гео-кодов территорий, на которых не должен находиться пользователь для срабатывания условия. Пустой список игнорирует проверку гео-кода.
    * `array[uint64]`
    * default `[]`
        ```json
        "geoCodesBlacklist" : [225, 203],
        ```
* `init_countries`
    * Приложение должно быть запущено в одной из стран,
    * `array[string]`
    * default `[]`
        ```json
        "init_countries" : [
            "ru",
            "tr"
        ],
        ```
    * Примеры
        * `"*"` - применять правило для всех стран
        * `"ru"` - применять только для `ru`, название страны строго из двух символов, регистронезависимое, https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2
        * `"!ru"` - применять для всех стран, кроме `ru`, название страны строго из двух символов, регистронезависимое
* `install_referrer_sources`
    * Источники referrer'а, можно указать несколько значений, обрабатываются по ИЛИ
    * `array[string]`
    * default `[]`
        ```json
        "install_referrer_sources" : [
            "null",
            "broadcast",
            "gpl",
            "hms"
        ],
        ```
    * Возможные значения, регистрозависимые
        * `"null"` реферрер неизвестно откуда получен
        * `"broadcast"` реферрер получен из бродкаста
        * `"gpl"` реферрер получен через Google Play Store's Install Referrer API
        * `"hms"` реферрер получен через HMS
* `locale`
    * Правило сработает для пользователя с указанной локалью
    * `{language, country}`
    * default `{}`
        ```json
        "locale" : {
            "language" : "ru",
            "country" : "RU"
        },
        ```
    * `language`
        * Применять правило для указанного языка
        * `string`
        * default `"*"`
        * Примеры
            * `"*"` - применять правило для всех языков
            * `"ru"` - применять только для `ru`, название языка строго из двух символов, регистронезависимое
            * `"!ru"` - применять для всех языков, кроме `ru`, название языка строго из двух символов, регистронезависимое
    * `country`
        * Применять правило для указанной страны
        * `string`
        * default `"*"`
        * Примеры
            * `"*"` - применять правило для всех стран
            * `"RU"` - применять только для `RU`, название страны строго из двух символов, регистронезависимое
            * `"!RU"` - применять для всех стран, кроме `RU`, название страны строго из двух символов, регистронезависимое
* `models`
    * Устройство пользователя должно быть одним из списка. Если список пустой, устройство не проверяется.
    * `array[{vendor, model}]`
    * default `[]`
        ```json
        "models" : [
            {
                "vendor" : "vendor1",
                "model" : "model1"
            },
            {
                "vendor" : "vendor2",
                "model" : "model2"
            }
        ],
        ```
    * `vendor`
        * Регистронезависимое значение vendor/manufacturer устройства
        * `string`
        * default `""`, но тогда должно быть указано поле `model`
    * `model`
        * Регистронезависимое значение model устройства
        * `string`
        * default `""`, но тогда должно быть указано поле `vendor`
    *
* `operators`
    * Применять правило, если в устройстве пользователя установлена симка одного из перечисленных операторов. При пустом списке, проверка игнорируется. Работает в паре с `detectOperatorByIp`.
    * `array[string]`
    * default `[]`
        ```json
        "operators" : [
            "beeline",
            "megafon"
        ],
        ```
    * Значения регистрозависимые, берутся как индексы `_id` коллекции `localization._Operators`
    * Возможные значения, список может быть неактуальным
        * `"Avea_3", "Avea_4", "beeline", "kyivstar", "megafon", "mts"`
* `platforms`
    * Правило применяется, если операционная система устройства совпадает с одной из перечисленных
    * `array[{name, {version{from, to}}}]`
    * default `[]`
        ```json
        "platforms" : [
            {
                "name" : "android",
                "version" : {
                    "from" : "2.4",
                    "to" : "2.5"
                }
            },
            {
                "name" : "android",
                "version" : {
                    "from" : "4.1.1",
                    "to" : "4.2.2"
                }
            }
        ],
        ```
    * `name`
        * Значения регистрозависимые, берутся как индексы `_id` коллекции `localization._Operation_systems`
        * `string`
        * default `""`
        * Возможные значения, список может быть неактуальным
            * `"android", "apad", "ipad", "iphone", "windows", "wp"`
    * `version` - см. в `applications`
* `time`
    * Промежуток времени срабатывания условия. Любая из границ может отсутствовать.
    * `{from, to}`
    * default `{}`
        ```json
        "time" : {
            "from" : ISODate("2000-10-10T09:10:00.000Z"),
            "to" : ISODate("2010-10-10T09:10:00.000Z")
        },
        ```
        ```json
        "time" : {
            "from" : ISODate("2000-10-10T09:12:00.000Z")
        },
        ```
        ```json
        "time" : {
            "to" : ISODate("2010-10-10T09:10:00.000Z")
        },
        ```
    * `from`, `to`
        * Дата в формате `ISODate`
* `uuids`
    * Список uuid'ов, для которых срабатывает правило. Если список пустой, то для любого. uuid - уникальный идентификатор установки приложения для данного пользователя на данном устройстве (при обновлении приложения uuid не изменяется, но изменяется при переустановке)
    * `array[uuid]`, каждый `uuid` - `array[hex_char], length == 32`
    * default `[]`
        ```json
        "uuids" : ["65ed0db026eb4223864976aaf548d172"],
        ```
        ```json
        "uuids" : [
            "65ed0db026eb4223864976aaf548d172",
            "88aa0db026eea22fe15046aaf548d542"
        ],
        ```
