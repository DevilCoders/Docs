Описание структур сегментов и логики их сериализации.

Для каждого сегмента описаны:
* Мета возможных значений сегмента
* Тип выбранного сегмента
* Логика сериализации сегмента для сохранения в урл
* Логика десериализации сегмента из урла

Реализация логики сериализации сегментов перед запросом в апи лежит на стороне сервиса. _Возможно эту логику можно обобщить в контрибах, пока еще не понял_

Существование отдельной функции сериализации/десериализации сегмента в урле обусловлено тем, что структура сегментов может меняться, поэтому если просто использовать JSON.stringify, мы можем потерять старые сегменты.
