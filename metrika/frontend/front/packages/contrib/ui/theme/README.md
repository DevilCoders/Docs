# Темизация

Решает задачи:
- Переопределять стили для компонентов Lego (уже сейчас)
- Определять собственные css переменные на проект (на будущее)

## Обновление стилей

```
npx lerna run theme:build --scope=@metrika/contrib
```

## Описание

⚠️ На данном этапе — это MVP. В будущем, скорее всего, организация темы будет меняться.

Сейчас есть три темы:

- root
- metrika
- appmetrika

Тема __root__ — это общая тема, которая наследует стили Лего, __metrika__ и __appmetrika__ унаследованы от неё.

Мы можем переопределить значения как на всех проектах, так и на каких-то отдельных.

Кроме того, мы можем дополнительно переопределять значения для _desktop_ и _touch_.

Сейчас все темы пустые и возвращают __тему Лего по умолчанию__, кроме, __metrika@desktop__ в которой переопределено несколько свойств.

В будущем, возможно, мы захотим сделать одну тему на все проекты, либо что-то добавить или убрать. Пока не ясно.

Кроме того, возможно, для удобства захочется разделять YAML файлы не только по типу устройств, но и по смыслу переменных, например: _color_, _typography_, _components_.
