## Page Layout

Раскладка для страницы. Какие можно отсюда взять компоненты:
- *PageLayout*: Контейнер для страницы. Занимает height: 100%, width: 100%, поэтому стоит в проекте поставить 100% на html и body.
- *PageContent*, *HeaderContainer*, *FooterContainer*: контейнеры для содержимого страницы, шапки и футера
- *Paranja*: оверлей с затемнением и возможным размытием
- *BottomSheet*: выдвижной снизу блок. Есть крестик для закрытия. Использует леговский Drawer. Можно передать anchor - элемент, к которому будет тянуться Drawer, если он находится слишком далеко. 

### Z-Слои

Соглашения по z-уровням:


| Уровень | Компоненты |
| ------- | ---------- |
|   1   | *PageLayout* |
|  2-99 | всё что находится непосредственно на странице, под паранджой |
|  100  | *Paranja* | 
|  1400 | *BotttomSheet* |
| <1400 | Всё что над paranja и pullout |

### Bottom Sheet Managed
Компонент BottomSheetManaged может использоваться на любом уровне приложения, он передаёт children в корневой компонент для отрисовки. Для его функционирования достаточно использовать внутри PageLayout. 
Хелпер `anchorProps` возвращаen пропсы, которые нужно повесить на кнопку, к которой должен тянуться BottomSheet
