# Обработчик для удаления комментариев

Умеет удалять комментарии в JS и CSS файлах, работает с однострочными и блочными комментариями:

```
// comment

/* comment */

/**
 * comment
 */
```

## Пример

Использование `decomment` сводится к минимальной настройке обработчика:

```javascript
exports.entry = [
  'build/ym/*.js',
  'build/ym/*.css'
]

exports.decomment = {
  text: /\.css$/,
  override: true
}
```

# Параметры настройки обработчика

- entry – прямой путь или glob-паттерн, указывающий на необходимые файлы
- dest – **(опционально)** путь до конечной директории, куда будут скопированы обработанные файлы. Этот параметр будет проигнорирован, если параметр `decomment.override` установлен в значение `true`
- decomment – блок настроек обработчика
  - text – регулярное выражение, которое проверяет путь до файлы. при совпадении decomment переходит в текстовый режим (это нужно для удаления комментариев в файлах, отличных от JS)
  - override – при установке в значение `true` будет перезаписан оригинальный файл, в противном случае необходимо указать настройку `dest`, куда будет записан обработанный файл
