Блок для проксирования методов с сервера в клиент. Нужен для всяких api-блоков.

На примере блока `i-metrika-api`. В файле i-metrika-api.priv.js описываем все нужные методы. В файле i-metrika-api.common.js пишем

    BEM.decl('i-metrika-api', null, BEM.blocks['i-proxy'].initProxy('i-metrika-api', [
        'method1',
        'method2*'
    ]));

В массиве пишем список методов, которые нужно проксировать в браузер (имена методов), после чего эти методы будут доступны для вызова из браузера. Все методы должны возвращать promise и принимать/возвращать json-данные, т.е. никаких ссылок.

Если после имени метода стоит `*`, то вызовы этого метода будут кешироваться в LRU-кеше. Методы для получения данных имеет смысл делать кешируемыми, чтобы лишний раз запросы на сервер не делать.

В i-metrika-api.deps.js блок i-proxy должен быть описан в must-секции.

!!!Важно.!!! Если в браузере блок декларируется как BEM.DOM.decl, то у расширенного через `i-proxy` блока возникают проблемы. В таких случаях получается, что на клиенте нужно расширять BEM.DOM-блок, а на сервере -- BEM-блок. Поэтому делаем примерно следующее

    (function() {
        var B = BEM.DOM || BEM;
        B.decl('b-counters', null, BEM.blocks['i-proxy'].initProxy('b-counters', [
            'load'
        ]));
    })();

!!!Важно.!!! Все проксируемые методы должны следить за доступами, чтобы нельзя было из консоли получить то, что нельзя вызывать.
