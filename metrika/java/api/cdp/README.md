# CDP
**CDP** - Customer Data Platform, подсистема продукта Яндекс.Метрика, позволяющая пользователю загружать информацию о своих клиента из CRM или напрямую через API, и предоставляющая инструменты для работы и анализа загруженной информации.

## Глоссарий
1. Клиент - центральная сущность CDP. Клиент некоторого бизнеса, загруженный в CDP.
2. Идентификатор клиента - уникальный идентификатор клиента из внешней системы учёта клиентов некоторого бизнеса (например, CRM). В общем случае некоторая произвольная строка
3. `cdp_uid` - внутренний идентификатор клиента в системе CDP. Вычисляется как некоторый hash от Идентификатора клиента
4. Заказ - "обновляемое событие" отражающее покупку того или иного товара и приобретение услуги для некоторого бизнеса
5. Идентификатор заказ - уникальный идентификатор заказа из внешней системы учёта клиентов некоторого бизнеса (например, CRM). В общем случае некоторая произвольная строка
6. `order_id` - внутренний идентификатор заказа в системе CDP. Вычисляется как некоторый hash от Идентификатора заказа
7. Продукты - набор товаров или услуг некоторого бизнеса. Полный перечень объектов, обладающих идентификаторами, названиями и прочими полями.
8. Список -  множество элементов. Может быть всё что угодно: набор возможных марок машин в автодилере, цвета красок для окраски волос и т.д. Как и у продуктов каждые элемент списка.
Как и продуктов, каждый элемент списка обладает идентификатором, названием и прочими полями

## Кастомизация схемы
В CDP существует возможность кастомизировать схему существующих в системе сущностей. Реализованна эта возможность с помощью создания пользовательских атрибуты у следующих сущностей:
1. Клиенты
2. Заказы
3. Продукты
4. Список
