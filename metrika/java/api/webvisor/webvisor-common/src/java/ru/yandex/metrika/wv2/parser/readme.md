###Основное
####wv_type
0 - события вебвизора1

1 - страницы вебвизора1

2 - события вебвизора2 в виде json

3 - события вебвизора2 в виде protobuf, схема в директории proto

###Вебвизор1
Вебвизор1 обрабатывается отдельно, для него отделаяется поток исполнения по wv_type, поэтому далее он рассмотрен не будет.

###Вебвизор2

####Поддерживаемые типы 
**event** - события от действий пользователя. маленькие

**mutation** - мутации страницы - могут быть как маленькие, так и большие

**page** - слепок страницы, обычно большой

**chunk** - кусок сериализованного в некотором виде page или mutation (чтобы определить что это, нужно сначала собрать все чанки воедино).
Гарантируется что event едут только целиком и не могут попасть в chunk.

####Парсинг 

Для Вебвизора2 (wv_type >= 2) существует словарь wv_type -> EventParser<Type>.

Так как в плеер нужно отдавать в виде json, в EventParser должен быть определен метод toExternal

Для каждого EventParser существует свой индекс parserIndex.

Каждая версия сериализации в YT - serialization = 1000*parserIndex + serializationIndex
