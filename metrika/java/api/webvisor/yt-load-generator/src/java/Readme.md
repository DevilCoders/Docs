Приложение ходит по заранее заданному файлу в dataGenerator->path.

На каждой итерации ключ модифицируется чтобы шардировать равномерно.

Пытаемся вгружать в YT с максимальной скоростью, приблизительно так же как в visor3d.

Файл с данными удобно получать выгрузкой из прод или тестовой таблицы кусочками по регионам ключа шардирования: 
```
yt select-rows "* from [//home/metrika/wv/webvisor2_events] where userIdHash > -9149585060559937602 and userIdHash <= -9131138316486228050" --format=json > tablet_data.txt
```

Можно регулировать количество тредов, но пока это не показало эффективности.
