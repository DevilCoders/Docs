## Внешние данные (External Data)

Внешние данные позволяют использовать в конфигах инфомацию, поставщиком которой не является bishop.
К таким типам данным сейчас относятся:
 - списки хостов из mtapi (conductor)
 - списки версий секретов (yav)
 - списки кластеров yandex cloud

Доступ к данным осуществляется из шаблонов по списку строго заданных имён (эти имена не могут быть использованы в качестве имён переменных окружений)

Данные обновляются раз в N времени или по запросу пользователя через UI. При обновлении данных система проверяет, были ли изменены данные, и если были - проверяет, какие конфигурации были затронуты и создаёт новые версии конфигов и запросов на активацию, если это необходимо.

При удалении части данных конфигурации, которые их использовали, будут помечены, как "сломанные" и будут ожидать действий пользователя, направленных на их "починку".

Концептуально внешние данные помогают отслеживать изменения конфигурации при изменении состава кластеров (добавление/удаление хостов в них) или при изменении версии секретов.
