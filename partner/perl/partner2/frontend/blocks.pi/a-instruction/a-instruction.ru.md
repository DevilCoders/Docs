# Блок a-instruction

Данный блок формирует полноценную страницу с интерактивной инструкцией, правда без шапки и футера.

## Зависимости

* Модуль `history.js`. Нормализует `history api` в разных браузерах.
* Библиотека `bem-mvc`. Реализует MVC-паттерн в `i-bem`.

## Модельки

Зависит от следующих моделей:

* `m-instruction`. Существует в единственном экземпляре и является, пожалуй, самой главной моделью, которая описывает состояние страницы.
* `m-checkbox`. Создается для каждого чекбокса в интерактивном окне (оформлено в стиле Windows 8). Хранит состояние чекбокса (мод `_checked_yes`) и значение атрибута `value`. Используются для формирования кода командной строки.

Так же зависит от биндинга `a-control`, который реализует взаимосвязь между контролами (блок `a-control`) и полями моделей.
