Привязка сигналов к графу
===

Тулза на основе [maps/analyzer/libs/graphmatching](../../analyzer/libs/graphmatching) для привязки сигналов к графу.<br>
Может работать как аналог [maps/tools/naviguide](../naviguide), но основное предназначение — генерация эталонов для него.<br>

На вход принимает сигналы в yson формате, работает как reducer

При задании флага `--output-segments` также выведет приматченные сегменты, а не только привязанные сигналы (точки).<br>
При генерации эталонов надо передать флаг `--interpolate` — помимо привязанных сигналов будут выданы промежуточные точке на границе сегментов. Нужно это затем, чтобы сегмент из двух соседних точек всегда лежал на эталонном сегменте, это нам нужно при замере качества (т.к. результат будет проецироваться на отрезок между соседними эталонными точками, и эта проекция должна быть на эталоне).<br>
В этом случае тулза может упасть с ошибкой, если интерполировать не удалось. Однако можно указать флаг `--split`, и трек будет нарезан на несколько частей в случае ошибок интерполяции.
