# Quotateka

Система управления квотами на использование API сервисов Геоплатформы.
Предназначена для распределения вычислительных ресурсов между клиентами API и контроля их использования.

### Основные термины

**Провайдер** - сервис Геоплатформы, предоставляющий API для других проектов и команд. Система идентифицирует провайдера по ABC-сервису.

**Ресурс** - подмножество ручек из API провайдера с общей квотой на доступ.  
В простейшем варианте всё API провайдера целиком представлено одним ресурсом.  
В рамках ресурса для каждой ручки задана стоимость, определяющая расход квоты на каждый запрос.    

**Квота** — целое число, задающее лимит на количество запросов к API в секунду.  
Квота привязана к конкретному ресурсу провайдера.  
Единица измерения квоты `qp` - это условные 'попугаи', в которых задана стоимость ручек в ресурсе. 
Квоты разных ресурсов между собой не конвертируются (`qp` разных ресурсов в общем случае никак не связаны) 

**Клиент** — потребитель API одного или нескольких провайдеров.  
Каждому клиенту взаимно-однозначно соответствует ABC-сервис.

**Аккаунт** — сущность для деления квоты на части и передачи этой конфигурации провайдеру.  
Клиент создает аккаунты, ассоциированные с определенным провайдером, и распределяет между ними выданную квоту.
