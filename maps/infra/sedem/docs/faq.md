## Обновление Sedem CLI

Sedem CLI использует механизм раскатки новых версий через конфиг `ya tool`. Чтобы обновить `ya tool`, нужно обновить версию транка.

Проверьте, что ya указывает на корень Аркадии `path/to/arcadia/ya`, а не в `/usr/local/bin/ya`. В последнем случае задайте явный alias вида "`alias ya='path/to/arcadia/ya'`"

## Ошибки доступа к различным api в процессе работы c Sedem CLI

Если в процессе работы c sedem из командной строки вы сталкиваетесь с ошибками доступа к различным API: `permission / access denied`, `http status code 403`, и при этом уверены, что у вас есть все нужные доступы, то убедитесь, что вы НЕ передаете токен седема в явном виде через переменную окружения `SEDEM_TOKEN`.

Если же вы по какой-то причине так делаете, то перейдите [на страницу выдачи oauth-токена для sedem'а](https://oauth.yandex-team.ru/authorize?response_type=token&client_id=f7296c3dcb94487889851beca3078a34) и подтвердите все права, нажав на кнопку "Разрешить".
Если при переходе по ссылке вы сразу получаете OAuth-токен, то все нужные права на ваш токен уже выданы.
Посмотреть список и отозвать токен можно в [вашем аккаунте внутреннего паспорта](https://passport.yandex-team.ru/profile/access).

Дело в том, что по-умолчанию седем получает ваш oauth-токен, обменивая его на ssh-ключ, и при этом автоматически выдаются права на все новые системы, которые мы добавляем в oauth-аккаунт седема. Однако если передавать токен явно через переменную окружения, автоматического расширения прав не происходит.
