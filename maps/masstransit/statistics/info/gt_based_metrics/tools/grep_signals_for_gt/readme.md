### Назначение
Во время записи GT не указывается, на ТС с каким vid он записывался. Цель этой тулы - по сигналам GT за определённый период и по собранным за этот период сигналам определить наиболее подходящий vid.

### Входные и выходные данные
Входные данные содержат две таблицы. Первая - с сигналами за весь день (колонки `clid`, `uuid`, `timestamp`, `lon` и `lat` отсортированные по `clid`, `uuid`, `timestamp`). Вторая - с сигналами gt (колонки `gtid`, `timestamp`, `lon` и `lat` отсортированные по `gtid`, `timestamp`). Результат так же состоит из двух таблиц. Первая - таблица с наилучшим совпадением (колонки `gtid`, `clid`, `uuid`, `score` отсортированные по `gtid`). Вторая - с выделенными сигналами (копирует все сигналы с vid из таблицы с матчингом добавляя соответствующие `gtid`) отсортированная по `gtid`, `clid`, `uuid`, `timestamp`. Притом эта таблица может содержать несколько копий сигналов с разными `gtid` (сколько раз встречается vid в таблице с матчингом, столько раз и копируется).

### Пример запуска
```
./grep_signals_for_gt -s //home/yatransport/mtinfo-metrics/signals/raw/2019-08-12_moscow -g //home/yatransport/mtinfo-metrics/gt/processed/2019-08-12_moscow -m //home/yatransport/mtinfo-metrics/gt/matched_gt/2019-08-12_moscow -o //home/yatransport/mtinfo-metrics/gt/signals/2019-08-12_moscow
```
