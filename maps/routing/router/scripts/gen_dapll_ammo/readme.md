Чтобы обстрелять парковочные альтернативы в роутере,
нам нужно подготовить патроны.

Во-первых, для этого надо сходить в YT и скачать оттуда
координаты центров кластеров, которые станут dap'ами,
и сформировать соответствующие параметры rll и dapll. 

Во-вторых, дополнительная сложность подготовки патронов 
состоит в том, что при запросе альтернатив 
надо передать в теле запроса протобуф с текущий маршрутом
пользователя, а он зависит от версии графа.
Поэтому невозможно подготовить патроны раз и навсегда.

Для этого в лунапарке есть возможность подготовить патроны 
отдельным скриптом, который будет предварительно
ходить на роутер за запросом маршрута
и вытаскивать из ответа описание маршрута, чтобы
сформировать правильное тело для запроса альтернатив.

Соответственно, подготовка патронов состоит из двух частей:
prepare_queries и gen_dapll_ammo.

Сначала надо один раз локально (на разработческой машине 
или т.п.) запустить скрипт prepare_queries, 
который сходит на YT и подготовит запросы с rll и dapll.
Это надо сделать два раза, чтобы подготовить запросы 
без dapll и точно такие же, но с dapll
(это управляется параметром -f у prepare_queries).
Полученные файлы надо положить в каталог query_templates,
обновив тот файлы, которые там лежат сейчас.
Файл templates_without_dapll не содержит dapll 
(соответствует параметру -f 0),
файл templates_with_dapll содержит (-f 1).

После этого надо собрать пакет yandex-maps-router-gen-dapll-ammo,
который включает скрипт gen_dapll_ammo и файлы query_templates/*,
и выкатить этот пакет в testing.

Стрельбы должны быть настроены так, чтобы перед началом стрельбы
для окончательного формирования патронов каждый раз запускался 
скрипт make_ammo с указанием нужного входного файла
из пакета (templates_0 или templates_1).

Если надо будет обновить список запросов,
надо локально перезапустить скрипт prepare_queries,
закоммитить новые файлы в аркадию и пересобрать и перевыкатить пакет
yandex-maps-router-gen-dapll-ammo.

Если надо просто пострелять один раз, то в интерфейсе лунапарка
есть возможность загрузить файл патронов с локальной машины.
Тогда надо локально запустить оба скрипта один за другим,
и сформированный файл патронов уже загрузить в лунапарк.

Пример запуска:

    ./prepare_queries -o ../query_templates/templates_with_dapll
    ./prepare_queries -o ../query_templates/templates_without_dapll -f 0
    ./gen_dapll_ammo -i ../query_templates/templates_with_dapll -o ammo
