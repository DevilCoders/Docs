## renderer-tilemill-serv API

### GET /tiles

Параметры запроса:
- `x`, `y`, `z` - координаты и зум тайла
- `source_uri` - URI тайлсета вида `tilemill://s3_key`
- `format` (опционально для отладки) - формат в котором нужно отдать тайл (`protobuf`/`proto`, `text`, `json`)

Используемые HTTP коды ошибок:
- 200 - OK
- 400 - невалидные параметры запроса
  - формат параметра `source_uri`
  - координаты или зум тайла: например, вне набора зумов, на которых были напилены тайлы
- 404 - тайлсет отсутствует в S3
- 500 - невалидный тайлсет в S3, другие ошибки

### GET /meshes

Параметры запроса:
- `source_uri` - URI директории с mesh-моделями вида `tilemill://s3_key`
- `id` - идентификатор mesh-модели, строка вида `EHH2KNMFZPKVIJPXUJH5GFN64Q2OBI6X`

Используемые HTTP коды ошибок:
- 200 - OK
- 400 - невалидные параметры запроса
  - формат параметра `source_uri`
  - отсутствует параметр `id`
- 404 - mesh-модель отсутствует в S3
- 500 - ошибки запросов к S3, другие ошибки

### GET /indoor/plans

Параметры запроса:
- `source_uri` - URI директории с indoor-планами вида `tilemill://s3_key`
- `id` - идентификатор indoor-плана, строка вида `4DE27F8B93370BEEEF6C1546042B9EC7`

Используемые HTTP коды ошибок:
- 200 - OK
- 400 - невалидные параметры запроса
  - формат параметра `source_uri`
  - отсутствует параметр `id`
- 404 - indoor-план отсутствует в S3
- 500 - ошибки запросов к S3, другие ошибки
