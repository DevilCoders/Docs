# Граф задач

Если представить [задачи](task.md) в виде узлов, а зависимости по входным [ресурсам](resource.md) в виде ребер, то получится граф задач, который исполняет Огород.

Каждый [модуль](module.md) содержит свой граф задач. Графы задач должны быть совместимы по именам ресурсов, т.е. входные ресурсы одного модуля должны быть выходными ресурсами другого модуля, чтобы можно было бы из графов задач отдельных модулей собрать один общий граф задач. На практике сборка общего графа никогда не выполняется.

Граф задач определяет порядок их исполнения. Задачи в графе могут быть исполнены, когда доступны все входные ресурсы, т.е. когда завершены все предшествующие билды, производящие данные ресурсы.
