# Operation Fetcher

Тул `operation_fetcher` периодически запрашивает список операций из YT и записывает информацию о них в Монго-коллекцию `task_operations`.

Когда таска завершает работу, то в Монгу пишется:
* время работы операции или джобы
* алерты, которые сгенерировал YT
* потреблённые вычислительные ресурсы: cpu и ram

Запрашиваются как ванилла, так и MR-операции.

Данные из коллекции `task_operations` используются в ручке сервера `/modules/<module_name>/builds/<int:build_id>/full_info/` для отображения списка операций на странице билда в UI (Perf Analyzer).

Тул работает в виде демона под управлением supervisor в докер-контейнере с шедьюлером Огорода.

Если тул падает, то запущенные операции перестают отображаться в UI. Это не является критической поломкой. Чинить тул ночью и в выходные нет необходимости.
