# Тулы в Огороде

## Тулы, которые находятся в одном докере с шедьюлером

### Тулы, которые работают непрерывно в режиме демона

* `module_monitorings` - проверяет состояние огородных модулей и отправляет события в Juggler.
* `operation_fetcher` - запрашивает статусы дочерних операций в YT и записывает в коллекцию `task_operations` в Монге. Далее эта информация используется в UI на странице билда.
* `perf_normalizer` - перекладывает записи из коллекции `task_log` в коллекции `task_statistics` и `resource_statistics`. Также эта информация используется в UI на странице билда.

### Тулы, которые работают по крону

* `yt_cleanup` - удаляет старые временные файлы из Кипариса.
* `releases_monitoring` - создаёт тикеты в очереди `MAPSLSR`, если свежие данные не были зарелизены.
* `stat_updater` - отправляет в statface информацию о билдах в Огороде. Далее по этой информаци строится дашборд.
* `pin_builds` - запинивает старые билды модуля `renderer_publication` по средам - для возможности быстрого отката на старый релиз данных.
* `module_version_sync` - запускается только в тестовом Огороде - синхронизирует версии модулей со стабильным Огородом.

### Прочие тулы

* `yt_module_executor` - заливается в Кипарис на старте шедьюлера, используется внутри ванилла-операций для контроля работы огородного модуля.
* `cron_monitorings` - используется для мониторинга тулов, которые запускаются по крону.
* `unaccounted_resources` - умеет искать утёкшие ресурсы во внешних хранилищах. Автоматически не запускается. Его нужно запускать вручную, подключившись по ssh.

## Прикладные тулы

* `garden_cli` - позволяет загружать локальные версии огородных модулей в дев-контура в Огороде.
* `garden_cli_launcher` - обёртка над `garden_cli` - приносит свежую версию тула на дев-машину.
* `playground` - набор конфигов и тулов для работы с плейграундами.
* `ymapsdf_pg` - тул для конвертации YT-таблиц в Postgres
* `idm_tool` - тул дя массового отбора или выдачи прав в idm сервисах Огорода

## Прочее

* `mongo_reader` - набор хелпер-функций по анализу данных в Монге.
* `jupyter` - ядро Jupyter, собранное в Аркадии с поддержкой огородных библиотек.
* `translation` - часть модуля `ymapsdf` - набор скриптов для перевода названий между разными языками.
