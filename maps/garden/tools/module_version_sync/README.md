# module-version-sync

## Описание
Тул используется для синхронизации информации о версиях модулей из stable в остальные инстансы Огорода.

## Детали
Модули, собранные на `trunk` на закоммиченном коде, собираются через Sandbox и релизятся через SEDEM/Sandbox.
Данные о таких версиях модулей записываются только в stable монго-базу.
Тул используется для синхронизации данных из stable во все остальные окружения.

В testing тул работает по cron, запускаясь раз в 5 минут.

В плейграундах запуск по cron не включен. Тул автоматически разово запускается во время старта сервера.
Если без перезапуска сервера необходимо синхронизировать системные версии, то нужно запустить следующий скрипт:
```/scripts/run_module_version_sync.sh```.

Тул синхронизирует только версии, зарелиженные в testing и stable. unstable и hotfix версии игнорируются.

Тул синхронизирует только системные версии, игнорируя пользовательские.

Физически копирование осуществляется посредством вызов серверных ручек для регистрации, релиза и удаления верси модуля.
