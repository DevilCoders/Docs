В `source_module.py` содержится набор функций для написания простого
source-модуля, который наблюдает за некоторым путём в YT. По этому пути
появляются таблицы с именами в формате `YYYY-mm-dd`. Каждая из таких таблиц
содержит некоторые данные за какой-то день. По мере того как таблицы появляются,
для каждой из них запускается билд этого модуля, содержащий один выходной ресурс
`YtSourcePathResource`. Дата из имени таблицы записывается в свойства
"shipping_date" ресурса и билда, её видно в UI.

Предполагается, что потребители этих таблиц будут использовать сразу несколько
из них. При этом нужно учитывать два фундаментальных ограничения Огорода:

1. Все входные ресурсы некоторого билда некоторого модуля должны иметь разные
имена. Нельзя подать на вход билду два ресурса с одним и тем же именем
"some_source_table"
2. Множество имён входных ресурсов должно быть фиксированным. Нельзя постоянно
генерировать всё новые и новые имена ресурсов для этих таблиц. Функция
`fill_graph` ниоткуда не может получить информацию о том, какие имена ресурсов
вообще используются, поэтому должна использовать набор имён, известных априори.

Чтобы с этим справиться, этот модуль циклически использует имена
`source_<index>`, где `<index>` варьируется от 0 до 30 (зашито в коде). Для
наглядности, этот индекс пишется в свойства `release_name` ресурса и билда, его
тоже видно в UI. Отсюда возникает неочевидное ограничение: на вход модулю нельзя
подать две таблицы, которые отличаются на количество дней, кратное 30. Константу
при желании можно значительно увеличить.
