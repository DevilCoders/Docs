# Библиотеки, которые используются сервером, шедьюлером и тулами Огорода.

## Общие

* `common` - общие классы, исключения и вспомогательные функции, а также классы по работе с контурами.
* `application` - глобальное состояние сервера и шедьюлера, а также вспомогательные функции для Flask.

## Работа с Монгой

* `module` - доступ к версиям модулей в Монге, также умеет запускать модули локально в продпроцессах.
* `build` - доступ к текущим и историческим билдам в Монге, также проверяет разные ограничения на запуск билда и удаляет старые билды.
* `resource_storage` - доступ к огородным ресурсам в Монге.
* `task` - доступ к завершившимся и выполняющимся таскам в Монге.
* `log_storage` - доступ к событиями и логам модулей в Монге.
* `autostart` - управляет автостартерами и хранит записи об отложенных автостартах в Монге.
* `acl_manager` - доступ к правам доступа в Монге.
* `migration` - механизм изменения структуры базы данных.

## Планирование и запуск тасок в YT

* `graph` - содержит графы тасок всех билдов и решает, какие таски можно уже запускать.
* `yt_task_handler` - запускает таски в YT и следит за их завершением.

## Вспомогательные библиотеки

* `config` - функция чтения конфига с диска.
* `datatesting` - содержит список модулей, которые заливают данные в тестовый экстатик. Временная библиотека.
* `monitorings` - ряд общих функций для анализа работы модулей. Используется в мониторингах и при создании LSR-тикетов.
* `startrek` - ряд удобных функций для создания тикетов в Стартреке.
* `test_utils` - общие классы и функции для тестов сервера и шедьюлера.
