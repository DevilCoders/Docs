# Codestyle

Основой codestyle для нашего проекта служит [общекарточный codestyle](https://github.yandex-team.ru/islets/codestyle), который в свою очередь накладывает дополнительные ограничения на [общеяндексовый](https://github.com/yandex/codestyle).

Дальнейшие ограничения на стиль кода описаны в конфигурационных файлах eslint и tslint.
Проверки этих правил запускают git-хуком commit.

Приняты дополнительные ограничения (не проверяемых на данный момент линтерами) на стиль написания составных слов в особенных случаях:

-   все проектные куки и ключи localStorage именуются в snake_case;
-   все ключи в [танкере](tanker.yandex-team.ru/keysets/?project-id=loris) (аргументы глобальной функции i18n) именуются в kebab-case;
-   все новые ключи состояния для аналитики (в `nodeState`, `subtreeState` и т.п.) именуются в camelCase;
-   деструктуризация props / state в react-компонентах запрещена.
