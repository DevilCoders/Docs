Библиотека для автоматического разрешения координатных конфликтов
===

### Описание
Бибилиотека реализует алгоритм автоматического "растаскивания" точек для разрешения координатных 
конфликтов между ними, а также с заданным набором неподвижных точек. Для каждой точки можно передать 
индивидуальное расстояние конфликта, в пределах которого не должно оказаться других (подвижных или 
неподвижных) точек.

Кроме того, алгоритм получает на вход набор полигонов, в пределах которых должны быть расставлены 
точки. Точки, которые лежали вне объединения полигонов, будут перемещены внутрь, при условии, что 
внутри полигонов есть место для их бескнофликтого размещения. В противном случае, координаты точки 
не изменяются.

Алогоритм стремится минимизировать количество точек, которые необходимо подвинуть для разрешения 
конфликтов, а также расстояния, на которые эти точки будут подвинуты.

### Использование алгоритма
Функция [solvePointConflicts](https://a.yandex-team.ru/arc/trunk/arcadia/maps/poi/libs/conflict_solver/include/conflict_solver.h?rev=r8496360#L34)
принимает следующие параметры:
- `mercPointsToMove` координаты точек, для которых необходимо разрешить конфликты. Каждая точка
передаётся вместе с индивидуальным расстояниям конфликта.
- `mercFixedPoints` координаты неподвижных точек, вместе с индивидуальными расстояниями конфликтов.
- `mercPolygons` полигоны, внутри которых должны быть расставлены подвижные точки.
- `mercPolygonsBoundaryConflictDistance` отступ от границ объединения полигонов. Может быть как
положительным, так и отрицательным.

Функция возвращает новые координаты подвижных точек, в том же порядке, в котором они были переданы
на вход.

### Подробное описание алгоритма
Полигоны из `mercPolygons` объединяются в один мультиполигон алгоритмом
[cascaded union](https://a.yandex-team.ru/arc/trunk/arcadia/maps/libs/geolib/include/yandex/maps/geolib3/algorithm.h?rev=r8317603#L69),
от границ которого берётся отступ равный `mercPolygonsBoundaryConflictDistance`. Затем, для каждой
неподвижной точки берётся круг с центром в этой точке и радиусом равным расстоянию конфликта для этой
точки, и вырезается из имеющегося мультиполигона. Получившийся мультиполигон с вырезанными кругами
назовём "сырным". Далее, для каждой подвиждой точки проверим, лежит ли она в сырном мультиполигоне:
если лежит, то оставим её на месте (положим новые координаты равными старым), а если нет, то найдём
ближайшую к ней точку из сырного мультиполигона и возьмём её координаты в качестве новых координат
исходной точки. После этого, вырежем из сырного мультиполигона круг с радиусом равным расстоянию
конфликта вокруг новой координаты точки. Сделаем так для всех подвижных точек, и получим их новые
координаты. Если в процессе работы алгоритма сырный мультиполигон оказался пустым, то алгоритм
останавливается, и оставшиеся подвижные точки остаются на своих местах.

В процессе работы алгоритма подвижные точки сортируются в порядке количества конфликтов – вначале
обрабатываются точки, имеющие меньшее количество конфликтов. Это позволяет сначала решить локальные
конфликты, а затем переходить к растаскиванию больших скоплений точек.
