# Дополнительные параметры для формирования сегментов

Преимущественно, сегменты собираются по уже подготовленным
таблицам и большую часть информации можно получить из них.
Но существуют дополнительные критерии для отбора данных для сегментов.
В этой директории собраны все дополнительные скрипты.
Перед добавлением нового скрипта, следует проверить
созможность добавления нового ограничения в существующие.
Например в **emails_with_adv** определяются email-ы с открутками
за последние 360 дней, но туда легко добавить и email-ы с открутками на вчерашний день.

### [Nirvana](https://nirvana.yandex-team.ru/flow/653b1482-4974-4be1-8d71-f10e346697be/d83d3505-a8c7-4846-93d5-031cc31f97b9/graph)

### Дополнительные данные
* **emails_with_adv** - список email-ов с откруками (сейчас только за последние 360 дней).
* **sprav_permalinks** - список пермалинков из Справочника с основными параметрами.
* **geoadv_business_snapshots** - аналогично **sprav_permalinks**, но для бизнес-снапшотов.
* **sprav_and_biz_permalinks** - объединённые данные из **sprav_permalinks** и **geoadv_business_snapshots**.
Особенностью является то, что в поле `permalink` записываются уникальные значения как permalink-ов
из Справочника, так и идентификаторы бизнес снапшотов. Существует поле `permalink_type` по которому
можно определить, к какому типу относится permalink. `sprav_permalink` - указывает на пермалинк из
Справочника - актуально только для бизнес снапшотов. С помощью этой таблицы проще матчится
с логами Бизнеса, так как там используется аналогичный принцип матчинга РК к permalink-у.
