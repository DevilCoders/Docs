## Фильтрация пушей

Не хотим рассылать пуши, если пользователь явно сказал, что ему этот тип пушей не интересен (признак `useless` в логах [SUP](https://yt.yandex-team.ru/hahn/navigation?path=//home/search-functionality/sup/push_stats)). И не хотим регулярно отсылать одному и тому же пользователю один и тот же пуш. Поэтому фильтруем пуши, которые пользователь открыл.

Логика в коде проста:
1. берём логи SUP-а
2. достаём из них записи с нужным нам `push_type`
3. фильтруем из них открытые и `useless` пуши
4. join-им открытые пуши по `uid` с отправленными ранее пушами
5. оставляем для рассылки только те пуши, которые не `useless` и которые не были открыты
