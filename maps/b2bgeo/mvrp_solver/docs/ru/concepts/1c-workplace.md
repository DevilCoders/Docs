# Описание рабочего места логиста

Кнопки рабочего места логиста описаны ниже.

**Данные**

Поле |Описание
:--- |:---
Заполнить список | Копирует сведения из стандартных документов 1С и заполняет форму **Данные**. Состав документов, из которых берется информация, зависит от используемой конфигурации. Предварительно нужно заполнить поля **Склад** и **Период**: по ним осуществляется фильтрация.
Геокодировать адреса | Отправляет на геокодирование адреса из строк заказов, где не заполнены поля **Координаты (Широта и Долгота)**. Заполняет эти поля координатами из сервиса геокодирования. Подсвечивает адреса, для которых координаты определены не точно:<ul><li>оранжевым, если не найден нужный дом (но найден дом с номером, близким к запрошенному, или найдена только улица);</li><li>красным (если улица не найдена и возвращаются только координаты центра города).</li></ul>
Отправить запрос | Отравляет [запрос на маршрутизацию по API]( https://yandex.ru/routing/doc/vrp/ref/MultipleVehiclesRouting/addMVRPTask.html), используя: <ul><li>сведения о заказах из формы **Данные**;</li><li>данные о выбранных машинах и складе; информацию из **Общих настроек**</li></ul>.
Выгрузить заказы в Excel | Выгружает список заказов в Excel в [формате](https://yandex.ru/routing/doc/vrp/concepts/excel-template-filling_index-page.html), пригодном для дальнейшей загрузки в веб-интерфейсе Яндекс.Маршрутизации.
Открыть настройки | Открывает форму **Общих настроек**.

**Результаты планирования**

Поле |Описание
:--- |:---
Перейти в интерфейс Яндекс.Маршрутизации | Открывает веб-интерфейс Яндекс. Маршрутизации по ссылке `https://yandex.ru/courier/companies/<company_id>/depots/all/mvrp/<task_id>`, где <ul><li>`<company_id>` — идентификатор вашей компании (параметр **ID Компании в Яндекс.Маршрутизация** из **Общих настроек**);</li><li> `<task_id>` — идентификатор задачи планирования.</li></ul>
Отправить мониторинг | Отправляет данные из формы в сервис мониторинга, используя API для создания/обновления: [складов](https://yandex.ru/routing/doc/delivery/ref/depots/api_v1_companies_company_id_depots-batch_post.html), [курьеров](https://yandex.ru/routing/doc/delivery/ref/couriers/api_v1_companies_company_id_couriers-batch_post.html), [маршрутов](https://yandex.ru/routing/doc/delivery/ref/routes/api_v1_companies_company_id_routes-batch_post.html), [заказов](https://yandex.ru/routing/doc/delivery/ref/orders/api_v1_companies_company_id_orders-batch_post.html).
Получить отчет о доставке | Получает информацию о фактических доставленных заказах через [API](https://yandex.ru/routing/doc/delivery/ref/courier_report/api_v1_companies_company_id_courier-quality_get.html). Обновляет для каждого заказа поля формы **Cтатус доставки** и **Фактическое время доставки**.
