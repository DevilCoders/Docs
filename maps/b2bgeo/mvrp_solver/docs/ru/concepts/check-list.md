# Чек-лист внедрения
Для упрощения контроля за процессом внедрения используйте следующий чек-лист:

## Ознакомление с продуктом

1. Пройдена [регистрация в продукте](quickstart.md).
1. Шаблон Excel заполнен реальными данными по заказам, машинам и складам за выбранный день.
1. Получен первый результат планирования на собственных данных.

## Тестирование продукта

1. Изучены возможности и настройки продукта согласно [документации](https://yandex.ru/routing/doc/vrp/concepts/properties-general.html).
1. Проведен ряд итераций планирования с использованием шаблона Excel с различными исходными данными и настройками и получен хороший результат маршрутизации с использованием Excel. Принято решение о пилотном проезде по маршрутам и интеграции.

## Проезд по маршрутам

7. Выполнен тестовый проезд (возможно, с использованием [Мониторинга](https://yandex.ru/routing/doc/delivery/concepts/about.html)) по маршрутам, которые были построены сервисом.
1. Проанализирован план-факт, собраны выявленные проблемы и несоответствия по итогам тестового проезда.

## Интеграция

9. Выбрана схема интеграции (Excel, API c редактированием в UI, API без редактирования в UI).
1. Проработан вопрос с получением координат:
   - Запланировано подключение Геокодера или разовое заполнение координат.
   - Обеспечен корректный ввод информации по адресам (без лишних примечаний).
1. Обеспечены ресурсы для интеграции (собственные разработчики или подрядчик).
1. В рамках плана разработки и внедрения определена последовательность:
   - выполнения доработок (деление на минимально необходимые и те, которые можно выполнить после запуска в опытную эксплуатацию);
   - подключения площадок (если их несколько);
   - интеграции с продуктами (Геокодер, Планирование, Мониторинг).
1. Реализована интеграция с API. Интеграция проверена на тестовых данных.
1. Организовано заполнение необходимых справочных данных (временные окна по точкам, транспортные средства, совместимость машин и заказов и т.&nbsp;п.).

## Опытная эксплуатация

15. Начаты запуски планирования (возможно, параллельные) на реальных данных.
1. Проведена оценка результатов маршрутизации с участием логистов.
1. При необходимости скорректированы настройки и исходные данные для построения маршрутов.
1. Начаты регулярные проезды по маршрутам. Организован периодический сбор и анализ обратной связи от логистов и водителей.
1. Принято решение о переходе в на постоянное использование сервиса.

## Промышленная эксплуатация (тиражирование)

21. Сформулирован план дальнейшего тиражирования.
