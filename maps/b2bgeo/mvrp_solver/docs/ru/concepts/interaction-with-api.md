# Работа через API с редактированием решения через интерфейс

Мы рекомендуем следующий сценарий работы:

1. Решение отправляется по API из системы-источника;
2. Бизнес-пользователь заходит в интерфейс [Яндекс Маршутизации](https://yandex.ru/courier), затем [просматривает и/или редактирует решение](interface-opportunities.md#redaktirovanie-resheniia);
3. Если решение редактировалось, то пользователь [экспортирует полученное решение](https://yandex.ru/routing/doc/vrp/concepts/interface-result.html#eksport-resheniia) во внутреннюю систему Маршрутизатора. Связь отредактированного решения с исходным будет сохранена автоматически. Для получения скорректированного решения в дальнейшем можно использовать запрос [/children](https://yandex.ru/routing/doc/vrp/ref/ChildTasks/getChildTasks.html) с ID оригинальной задачи. Если решение не редактировалось то пользователь на своей стороне подтверждает, что его устраивает полученное решение;
4. Итоговое решение скачивается обычным образом через API.

{% note %}

Итоговое решение после ручных корректировок повторно оптимизируется, поэтому с ним нужно работать так же, как и с остальными задачи для API построения оптимальных маршрутов, то есть дождаться завершения этой задачи.

{% endnote %}
