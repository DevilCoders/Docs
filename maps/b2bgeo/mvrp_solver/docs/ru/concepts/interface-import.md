# Окно импорта

Здесь на различных вкладках меню указываются настройки и исходные данные задачи.

Шаги работы в окне импорта:

1. Загрузка исходных данных. Если вы перешли в окно импорта, не выбирая файл на предыдущем шаге, на вкладках меню будут отображены некоторые стандартные значения. Эту информацию можно редактировать непосредственно в этом окне.
   Также вы можете импортировать данные, нажав кнопку **Загрузить файл** или перетащив документ в окно.

1. Исправление ошибок и ознакомление с предупреждениями. Рядом с названиями вкладок отображаются значки, которые показывают статус загруженной информации:

   ![Данные верны](https://courier.yandex.ru/vrs-doc/ok.png "Данные верны") — данные верны;

   ![Есть ошибки](https://courier.yandex.ru/vrs-doc/error.png "Есть ошибки") — есть ошибки, которые необходимо исправить до начала планирования;

   ![Есть предупреждения](https://courier.yandex.ru/vrs-doc/warn.png "Есть предупреждения") — есть предупреждения, но планировать можно.

1. Определение [настроек](#настройки) оптимизации.

1. Запуск маршрутизации. Кнопка **Планировать** активна, если в исходных данных нет ошибок.

Когда задача будет решена, откроется [результат маршрутизации](interface-result.md). Его можно просмотреть и отредактировать.

Ниже подробнее описаны вкладки окна импорта.


## Настройки

Здесь нужно определить настройки задачи:

- **Дата маршрута** — чтобы прогноз пробок был актуальным, важно указывать реальную дату доставки заказов.

- **Часовой пояс** — зона, для которой заданы все временные окна.

- **Метод передвижения** — настройка по умолчанию для всех курьеров, у которых явно не задан [свой метод](https://yandex.ru/routing/doc/vrp/concepts/properties-of-vehicles.html#rezhim-marshrutizatsii).

- **Настройка качества решения** — значение `Быстро` подходит для оперативной проверки результата на наличие ошибок, `Оптимально` — для полноценной маршрутизации.

В блоке **Примеры файлов** можно скачать демонстрационные книги Excel (см. также раздел [Пример планирования](https://yandex.ru/routing/doc/vrp/concepts/example.html)).

## Карта

На вкладке представлены список адресов заказов и карта. В зависимости от наличия координат в исходных данных адрес располагается в одной из групп списка: **С координатами** или **Без координат**.

Для запуска планирования все адреса должны быть геокодированы: им должны быть сопоставлены широта и долгота. Это можно сделать автоматически или вручную.

### Автоматическое геокодирование

Запускается нажатием на кнопку **Найти по адресу**.

Если нужно ограничить границы поиска адресов при геокодировании, включите опцию **Искать в видимой области карты**. Это помогает правильно определить координаты адресов, у которых, например, не указан город. Также с помощью этой опции можно разбить процесс геокодирования на несколько шагов, меняя область видимости на разных итерациях.

### Ручное геокодирование

Чтобы назначить координаты точке вручную:

1. Нажмите на адрес в списке рядом с картой.
1. В появившемся диалоге отредактируйте адрес и выберите подходящий вариант в выпадающем списке.

![](https://courier.yandex.ru/vrs-doc/manual-geocoding-dialog.png "")

1. Нажмите кнопку **Дальше** для перехода к следующему адресу.

Уточнить положение точки, отображаемой на карте, можно также с помощью перетаскивания ее флага.

### Результат геокодирования

Точно позиционированные адреса появятся в группе **С координатами** и будут отмечены зеленым цветом на карте.

Если местоположение точки было определено примерно, она будет показана в группе **Приблизительно** и отмечена на карте желтым цветом. Такие адреса нужно проверить на предмет правильности найденных координат. Чтобы уточнить место на карте:

   1. Нажмите на адрес в списке.
   1. Измените расположение флага на карте. Используйте переключатель слоев, если нужно лучше разглядеть местность.
   1. Флаг и адрес в списке будут отмечены зелеными цветом.

Чтобы сохранить результат геокодирования в файл, нажмите **Скачать Excel**.

## Другие вкладки

На следующих вкладках окна импорта приведены таблицы с остальными исходными данными задачи планирования:

- **Заказы** — сведения о [заказах](https://yandex.ru/routing/doc/vrp/concepts/excel-fill-guide.html#orders);
- **Машины** — информация о [курьерах](https://yandex.ru/routing/doc/vrp/concepts/excel-fill-guide.html#vehicles);
- **Склад** — данные о [складе](https://yandex.ru/routing/doc/vrp/concepts/excel-fill-guide.html#depot);
- **Несовместимые заказы** — типы заказов, которые [не должны одновременно находиться](https://yandex.ru/routing/doc/vrp/concepts/excel-fill-guide.html#incompatible-order-types) у курьера;
- **Группы заказов** — данные о заказах, которые нужно [доставить одним курьером](https://yandex.ru/routing/doc/vrp/concepts/excel-fill-guide.html#location-groups);
- **Дополнительные настройки** — перечень [опций маршрутизации](https://yandex.ru/routing/doc/vrp/concepts/excel-fill-guide.html#options);
- **Маршруты** — сведения о [предопределенных маршрутах](https://yandex.ru/routing/doc/vrp/concepts/excel-fill-guide.html#options).

Видимость некоторых вкладок в меню окна импорта зависит от того, какие листы были представлены в загруженном файле (подробнее см. раздел [Описание файла Excel](https://yandex.ru/routing/doc/vrp/concepts/excel-fill-guide.html)).

### Работа с таблицами

При работе с таблицами на этих вкладках вы можете:

- копировать данные из Excel;
- корректировать значения ячеек после двойного нажатия левой кнопкой мыши;
- распространить содержмое одной ячейки в диапазоне, растягивая рамку выделения в пределах столбца;
- удалять, вставлять, дублировать строки с помощью контекстного меню, доступного по правой кнопке мыши на строке;
- сортировать данные по столбцу, вызвав контекстное меню на любой его ячейке или нажав на заголовок столбца.

Если система обнаружит ошибки в исходных данных, информация об этом появится в перечне замечаний рядом с таблицей. Нажмите на сообщение в списке, чтобы подсветить проблемную ячейку.
