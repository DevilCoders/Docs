# Ключ API

## Общие сведения
 Для аутентификации в API планирования маршрутов используется ключ доступа. Его  нужно передавать в запросе c HTTP-методом POST как значение параметра `apikey`.

Важно иметь в виду:

- По ключу ведется тарификация, поэтому его должны знать только доверенные лица.

- На ключ накладывается лимит по количеству запросов в сутки (в целях обеспечения безопасности сервиса). Как правило, для нормальной работы выделенного количества запросов достаточно. Если в связи с потребностями бизнеса вам нужно увеличить лимит, обратитесь в техническую поддержку по электронной почте [routing-support@yandex-team.ru](mailto:routing-support@yandex-team.ru) или через форму [Обратной связи](https://yandex.ru/routing/doc/vrp/concepts/feedback.html).

Получить ключ можно двумя способами:

- Через [Кабинет Разработчика](https://developer.tech.yandex.ru/) — специальный портал, который позволяет генерировать ключи для API различных сервисов Яндекса. Полученный таким образом ключ будет иметь ограничения по сроку действия и некоторым другим характеристикам.

- В веб-интерфейсе сервиса в разделе **Компания**. Если вы уже самостоятельно зарегистрировались в Яндекс.Маршрутизации, ключ был создан автоматически и получать его в Кабинете Разработчика не нужно. После оплаты сервиса срок действия ключа продлевается.

## Получение ключа в Кабинете Разработчика

1. Авторизуйтесь на Яндексе c логином, который будет использоваться для работы с API.

1. Перейдите в [Кабинет Разработчика](https://developer.tech.yandex.ru/) и нажмите  **Подключить API**.

1. Выберите **Маршрутизация: Планирование и Мониторинг** и нажмите **Продолжить**.

1. Заполните анкету и нажмите **Продолжить**.

1. В открывшемся окне нажмите **Перейти к API**.

1. Откроется страница ваших ключей для API Яндекс.Маршрутизации. Созданный ключ был активирован автоматически, его можно использовать.

Дополнительную информацию о работе с API-ключами в Кабинете Разработчика можно найти в [документации сервиса](https://yandex.ru/dev/developer-help/doc/index-docpage/).
