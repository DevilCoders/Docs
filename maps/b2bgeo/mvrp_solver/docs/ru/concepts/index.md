# Введение

[Яндекс.Маршрутизация](https://yandex.ru/routing/) состоит из 2-х сервисов, которые могут использоваться либо вместе, либо отдельно и независимо друг от друга:

- сервис планирования маршрутов (ниже документация к этому сервису);
- сервис выполнения заказов ([документация к сервису](https://yandex.ru/routing/doc/delivery/concepts/about.html)).

Сервис планирования маршрутов позволяет строить оптимальные маршруты либо через web-интерфейс с помощью шаблона Excel, либо через API с помощью json-запроса.

## Как зарегистрироваться?

Зайдите на [страницу сервиса](https://yandex.ru/courier/).

Для регистрации рекомендуется завести отдельный аккаунт на [Яндекс.Паспорте](https://passport.yandex.ru/) (желательно не использовать личный аккаунт, возможно потом потребуется передать логин\пароль другим сотрудникам вашей компании). Под  данным логином в дальнейшем можно будет давать доступ другим аккаунтам

При регистрации автоматически создается ключ доступа сервису распределения заказов, который привязывается к указанному логину.

## Как начать?

Воспользуйтесь разделом [Быстрый старт](https://yandex.ru/routing/doc/vrp/concepts/quickstart.html).

После регистрации в продукте, в интерфейсе запуска планирования можно скачать тестовый пример Excel (примеры также доступны здесь в разделе [Пример шаблонов Excel](https://yandex.ru/routing/doc/vrp/concepts/example.html)), на основе которого вы сможете заполнить шаблон своими данными. Более подробно заполнение файла описано в разделе [Инструкция по заполнению Excel](excel-fill-guide.md).

Далее вы можете [запустить планирование в web-интерфейсе](https://yandex.ru/routing/doc/vrp/concepts/interface-opportunities.html).

Более подробно ознакомиться с функциональными возможностями можно в [справочнике атрибутов и опций](https://yandex.ru/routing/doc/vrp/concepts/reference.html). Дополнительную информацию можно узнать из [видео-инструкций](https://yandex.ru/routing/video).

## Как интегрироваться через API?

Для работы через API вам необходим ключ. При регистрации в [интерфейсе](https://yandex.ru/courier/) он создается автоматически и доступен в настройках компании.

Спецификация запроса и ответа API доступна в разделе с [описанием запроса на планирование](https://yandex.ru/routing/doc/vrp/ref/MultipleVehiclesRouting/addMVRPTask.html) и разделе с [описанием ответа API](https://yandex.ru/routing/doc/vrp/ref/MultipleVehiclesRouting/getMVRPTaskResult.html).

Много примеров запросов и ответов API доступно в [справочнике атрибутов и опций](https://yandex.ru/routing/doc/vrp/concepts/reference.html). Под большинством разделов, описывающих ту или иную функциональность, предоставлен пример планирования с использованием описанных возможностей.

Если ваша система 1С (версии не ниже 8) – для интеграции можете воспользоваться готовыми модулями для [базовой версии](https://yandex.ru/routing/doc/vrp/concepts/1c-guide-of.html) и [версии на управляемых формах](https://yandex.ru/routing/doc/vrp/concepts/1c-guide-uf.html).

## Как это работает?

Для оптимизации маршрутов необходимо передавать данные об машинах\курьерах, заказах и складе (в Excel или в запросе к API). Данные по расстояниям с учетом графа дорог и времени проезда с учетом пробок рассчитываются на стороне Яндекс.Маршрутизации.

Сервис выполняет оптимизацию маршрутов в среднем за 15 минут (на небольших задачах – меньше минуты). В ответе будет распределение заказов на машины, последовательность посещения заказов, плановое время прибытия на каждый заказ.

О том, как именно работает алгоритм планирования, описано в разделе [Алгоритм маршрутизации](https://yandex.ru/routing/doc/vrp/concepts/routing-algorithm.html).
