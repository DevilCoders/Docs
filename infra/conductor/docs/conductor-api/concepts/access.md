# Доступ к API

Доступ к API Кондуктора имеют пользователи с ролью не ниже [Audience](https://doc.yandex-team.ru/Debian/conductor-guide/concepts/access.html#access__audience). API поддерживает два механизма авторизации:

#### OAuth-авторизация

Протокол OAuth 2.0 позволяет приложениям работать с сервисами Яндекса от имени пользователя. Доступ каждого приложения явно ограничивается правами, заданными при его регистрации. С базовыми принципами OAuth, а также с особенностями реализации протокола в Яндексе вы можете ознакомиться в документе [OAuth-авторизация. Руководство Разработчика](https://tech.yandex.ru/oauth/doc/dg/concepts/about-docpage/).

Чтобы воспользоваться OAuth-авторизацией:

1. [Зарегистрируйте](https://oauth.yandex-team.ru/client/new) приложение на OAuth-сервере Яндекса.
    
1. [Получите](https://doc.yandex-team.ru/oauth/dg-internal/reference/web-client.html) OAuth-токен одним из доступных способов.
    
1. Настройте отправку запросов к API в соответствии с [форматом](query-format.md).
    

#### Cookie-авторизация

Авторизация с помощью Cookie `Session-id` позволяет отправлять запросы к API Кондуктора из адресной строки браузера. Чтобы использовать `Session-id` в своем приложении:

1. Авторизуйтесь во [внутреннем Паспорте](https://passport.yandex-team.ru).
    
1. Скопируйте Cookie `Session-id`.
    
1. Настройте отправку запросов к API в соответствии с [форматом](query-format.md).
    
1. Передавайте `Session-id` с каждым запросом к API Кондуктора.
