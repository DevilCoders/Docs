## route-lost worker

1. Фильтрует сходы из //home/maps/jams/production/route-quality/route_lost/<days>
- выбирает только true сходы
- фильтрует по геометрии

2. Кластеризует сходы по тройке persistent id
(до схода, маршрут после схода, трек после схода)

3. Вычисляет количество треков, сматченных на каждое ребро по табличке одного дня из //home/maps/jams/production/data/travel_times/<day>

4. Полученные кластеры сходов сортирует в порядке увеличения количества сходов / количество треков

5. Топ гипотез записывает в виде тасков фидбека в social.feedback_task
