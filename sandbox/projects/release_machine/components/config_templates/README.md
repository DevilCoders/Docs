# Конфигурирование компонент

Здесь лежат шаблоны конфигов компонент для различных релизных циклов:

- `reference_branch_config.py` — для бранчевых компонент
- `reference_tagged_config.py` — для тэговых компонент


## Конфиг

Для каждой компоненты RM должен быть создан конфиг по соответствующему шаблону. 
Конфиг нужно положить в [sandbox/projects/release_machine/components/configs](https://a.yandex-team.ru/arc/trunk/arcadia/sandbox/projects/release_machine/components/configs).
Лучше всего дать ему название, аналогичное названию компоненты (`name`
в самом конфиге) — так нам будет проще разбираться. 

В конфигах живет в основном статичная, "настроечная" логика. Конфиг используется
для порождения класса component info (`c_info`), в котором сосредоточена логика
работы с компонентой. При необходимости component info можно кастомизировать под
нужды своей компоненты — этой цели служат модули в 
[sandbox/projects/release_machine/components/custom](https://a.yandex-team.ru/arc/trunk/arcadia/sandbox/projects/release_machine/components/custom).


## Как читать шаблон

Все атрибуты, не скрытые под комментарием, являются обязательными.
Все закомментированное — опционально.
Для каждого атрибута мы постарались предоставить подробное объяснение
его назначения и приложить примеры. Если все же что-то непонятно, можно
посмотреть примеры конфигов других компонент или обратиться в чатик
поддержки RM.
