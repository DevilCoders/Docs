# Tanker Tool

- [Описание на вики](https://wiki.yandex-team.ru/users/fedot303/tankertool/)

Для использования тулзы необходимо создать файл с конфигурацией. Все пути в файле (как и путь к самому файлу) пишутся относительно того места, откуда была вызвана тулза. Всегда первым аргументом необходимо писать путь до файла с конфигурацией. Запустить tanker-tool можно двумя способами: 

1. переходим в корень репозитория тулзы, вызываем ```swift run tanker-tool <аргументы>```
2. переходим в корень репозитория тулзы, вызываем ```swift build```, возвращаемся туда, откуда хотим запускать тулзу, вызываем ```./<путь до корня tanker-tool>/.build/debug/tanker-tool <аргументы>```

Аргументы:

1. ```--remove-unused-keys``` – удаление ключей из танкера, которых нет в коде. Пример использования:

 ```swift run tanker-tool "<Путь до файла конфигурации>" --remove-unused-keys```
 
2. ```--init``` – инициализация тулзы (запись OAuth токена в локальный файл). Пример использоваться:

 ```swift run tanker-tool <Путь до файла конфигурации> --init "<OAuth токен>"```
