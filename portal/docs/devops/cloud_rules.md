# Как завести новый сервис в Nanny
Сервисы заводятся в форме: https://nanny.yandex-team.ru/ui/#/services/create-service

### Идентификатор сервиса {#id}
Должен состоять из 3 или 4 частей, соединённых дефисом:
```
{окружение}-{проект}-{сервис}[-{локация}]
```
Примеры :
* `stable-portal-androidwidget-sas` – продовый сервис виджетов для ПП Андроида, все поды находятся в Сасово.
* `testing-portal-mordago` -  тестовый контур сервиса Morda Go, внутри сервиса могут быть поды в разных ДЦ.

##### Окружение {#environment}
Одно из следующих значений:
* `testing`
* `hamster`
* `stable`

##### Проект {#project}
Всегда используется `portal`.


##### Сервис {#service}
Имя нового сервиса, который должен легко находиться в [каталоге сервисов](https://nanny.yandex-team.ru/ui/#/services). Желательно __не__ использовать дефис в имени.

##### Локация {#location}
Если планируете создавать сервисы полокационно (то есть поды в каджом сервисе находятся внутри одного ДЦ), то в идентификаторе нужно отразить локацию. Используются следующие значения:
* `vla`
* `sas`
* `man`

Есть дата-центры в Ивантеевке и Мытищах, но создавать сервисы только по необходимости:
* `iva` 
* `myt`


### Категория сервиса {#category}
Указывает местоположение сервиса в каталоге Nanny. Рекомендуется использовать `/portal/{сервис}`

### Тэги {#tags}
* `prj` – обозначает проект, используйте имя сервиса.
* `itype` – определяет тип используемых инстансов (YP-подов). Можно использовать имя сервиса.
* `ctype` –  используйте указанное выше окружение.
* `metaprj` – оставьте пустым
