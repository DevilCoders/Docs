## Что это
Key-value хранилище холодных данных в виде микросервиса под аппхостом.

## Какие проблемы решает, какая цель
В рамках выноса блока новостей для ПП из перловой морды в схему на го под аппхостом возникла необходимость в воспроизведении механизма фоллбэка при неответах бэкенда новостей. В перловой морде (монолите) он был завязан на кэш memcached, в условиях микросервисной архитектуры такой подход был признан нецелесообразным и было решено создать микросервис, который можно будет встраивать в любые аппхостовые графы для реализации фоллбэков.
Конечной целью является полное замещение старого кэша.

## Каким методом решается задача
Основным хранилищем данных является redis.
Свежие данные до хранилища доставляются с экспортящих ферм с помощью export-puller через http ручку greenbox.
Для снижения сетевой нагрузки на хранилище и времени ответа, а также для повышения надежности (на случай сетевых аварий) greenbox кэширует ответы хранилища в памяти.
