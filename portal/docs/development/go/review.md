## Review кода

### Когда проводим
* Мы всегда проводим ревью кода на любую задачу (даже если она из одной строки)

### Для чего оно нужно
* Проверка согласованности архитектуры сервиса и решения
* Проверка соответствия с правилами разработки (написанных в docs)
* По возможности проверять на баги

### Доп ревью
* Можно запросить у разработчика, который будет ревьювить ваш код дополнительную проверку бизнес логики
* Такой возможностью не стоит злоупотреблять, по скольку такое ревью заниманиет намного больше времени
