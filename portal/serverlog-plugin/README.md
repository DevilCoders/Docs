## Сборка и обновление расширения

1. Версия расширения = поле version в поле `package.json`.

2. Перед инкрементируем версию
    `npm version patch` для исправления
    `npm version minor` для новой фичи/изменения настроек
    Автоматически будут собраны расширения.

3. После сборки публикуем релиз в гитхабе, прикладывая руками к нему собранные файлы.

### FF

Для Файерфокса при инкременте версии новая версия пропишется в `updates/ff.json`. `update_link` - путь до файла `logobserver.xpi`, который приложен к релизу выше.

### Chrome & YaBrowser

В Яндекс.Браузере расширение добавлено в whitelist. Для сохранения данного состояния и возможности обновления плагина без переустановки при сборке нового релиза нужно изменить версию плагина на актуальную вот тут https://github.yandex-team.ru/broext/whitelist/blob/master/update_manifest.xml, а путь до файла `codebase` - на собранный файлик `log-observer-chrome.crx` из релиза выше. Id расширения - `ifjblmdfnllboncggckjcdjcegdcppkj`.
