# Geo-саджест

В этом разделе описан способ получения списка регионов с идентификаторами (из геобазы), названия которых начинаются с заданной строки.

Для получения списка регионов следует обратиться к следующему ресурсу:

```no-highlight
http://tune.yandex.ru/api/search/2/search.xml
```

{% note info %}

К ресурсу следует обращаться в текущем национальном домене `ua, kz, by` и пр.

{% endnote %}


**Параметры запроса:**
- `part` — строка, по которой будет осуществляться поиск. Значение по умолчанию — пустая строка;
- `types` — [типы регионов](http://wiki.yandex-team.ru/LeonidMovsesjan/Geobase/ServantMethodsDocumentation), по которым надо осуществлять поиск. Значение по умолчанию — 6,7,8;
- `count` — количество результатов в выдаче. Значение по умолчанию — 10;
- `callback` — функция обратного вызова для `jsonp`;
- `lang` — предпочтительный язык результата запроса. Значение по умолчанию — `ru`;
- `format` — формат выдачи названия региона, с идентификатором или без. По умолчанию регионы выдаются с идентификатором. Если указать `format=plain`, то названия регионов будут выдаваться без идентификатора.

В ответ на запрос приходит ответ в формате JSON следующего вида:

```javascript
["part",[["region1":id],["region2":id]],{r:revision_id}]
```

Если передан параметр `callback`, то результат запроса будет «обёрнут» следующим образом:

```
try{callback(json)}catch(e){}
```

**Примеры:**

Запрос регионов, название которых начинается с "ла":

```
Запрос:
http://tune.yandex.ru/api/search/2/search.xml?part=ла&count=3

Ответ:
["ла",[["Лахти",21126],["Лангепас",11181],["Лабытнанги",23329]],{r:2}]
```

Запрос функции обратного вызова:

```
Запрос:
http://tune.yandex.ru/api/search/2/search.xml?part=ла&count=3&callback=some_function

Ответ:
try{some_function(["ла",[["Лахти",21126],["Лангепас",11181],["Лабытнанги",23329]],{r:2}])}catch(e){}
```

Запрос по континентам:

```
Запрос:
http://tune.yandex.ru/api/search/2/search.xml?part=е&types=1

Ответ:
["е",[["Европа",111],["Евразия",10001]],{r:2}]
```
