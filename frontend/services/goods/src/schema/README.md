# Schema

Для нормализации данных, приходящих с бэкенда, используется библиотека `normalizr`.

### Общая структура


```bash
├── schema/
|   ├── entities/                                — директория с описанием сущностей
|   |   ├── screens/                             - скрины из hermione
|   |   |   └── ...
|   |   ├── common.ts                            - сюда можно сложить dummy-типы, совсем пустые, наследуемые из /typings
|   |   ├── product.ts                           - схема отдельной сущности, в данном случае - product.
|   |   └── ...                                  - остальные сущности
|   |
│   ├── entity/                                  — директория определенного типа данных, приходящих от бэкенда
|   |   ├── api.ts                               - перекладывание данных для API
|   |   ├── page.ts                              - перекладывание данных для построения state приложения, если по данным строится страница
|   |   ├── normalize.ts                         - общий хелпер нормализации
|   |   ├── schema.ts                            - описание схемы нормализации
|   |   └── types.ts
|   |
│   ├── common/                                  — общие части store между многими страницами
|   |   ├── user.ts                              - хелпер для формирования ветки `user`
|   |   └── ...
|   |
|   └─── ...
```
