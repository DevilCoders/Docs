# progress

Используется для индикации выполнения процесса — полоса загрузки. Применяется в блоке `header2`, в качестве элемента `header2__progress`.

Растягивается по ширине контейнера от 0 до 100%.

## Обзор

### Модификаторы блока

| Модификатор | Допустимые значения | Способы установки | Описание |
| ----------- | ------------------- | ----------------- | -------- |
| [timing](#mods-timing) | `linear` | `JS` | Анимация при заполнении полосы загрузки. |

## Подробное описание

### Модификаторы блока

<a name="mods-timing"></a>

#### Модификатор `timing`

Определяет способ (вид) анимации или ее отсутствие при заполнении полосы загрузки. Выставляется в методе блока `update(value, timing)`.

> **Примечание.** Добавить новые значения модификатора можно в `CSS` блока на уровне сервиса, по аналогии с [linear](progress.css#L9).

| Допустимые значения | Описание |
| --- | --- |
| `linear` | Анимация длительностью 200 мс. |

Способ установки: `JS`.

```js
[
    {
        block: 'x-table',
        yhead: ['timing', 'linear', 'Без анимации'],
        xhead: ['value', '10%', '50%', '100%'],
        body: ['linear', ''].map(function(timing) {
            return [0.1, 0.5, 1].map(function(value) {
                return {
                    block: 'progress',
                    js: {
                        value: value,
                        timing: timing
                    }
                };
            });
        })
    },
    {
        block: 'x-deps',
        content: [
            {
                block: 'i-bem',
                elem: ['dom']
            }
        ]
    }
]
```
