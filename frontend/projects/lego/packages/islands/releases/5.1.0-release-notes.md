# islands 5.1.0

В этом релизе много важного:
- в `islands` появились новые блоки — `select2`, `user2`, `textinput`, `textarea`,
- в Лего началась официальная поддержка React,
- внутрь `islands` перенесена библиотека `meccano`,
- блоки библиотеки переведены на индонезийский язык.

Изменения, вошедшие в релиз islands 5.1.0, разделены на группы:

* [минорные изменения](#Минорные-изменения) — задачи, в рамках которых произошло расширение API блоков;
* [исправлено и улучшено](#Исправлено-и-улучшено) — различные исправления, не затрагивающие API блоков;
* [дополнительно](#Дополнительно) — задачи, не касающиеся кода блоков (документация, инфраструктура и т.д.).

Все важные изменения подробно описаны [ниже](#Пояснения).

## Пояснения

<a name="ISL-3103"></a>
### [asserts: Поправить срабатывания по коду][ISL-3103]

Были внесены следующие исправления:

- В ряде блоков с префиксом `m-*` не передавался модификатор `theme`, теперь передаётся явно с нужным значением.
- В блоке `m-editor-popup` теперь используются поля блока `input` вместо атрибутов элемента `control`.
- Для единообразия именования полей в BEMJSON поле `mods` элементов заменено на `elemMods`, поскольку поле `mods` может описывать модификаторы блока (BH).
- Модификатор `size` для кнопки в блоке `alert-controller` не передавался. Теперь используется размер `'m'`.
- Атрибут `target` для элементов `share-item` блока `share-dropdown` выставлялся некорректно (BH).

<a name="ISL-3091"></a>
### [button2: Доработать тему websearch][ISL-3091]

Модификатор `size` со значением `ws-head` теперь не нужен для отображения поисковой кнопки. Модификатора `theme` со значением `websearch` вполне достаточно.

<a name="ISL-2532"></a>
### [button: У круглых кнопок-ссылок неправильно выравнивается иконка][ISL-2532]

В блоке `button` с модификатором `round` добавлено свойство `line-height`. Это сделано для того, чтобы при использовании тега `<a>` свойство `vertical-align` работало как ожидается. Возвращен стандартный отрицательный `margin` для `button__icon`.

<a name="ISL-3065"></a>
### [domik: Поехала верстка для hy и ka языков][ISL-3065]

Для блоков `auth`, `copyright`, `header2`, `i-services` и `user` обновлены переводы для грузинского и армянского языков (`hy` и `ka`).
Элементу `haunter` блока `auth` добавлено свойство `max-width`, чтобы слишком длинный текст переносился на другую строку. Элементу `label` блока `checkbox` в составе блока `auth` добавлено свойство `line-height: normal`, чтобы при переносе высота строки не была слишком большой.

<a name="ISL-3223"></a>
### [header2: Добавить новое лого][ISL-3223]

В элемент логотипа `header2` были добавлены пресеты с новыми логотипами:
  * `header2__logo_preset_ys-ru`
  * `header2__logo_preset_ys-en`

Для перехода на новый логотип достаточно заменить значение модификатора preset.

Было:
```
({
    block: 'header2',
    logo: {elem: 'logo', elemMods: {preset: 'ru'}}
})
```

Стало:
```
({
    block: 'header2',
    logo: {elem: 'logo', elemMods: {preset: 'ys-ru'}}
})
```

Пресеты для старого логотипа объявлены устаревшими и будут удалены в следующем мажорном релизе.

Так же, устаревшим был отмечен элемент `nameplate`, поскольку он был нужен для старого логотипа.

<a name="ISL-3097"></a>
### [i-bem: changeThis — объявить deprecated][ISL-3097]

В блоке `i-bem` объявлен устаревшим метод `changeThis`. Он использовался для изменения контекста передаваемой функции.

Теперь вместо `changeThis` следует использовать `Function.prototype.bind`.

**Было:**

```javascript
._getProvider().get(val, pos, this.changeThis(this._onData));
```

**Стало:**

```javascript
._getProvider().get(val, pos, this._onData.bind(this));
```
или

**Было:**

```javascript
this.changeThis(fn, ctx);
```
**Стало:**

```javascript
fn.bind(ctx || this);
```

<a name="ISL-1828"></a>
### [select2: Новая версия блока select][ISL-1828]

Мы создали новый блок – `select2`.

Особенности блока:
 * Блок поддерживает три режима работы: одиночный выбор, мульти-выбор и возможность выбрать одну опцию или не выбирать ничего
 * На touch-устройствах возможно использовать родной `<select>` для выбора
 * Клавиатурный выбор опций как в родном для ОС `<select>`
 * Возможно показывать иконку на кнопке при смене опций
 * Разные варианты выбора ширины кнопки, поведения смены текста на кнопке при выборе опций
 * Блок использует новые Лего-блоки: `popup2`, `button2` и `menu`.

```js
({
    block: 'select2',
    mods: {type: 'radio', theme: 'normal', size: 's', text: 'vary'}
    control: true,
    name: 'myselect',
    val: '1',
    items: [
        {val: '1', text: 'Раз'},
        {val: '2', text: 'Два'},
        {val: '3', text: 'Три'},
    ]
});
```
