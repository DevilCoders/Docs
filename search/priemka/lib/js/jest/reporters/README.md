# Что это такое
Это класс репортера для фреймворка тестирования всяких фронтенд-штук [https://facebook.github.io/jest/ Jest]. Лучше и больше всего подходит для тестирования [https://facebook.github.io/react React'a].

Кастомный репортер понадобился для запуска тестов фронтенда в sandbox-задачах, логирования результатов исполнения тестов и выплёвывания результатов в упрощённом формате.

## Текущие ограничения
1. Не поддерживаются вложенные *Test Suites*. Репортилка прекрасно работает со standalone-тестами и иерархией *Test Suite* => *Test*.
2. Runtime-ошибки пока ловятся только на коде возврата подпроцесса, гоняющего тесты. Репортилка их не ловит.
3. Работает на node версии 6 или более.

## NodeJS в Аркадии? Это же нелегально!
Это не компонент бэкэнда на JS - это всего лишь компонент системы сборки и тестирования фронтенда.
Времена сейчас настали такие, что без автоматизированного тестирования и сборки JavaScript'a никуда (если вы не большой оригинал и продолжаете верстать странички на голом html, анимируя их с помощью jQuery).

# Как запустить тесты, тестирующие этот компонент для тестирования
```
npm install
cd <curDir>/specs
<PATH TO JEST> test_jest_reporter
```
`PATH TO JEST` - `jest`, если Jest установлен глобально в системе, или `node <curDir>/node_modules/jest/bin/jest.js`, если установлен локально.
