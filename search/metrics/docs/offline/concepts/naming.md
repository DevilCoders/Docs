# Формат именования метрик

В разделе описаны форматы именования офлайн-метрик. Знание формата может понадобиться, чтобы:

- расшифровать название метрики в интерфейсе [Metrics](http://metrics.yandex-team.ru);
- корректно назвать новую метрику при ее создании.

## Метрики качества поиска {#common-format}

Метрики качества поиска — это [оценочные](./basicinfo.md#judge), [информационные](./basicinfo.md#info) и [технические](./basicinfo.md#alert) метрики, которые предназначены для оценки SERP по различным параметрам. Например, общая оценка качества поиска ([pfound](p.md#pfound)) или заполненность первой страницы поисковой выдачи ([small-serp](s.md#small-serp)).

Метрики могут рассчитываться как по по всем найденным документам, так и с некоторыми ограничениями:

- На глубину выдачи, когда при расчете метрики учитываются только первые n документов выдачи.
    
- На оцененность документов, когда при расчете метрики учитываются только оценненные документы, а остальные исключаются из рассматриваемой выдачи (подробнее см.  [Классические и judged метрики](http://doc.yandex-team.ru/analytics/metrics/concepts/judgedNnotjudged.xml) документа [Оценка качества поиска](http://doc.yandex-team.ru/analytics/metrics/concepts/About.xml)).
    
    Частным случаем метрик, при расчете которых учитывается оцененность документов, являются [метрики оцененности выдачи](#judged-format).
    

В данном документе приводится описание метрик качества поиска без учета [глубины выдачи](#d2) и [оцененности](#judged) документов (исключение составляют [метрики оцененности](../reference/term.md#evaluation-metrics) поисковой выдачи, представленные в разделе [J](j.md)).

Название метрики качества поиска может складываться следующим образом:

```no-highlight
judged-<Название метрики><-Глубина выдачи/Количество оцененных документов>
```

#|
|| **Составляющая названия метрики** | **Описание** ||
|| `judged-` {#judged}| Признак того, что при расчете метрики учитываются только оцененные документы. Опциональная составляющая.

Например, <q>pfound-10</q> — это вероятность того, что пользователь найдет информацию, соответствующую интенту запроса, среди документов в первых десяти результатах выдачи, а <q>judged-pfound-10</q> — среди оцененных документов в первых десяти результатах выдачи.

Подробнее см.  [Классические и judged метрики](http://doc.yandex-team.ru/analytics/metrics/concepts/judgedNnotjudged.xml) документа [Оценка качества поиска](http://doc.yandex-team.ru/analytics/metrics/concepts/About.xml). ||
|| <`Название метрики`> | Название метрики, которое указывает на ее свойства и правила расчета. Обязательная составляющая. ||
|| <`-Глубина выдачи/`
`Количество оцененных документов`> {#d2}| Глубина выдачи или количество оцененных документов, участвующих в расчете метрики. Опциональная составляющая.

Если глубина не указана, то используется глубина серпа, по которому рассчитывается метрика.

Также данная составляющая названия может быть использована для указания на позицию документов в выдаче, для которой рассчитывается метрика (в этом случае задается в формате « `p<Позиция документа> `»).

Например, метрика <q>spamDCG-DORVEY-10</q> отражает долю спама типа Dorvey в первых десяти результатах выдачи, а метрика <q>images-p1</q> — среднюю оценку картинки на первой позиции. ||
|#


## Метрики оцененности выдачи {#judged-format}

Метрики оценненности выдачи — это [технические](./basicinfo.md#alert) метрики, которые отражают полноту оценнености выдачи по определенному критерию (шкале). Например, доля документов выдачи, для которых известна оценка языка ([judged-language](../concepts/j.md#judged-language)), или доля документов поисковой выдачи, оцененных по шкале асессорской авторитетности ([judged-tw](../concepts/j.md#judged-tw)).

В названии метрик оцененности выдачи обязательно присутствует префикс <q>judged</q>.

Название метрики может складываться следующим образом:

```no-highlight
judged<Глубина выдачи>-<Название шкалы оценки>
```

{% note info %}

В данном документе приводится описание метрик оцененности поисковой выдачи без учета [глубины выдачи](#d1) (см. раздел [J](j.md)).

{% endnote %}


#|
|| **Составляющая названия метрики** | **Описание** ||
|| `judged` | Признак того, что при расчете метрики учитываются только оцененные документы. Обязательная составляющая. ||
|| <`Глубина выдачи`> {#d1}| Глубина выдачи, рассматриваемая при расчете метрики.

Если глубина не указана, то используется глубина серпа, по которому рассчитывается метрика. Опциональная составляющая.

Например, метрика <q>judged10-tw</q> отражает долю документов, оцененных по шкале асессорской авторитетности,
 в первых десяти результатах выдачи. ||
|| `<Название шкалы оценки`> | Название критерия или шкалы, по которой оценивались документы поисковой выдачи. Обязательная составляющая.

Пример шкалы: [шкала доверия TRUSTWORTHINESS](https://wiki.yandex-team.ru/trustworthiness#shkalaocenki).

Пример метрики: <q>judgedN-duplicate-images</q> — метрика отражает долю картинок, оцененных по шкале наличия дублей, в первых n результатах поисковой выдачи. ||
|#
