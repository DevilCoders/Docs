# Терминология

#### Соседний SERP {#neighbor-serp}

SERP, с которым проводится сравнение.

#### Идеальный ответ {#ideal-resp}

Выдача, упорядоченная по убыванию релевантности.

Если при расчете метрики учитываются результаты поиска нескольких поисковых систем, то идеальный ответ строится на основе результатов всех систем, участвующих в расчете метрики. При этом, если какой-либо элемент содержится в выдаче сразу нескольких систем, в идеальную выдачу он попадает только один раз.

#### Разгруппировка {#ungroup}

Возможность показывать на странице результатов поиска более одного документа с сайта.

Подробная информация о разгруппировке представлена на странице [Разгруппировка ресурса](http://wiki.yandex-team.ru/JandeksPoisk/KachestvoPoiska/ungrouping/.show#chtobyloranshe) в [Вики](https://wiki.yandex-team.ru).

#### Метрики оцененности выдачи {#evaluation-metrics}

[Технические](../concepts/basicinfo.md#alert) метрики, показывающие полноту оценнености выдачи по определенному критерию (шкале). Например, доля документов выдачи, для которых известна оценка языка ([judged-language](../concepts/j.md#judged-language)) или доля документов поисковой выдачи, оцененных по шкале асессорской авторитетности ([judged-tw](../concepts/j.md#judged-tw)).

В названии метрик оцененности выдачи обязательно присутствует префикс <q>judged</q>. В данном документе описание метрик оцененности выдачи представлено в разделе [J](../concepts/j.md).
