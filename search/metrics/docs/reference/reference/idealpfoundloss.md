# Фильтры исторического графика «Потери ideal pfound»

Список потоков, отображаемых на [историческом графике Потери ideal pfound](../concepts/history-graph-types.md), с описанием причины фильтрации документов представлен в таблице ниже.

Тип фильтра | Описание
----- | -----
`NOT_IN_BASE` | Документ отсутствует в индексе.<br/><br/>График, соответствующий данному потоку, отражает значение метрики, которое может теоретически быть достигнуто с набором документов, имеющихся в базе.
`CAN_NOT_ACCESS` | Отсутствует доступ к мусорному шарду (см. [Глоссарий терминов поиска](http://wiki.yandex-team.ru/JandeksPoisk/Dokumentacija/Glossarijj#littershard)), содержащему информацию о документе.<br/><br/>Мусорный шард имеет только одну реплику (тестовые отсутствуют). Batch Mode не имеет доступа к мусорному шарду, но по индексу может определить, что документ находится в его базе. При этом оценка возможности прохождения данного фильтра отсутствует и предполагается, что документ не пройдет данный фильтр.
`CLASSIFIER_ERROR` | Ошибка классификатора на метапоиске. Запрос не передается на требуемый шард (содержащий информацию о документе).<br/><br/>Документ найден на одном из шардов, но метапоиск не передал запрос этому шарду. Например, документ найден в украинской базе, но запрос не передается, так как задан на русском языке.
`NOT_TOUCHED` | Разновидность фильтрации по кворуму. Документ найден на базовых поисках, релевантен запросу, но не найден итератором при обработке запроса в поднятом наборе словопозиций.<br/><br/>Потери документов на данном этапе возможны, например, при поиске по синонимам. Если поисковая фраза расширена синонимами, но колдунщик не справился с поставленной задачей, то документ может не быть найден.
`FILTERED_BY_PRUNING` | Фильтрация по прюнингу. Исключение из дальнейшего ранжирования заведомо нерелевантных документов.
`FILTERED_BY_QUORUM` | Фильтрация по кворуму. Исключение из дальнейшего ранжирования документов, не содержащих минимального установленного количества слов запроса.
`FILTERED_BY_EXTRA_PRUNING` | Дополнительная фильтрация по прюнингу.
`FILTERED_BY_RESTRICTS` | Фильтрация по ограничениям на документ, задаваемым в запросе. Таким ограничением может быть, например, оператор `>>` (поиск по сайту).
`FILTERED_BY_CANTREAD_FILTER` | Фильтрация по невозможности расчета словопозиций.
`FILTERED_BY_DOCID_FILTER` | Фильтрация `Docid`. Техническая проверка, при штатной обработке на фильтрацию документов не влияет.
`FILTERED_BY_ANTISPAM` | Фильтрация антиспама.
`FILTERED_BY_THIN_PAGE` | Фильтрация по правилу «Thin page» (разновидность антиспама). Правило ограничивает число проиндексированных страниц хоста пятью документами.
`FILTERED_BY_PRON_YANDEXCOM` | Фильтрация документов при поиске по yandex.com.
`FILTERED_BY_PRON_NOCYR` | Фильтрация на русский язык. Наиболее вероятно, документ не содержит кириллических символов.
`FILTERED_BY_PORNO` | Фильтрация по порно.
`FILTERED_BY_DYNAMIC_ANTISPAM` | Фильтрация на стадии динамического антиспама.
`FILTERED_BY_NO_HITS` | Фильтрация за отсутствие текстовых и линковых хитов.
`FILTERED_BY_NO_TR_POS` | Фильтрация документа из-за отсутствия информации о словопозициях в индексе.
`FILTERED_BY_QUORUM_AFTER_FILTERING` | Фильтрация при повторной проверке кворума (динамический кворум).
`FILTERED_BY_FAKE_DOC` | Фильтрация документов, не содержащих тела (только заголовки).<br/><br/>Робот не имеет возможности обойти документ (например, индексация запрещена настройками `robots.txt`).
`FILTERED_BY_FASTRANK` | Фильтрация документа на стадии фастранка. Поток отражает потери метрик из-за документов, не дошедших до ранжирования с помощью Matrixnet.<br/><br/>Все стадии до фильтрации по фастранку проходят приблизительно 5-10 тысяч документов. Для данных документов рассчитывается набор «легких» факторов, по которым считается формула фастранка. После расчета формулы для дальнейшего ранжирования с помощью Matrixnet выбираются приблизительно 500 первых документов.
