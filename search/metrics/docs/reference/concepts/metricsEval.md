# Оцененность выдачи

В идеальном случае множество запросов, по которым рассчитываются метрики, должно быть полностью оценено. Это обеспечивает высокую степень достоверности значений метрик.

Выдача меняется во времени в связи с изменениями в:

- интернете (удаление старых документов и/или появление новых);
- алгоритмах поиска (появление в top-10 новых неоцененных документов).

Изменение выдачи может приводить к изменению (уменьшению) ее оцененности. Таким образом, оцененность выдачи меняется во времени и имеет тенденцию к уменьшению.

Для увеличения оцененности выдачи используется _дооценка_ — оценка документов, возвращаемых по ранее оцененным запросам, но попавшим в число top-документов после оценки.

В силу ограниченности асессорских ресурсов дооценка всех новых документов, попадающих в поисковую выдачу после каждого изменения в алгоритме, невозможна. Поэтому дооценка может быть запрошена для ограниченного числа top-документов (от top-1 до top-10).

Так как дооценка является трудоемкой и приоритетной задачей асессоров следует иметь в виду устойчивость различных метрик к неполной оцененности. Средняя оцененность серпа определяется значением метрики **judged10**. После анализа данных об оцененности поисковой выдачи можно обратиться к [экспериментальным данным](../reference/AssessmentMetricCorellation.md), отражающим поведение метрик при разном уровне оцененности.

Следует также иметь в виду, что [judged метрики](judgedNnotjudged.md) являются более устойчивыми, чем их классические аналоги (об этом свидетельствуют результаты [эксперимента](../reference/AssessmentMetricCorellation.md)).
