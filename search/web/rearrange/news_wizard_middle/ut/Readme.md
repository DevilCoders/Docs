При старте тестов подгружается файл ut_data.json, внутри которого словарь с данными для каждого из тестов. Отдельный тест работает с данными по соответствующему ключу.

Внутри каждого из тестов происходит следующее:
  1. Из данных берется группировка d:fresh
  2. Из данных берутся дополнительные настройки
  3. Запускаются начальные стадии переранжирования с настройками из пункта 2 поверх группировки из пункта 1
  4. Из данных берется QueryData (данные из kv-SaaS), подкладываются в контекст переранжирования между AfterMerge и AfterFetch
  5. Запускаются оставшиеся стадии
  6. SerpData документов из группировки newswizard сравнивается с эталоном из входных данных
