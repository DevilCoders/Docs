# Как правильно заполнять SPI?

Во-первых, надо заполнить основные вещи, без которых не будет ясно к какому инциденту относится **SPI**.

## Основная информация
- **Название тикета** - оно состоит из 2 частей:
    - **Дата**
    - **Компонента/Сервис**
    - **Датацентр**
    - **Краткое описание** - в 2х-3х словах, что случилось.
    - Пример: `[2019.10.10] PROD-PORTAL / STREAM: Рост ошибок стрима (ALL)`
- **Испольнитель** - человек занимающийся проблемой. Если их несколько ставим основного дежурного.
- **Компонента/Сервис**
- **SRO** - временные рамки проявления проблемы.
- **Линия поддержки**, которой передан для устранения инцидент. В частности:
    - marty - Дежурной смене Поиска
    - devops - Девопсам сервиса
    - noc - Дежурным NOC
    - platform - Дежурным инфраструктуры
    - service - Разработчикам сервиса

## Категории

Кроме того, в форме **SPI** есть поля рекомендуемые для заполнения:

 - **Что во время инцидента видели пользователи**
 - **Описание** - тут кратко излагается техническая сторона вопроса - что конкретно сломалось.
 - **Действия дежурного** - таймлайн действий человека, который в данный момент находится на дежурстве за сервис.
 - **Анализ** - выявление причин и способов решения проблемы. Также подсчет ydt.
 - **Диагностика** - в эту категорию попадают все материалы (логи, графики, алерты), которые будут полезны при анализе.
 - **Меры предотвращения** - действия, которые будут предприняты для предотвращения воспроизведение проблемы. Формируются **action items**.
 - **К каким проблемам относится** - связанные в этим инцидентом проблемы. Обычно прикрепляются в форме **SPI**.
 - **Смежные сервисы, которые затронуло инцидентом** - сервисы, пострадавшие в ходе инцидента. Обычно прикрепляются в виде **SPI**.

Разбор как правило проводится либо в чате протокола, а потом с помощью бота приносится в **SPI**, либо в комментариях.

## Качетсвенно заполненный SPI

**Качественно** заполненным считается инцидент, в котором:
 - Задано время начала инцидента - **SRO Begin Time**. Берётся по времени срабатывания алёрта\оповещения Марти\последнего флапа.
 - Задано время оповещения ответственного - **SRO Notification Time**. 
 - В инциденте описаны действия дежурного в виде таймлайна.
 - Описано произошедшее с точки зрения Марти
 - Приложен скриншот графика, на которм видна проблема
