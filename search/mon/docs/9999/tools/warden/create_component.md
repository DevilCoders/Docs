# Редактирование компоненты

Для редактирования параметров компоненты необходимо выбрать компоненту, параметры которой требуется изменить. Далее в верхнем левом углу страницы необходимо нажать на кнопку "Edit":

**todo** component_edit_button.png

Далее, в открывшемся окне можно изменить параметры компоненты. Параметры сгруппированы по вкладкам "Info", "Flow Settings" и "Report Settings".  

## Info

На вкладке "Info" представлены общие параметры компоненты:

**todo** component_edit_info_modal.png

Назначение большей части параметров очевидно из подсказки в поле ввода. Остальные параметры рассмотрим здесь:

- **Weight** - вес компоненты для расчета метрики YDT. Для горизонталей, к которым не применим самостоятельный расчет метрики YDT, а только как сумма пострадавших вертикалей, в качестве веса следует прописать 0 в процессе устаревания.
- **Horizontal** - наличие горизонтали у компоненты. Если поставить галочку, появится поле в котором нужно указать горизонталь.
- **Availability Metric** - метрика по которой будет оцениваться доступность компоненты.
- **ABC service** - привязка компоненты к сервису из ABC
- **Tags** - Здесь можно прописать теги для группировки "своих" компонент - компонент, за которые отвечает одна команда или один владелец. В дальнейшем будет реализован поиск по тегам.
- **Owners** - владельцы компоненты. Призываются в случае эскалации на вертикаль - начиная с момента отсчёта времени Т5(см.ниже).
- **Incident coordination chat** - чат для координации инцидента. Используется YaIncBot для оповещения дежурных компоненты об инцидентах.
- **Goal URL** - ссылка на цель по 9999 в голсятнице.
- **Tier** - текущий тир надежности компоненты.
- **Chats** - в данном поле команда сервиса может указать дополнительные чаты, которые она использует помимо координационного чата.
= **Environment** - в данном поле можно указать окружения, которые в дальнейшем можно использовать при заведении инцидентов через тикенатор. Они будут доступны в поле "Контур" Тикенатора.§правка
