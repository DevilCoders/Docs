# Описание и синтакс правил

Грамматики для Remorph описываются в виде набора файлов, содержащих так называемые правила. Файлы с правилами компилируются в бинарное представление и затем используются соответствующим матчером. В данном разделе описан синтаксис правил.


## Комментарии {#comments}

Строки, начинающиеся с символа #, являются комментариями и игнорируются.


## Предложения {#sentences}

Каждый файл состоит из одного или нескольких предложений. Каждое предложение может занимать несколько строк и обязательно заканчивается символом `;`.

Предложение начинается с ключевого слова, которое определяет дальнейшую структуру и смысл предложения. Допускаемые типы предложений зависят от конкретного модуля Remorph.

Предложения обрабатываются последовательно в порядке появления в файле(ах).


## Включения {#inclusions}

В правилах любого типа для включения в файл текста другого файла (по аналогии с C++ `#include`) используется предложение типа `include`:

```cpp
include "<имя файла>";
```

Включаемый файл ищется относительно директории текущего обрабатываемого файла.

Например,

```cpp
include "rules.def.remorph";
```

включает содержимое файла `rules.def.remorph` в текущий файл.


## Правила различных типов {#rules-types}

Каждый модуль Remorph имеет свои собственные типы предложений и диктует структуру, в которой они должны быть описаны.

- [Remorph - регулярные выражения над словами](rules-remorph.md).
- [Char - посимвольные регулярные выражения, выделяющие последовательности слов](rules-char.md).
- [TokenLogic - логические выражения над словами без учета порядка их следования](rules-tokenlogic.md)
- [Литералы - основная единица правил Remorph и TokenLogic](literals.md)
