# Ранжирование покрытий

Ранжирование покрытий - задача, решаемая при разрешении омонимии между различными вариантами покрытия исходного текста результатами работы Remorph (результатами работы того или иного матчера или фактами).

С точки зрения внутреннего API в коде разрешение двусмысленности всегда может быть выполнено с помощью функции `SolveAmbiguity(результаты[, метод ранжирования])`.


## Методы ранжирования {#metodyranzhirovanija}

Методы ранжирования - упорядоченная последовательность сортирующий критериев, которые последовательно применяются к парам покрытий для определения их порядка.

Критерии могут быть следующими:
- `RC_G_COVERAGE`, `"greater-coverage"` - покрытие тем лучше, чем больше суммарное количество захваченных токенов;
    
- `RC_G_WEIGHT`, `"greater-weight"` - покрытие тем лучше, чем вес покрытия больше;
    
- `RC_L_COUNT`, `"less-count"` - покрытие тем лучше, чем количество элементов в покрытии меньше.
    
По умолчанию используется следующий метод ранжирования: `RC_G_COVERAGE`, `RC_L_COUNT`, `RC_G_WEIGHT`.

При работе с API для представления метода ранжирования используется класс `TRankMethod` ([dict/remorph/common/rank.h](https://arcadia.yandex.ru/wsvn/arc/trunk/arcadia/dict/remorph/common/rank.h)).

При указании методов ранжирования при запуске утилит или при задании опций в фактах и каскадах используется строковый вариант, в котором критерии перечисляются через запятую (например, `"greater-coverage,less-count,greater-weight"`).
