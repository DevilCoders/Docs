# Internal REST-api avia-backend

## Пример ответа
```
{
    'status': 'ok|error',
    'data': 'something'
}
```

*status* - ok - запрос успешно отработан, error - что-то пошло не так
*data* - ответ ручки, тут может лежать все что угодно, для более подробной информации нужно идти в документацию по интересущей вас ручке. НО ответ можно вычитывать, только если *status* - ok

## Возможныые http статусы
200 - значит в status лежит ok
400 - значит запрос не прошел валидацию подробности можно посмотреть в *data*
500 - что-то пошло не так,  в data будут лежить подробности
