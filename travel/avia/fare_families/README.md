#### Как пользоваться
0.  Сборка и запуск проекта:
    ```shell script
    cd ${ARCADIA}/travel/avia/fare-families
    ya make
    export CONFIG_PATH=config/api/config.development.yaml
    ./cmd/bin/api
    ```

1.  Примеры использования отладочных ручек

    Посмотреть список загруженных файлов тарифных правил и сколько брендов тарифов в каждом из них:<br/>
    ```http://localhost:9000/fare-families/files```

    Посмотреть все загруженные тарифные правила а/к 26:<br/>
    ```http://localhost:9000/fare-families/dump?carrier=26```

    Посмотреть все загруженные тарифные правила а/к 26, где в названии бренда есть подстрока ```promo```:<br/>
    ```http://localhost:9000/fare-families/dump?carrier=26&brand=promo```

    Посмотреть все загруженные тарифные правила а/к 26, где в названии бренда есть подстроки ```promo``` или ```lite```:<br/>
    ```http://localhost:9000/fare-families/dump?carrier=26&brand=promo|lite```

    Посмотреть все загруженные тарифные правила а/к 26, под которые подходит код тарифа ```WBS```:<br/>
    ```http://localhost:9000/fare-families/dump?carrier=26&tariff_code=WBS```

    **Note**. Параметр ```brand``` - это регулярное выражение, по нему проверяется строка ```brand``` из описания тарифных правил.
    Параметр ```tariff_code``` - это строка, фильтр проверяет, подпадает ли она под регулярное выражение ```tariff_code_pattern```
    из описания тарифных правил. Если в URL использованы оба фильтра, выводятся только те тарифные правила, которые подпадают
    под оба фильтра.
