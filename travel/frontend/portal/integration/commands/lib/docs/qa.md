## QA атрибуты

Изменение имен классов или js разметки не должно ломать тесты. Поэтому предлагается использовать подход, который называется qa атрибуты. Можно посмотреть обоснование данного подхода в [best practices библиотеки cypress][cypress-best-practices].

Элементы, которые используются в тестах помечаются дата атрибутом `data-qa` со значением, которое описывает назначение данного элемента.

В тестах вместо обычного css-селектора мы передаем `qaName` для взаимодействия с элементом.

Чтобы этот подход хорошо работал, необходимо сделать возможным пробрасывать `qaName` в общие компоненты, чтобы можно было его задать в месте их использования.

[cypress-best-practices]: https://docs.cypress.io/guides/references/best-practices.html#Selecting-Elements
