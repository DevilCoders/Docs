# Бункер

Для работы с динамическим контентом/конфигами мы используем сервис [Бункер](https://wiki.yandex-team.ru/verstka/tools/bunker/).

### Проект в бункере

Ссылка на наш проект в бункере: [https://bunker.yandex-team.ru/travel](https://bunker.yandex-team.ru/travel).

### Права доступа

Запрос прав происходит через сервис [IDM](https://idm.yandex-team.ru/).
Права можно запросить на весь проект (узел travel), либо на какие-то конкретные узлы.

```
Бункер > travel [> конкретные узлы] > Выбору узла завершен > Уровень доступа
```

### Структура хранения

Основная единица хранения - это узел. Он содержит в себе структурированные текстовые данные.
А также связанные узлы детей.

Основные операции с узлом:

-   добавление нового узла (иконка плюса внизу под списком узлов)
-   удаление узла (шестеренка в списке)
-   редактирование, сохранение, публикация узла (редактор узла)

Редактор узла имеет два режима:

-   Normal доступен не всегда, если доступен, то предоставляет GUI для просмотра и редактирования
-   RAW доступен, отображает сырые текстовые данные

### Схемы для данных

Структуру данных **рекомендуется** защищать JSON схемой.
Данные проверяются схемой при сохранении, а также появляется GUI, чтобы редактировать контент.
Схемы хранятся также в узлах у нас это поддерево узла .schemes:
[https://bunker.yandex-team.ru/travel/.schemes/](https://bunker.yandex-team.ru/travel/.schemes/).

Чтобы использовать добавленную схему:

1. нужно добавить схему в [список схем проекта](https://bunker.yandex-team.ru/travel).
2. при создании узла указать новую схему для узла
