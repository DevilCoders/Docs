### CLI-утилита для инвалидации офферов

Позволяет инвалидировать оффера по заданным критериям в офферкеше и бойлере. Инвалидация приведёт к удалению офферов из офферкеша и перепрогреву их бойлером в порядке очереди (как если бы у них просто истёк TTL).

#### Использование

Инвалидация происходит в рамках одного отеля. \
Указание отеля возможно по пермалинку или hotelId. \
 В случае инвалидации по пермалинку, утилита сама разрезолвит его в набор hotelId-ов, сходив для этого в офферкеш.

Доступно два режима инвалидации:
* По checkin-checkout. Будут инвалидированы все записи с указанным checkin и checkout. При этом одну из границ можно не указывать. Это будет значить, что эта граница может быть любой, т.е. будут инвалидированы те оффера, у которых совпадают явно указанные границы.
* По интервалу инвалидированных дат (datefrom-dateto). Будут инвалидированы все записи, интервал проживания которых пересекается с datefrom-dateto (checkout в данном случае считается не включительной границей). Обе границы опциональны, значения по умолчанию для них - `-inf` и `+inf` соответственно.

##### Примеры запуска:

Инвалидация офферов в тестинге в отеле с пермалинком 123 с датой заезда 2020-09-26 и датой выезда 2020-09-27:

`./cache_invalidation_tool --permalink 123 --host travel-hotels-searcher-test.yandex.net by-inout --checkin 2020-09-26 --checkout 2020-09-27`

Инвалидация офферов в тестинге в отеле с partnerId - booking и originalId - 42, даты проживания которых содержат интервал с 2020-09-20 по 2020-09-22 (т.е. CheckIn <= 2020-09-22 && CheckOut > 2020-09-20):

`./cache_invalidation_tool --hotel-id PI_BOOKING.42 --host travel-hotels-searcher-test.yandex.net by-target --from-date-inclusive 2020-09-20 --to-date-inclusive 2020-09-22`
