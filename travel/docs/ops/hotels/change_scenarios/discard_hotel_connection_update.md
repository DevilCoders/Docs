## Отклонить изменения у отеля на [бумажном] договоре без перезапроса данных у партнера и обратно подключить отель

Из-за ограничений в формате данных на стороне партнера невозможно сделать идентичными данные на стороне партнера как в Балансе.  
Из-за этого приходится игнорировать расхождение данных, приходящие от партнера и отменять изменения.
Вот с таким отелем начали случаться проблемы: https://st.yandex-team.ru/HOTELPROCESS-6405
Пример тикета, который нужно обрабатывать текущей инструкцией: https://st.yandex-team.ru/HOTELPROCESS-6859
Нужно обратить внимание, что есть строка: *`Внимание! С Юр. лицом ... был заключен бумажный договор. В связи с этим смену реквизитов нужно делать вручную.`*  

В такой ситуации нужно указать отель и ID обновления в скрипте:
```sql
update hotel_connections
set state = 3
where hotel_code = '<partner_hotel_code>' and partner_id = <partner_id>;

update hotel_connection_updates
set state = 4
where id = '<hotel_connection_update_id>';
```
