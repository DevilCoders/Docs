# Трагический фильтр

Трагический фильтр — инструмент, основной задачей которого является определение трагичности контента документа на основании поиска в нем фраз из словаря трагических фраз.

На страницах с трагическим содержанием контекстная реклама не показывается.

## Принцип работы Трагического фильтра {#principles}

В основе работы Трагического фильтра лежит SVM —Support vector machines ([Метод опорныx векторов](http://ru.wikipedia.org/wiki/%D0%9C%D0%B5%D1%82%D0%BE%D0%B4_%D0%BE%D0%BF%D0%BE%D1%80%D0%BD%D1%8B%D1%85_%D0%B2%D0%B5%D0%BA%D1%82%D0%BE%D1%80%D0%BE%D0%B2)). Словарь трагических фраз сформирован SVM в результате обработки текстов Яндекс.Новостей. Документы, относящиеся к разделу «Происшествия», считались документами с трагическим содержанием, новости других разделов — нетрагическим содержанием.

Для поиска фраз из словаря в документе используется Onrop. На основании найденных фраз SVM предсказывает трагичность документа.

## Исключения для сайтов новостей {#exeptions}

Для сайтов новостей, входящих в РСЯ, в работе Трагического фильтра предусмотрены следующие исключения:

* если страница определена как «лента новостей» (список новостей), то реклама на ней показывается в любом случае, признак трагичности игнорируется;
* если страница определена как «страница новости», то выводы о трагичности ее контента делаются исключительно на основании текста самой новости, ленты новостей, которые могут располагаться на этой же странице и в которых могут быть обнаружены заголовки с трагическим содержанием, игнорируются.
