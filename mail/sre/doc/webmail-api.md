### Что делает сервис
Предоставляет api для непочтовых потребителей - виджеты на морде, каунтер непрочитанных писем в шапке. Самый топовый метод счетчики,но есть еще связанные со списком папок, настройками, информацией об аккаунте. Авторизация происходит по куками или tvm-тикетам.
Разные версии api живут в разных приложениях:
**webmail-api-v1** - /api/v[1-2]/* - основная нагрузка приходит сюда.
**webmail-api** - при переезде в Deploy переехал в [intapi](./intapi.md)

### Импакт на пользователей от отключения сервиса
Не будут работать соответствующие функциональности для внешних сервисов.

### Импакт остановки тачки/ДЦ
Балансер перераспределяет нагрузку между другими тачками/дц, внешнего импакта наблюдаться не должно.

### Особенности деплоя
Для webmail-api-v1 катим canary, смотрим на графики. При отсутствии проблем катим основные компоненты - их 4 поделены по ДЦ.

### Критичные метрики
####5xx

####4xx
Количество 4xx кодов ответа.

### Куда ходит
Во многие почтовые бэкенды.
