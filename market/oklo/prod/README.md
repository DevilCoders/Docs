**Основные правила работы в дерриктории dev:**

* В папке должны лежать только продовые скрипты, те те, которые уже проверены, работают стабильно. Тут не должно быть тестоывх скриптов.
* В корень дирректории складировать файлы **ЗАПРЕЩЕНО!!!**
* Работать необходимо в своей папке, которая повторяет название проекта или направления.
* В папке проекта **НЕЛЬЗЯ** творить любой треш и угар!
* Все файлики, которые находятся в директории должны назваться так же, как и фалик в YT.
* Структура папки dev в аркадии должна быть полностью зеркальна дерриктории //home/market/production/oklo/ на YT, за исключением папки users на YT, в аркадии ее аналог - dev. 

**Как запустить продовый скрипт (подсмотрено у аналитиков доставки)**

1. Создать новый файл и скопировать запрос из YQL-интерфейса, в названии указать название результирующей таблички + расширение .yql
2. Для запуска запроса необходимо создать кубик в Нирване по примеру (тут продовый воркфлоу и реакция которая его запускает): https://reactor.yandex-team.ru/browse?selected=11257257
    * Нужно будет единоразово получить токен Аркадии и положить его в секретницу: https://oauth.yandex-team.ru/authorize?response_type=token&client_id=5c407aafc5c242948b532842a9a07da6
    * Далее к графу можно прикруть реакцию и настроить регулярный расчет (см. пример выше)
    * Чтобы реакция выплевывала артефакт, нужно заполнить графу "Parametrs" -> "On sucsess" в настройках рекации (пример: https://reactor.yandex-team.ru/browse?selected=10269726)
3. Тык кнопочку "Commit" -> "Send to review"
4. Во вкладке "Pull Requests" необходимо сделать review и подтвердить мердж тыкнув "ship"
5. Дождать завершения мерджа
6. Profit!!!
