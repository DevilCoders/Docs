Пример использование тулзой deplate для создания и изменения стейджа

Для начала нужно тестовой приложение, можно воспользоваться уже готовым собранным из `example-app/`: [2425621076](https://sandbox.yandex-team.ru/resource/2425621076/view)
Приложение должно отвечать на ручку /ping

Далее по пунктам:
1. Собираем тулзу, для этого просто запускаем `ya make`
2. Генерим дефолтный набор параметров для нашего приложения:
   ```shell
   cmd/deplate/deplate values generate --abc marketito --environment unstable --name isonami-app --network-macro _SEARCHSAND_ --sas-pods 1 --hdd 10 --command /app/bin/example-app example-app.yaml
   ```
   Где:
   * `--abc` абц к которому относится сервис, простовляется внутри спеки и может использоватся для индификации сервиса
   * `--environment` один вариант из `unstable/testing/prestable/production`, так же проставится в спеке и прокинется в запускаймые процесы
   * `--name` имя приложения, используется в разных местах шаблона, к примеру используется в качестве имени деплой юнита
   * `--network-macro` сетевой макрос в котором запустится стейдж, для эксперементов можно использовать яндексовую песочницу `_SEARCHSAND_`
   * `--sas-pods` количество подов в `sas`, также можно указать `vla` или `man`
   * `--hdd` объем `hdd` вольюма выделеного под сервис

   Есть еще дополнительные параметрыб их можно увидеть в `--help`

   Даная команда сгенерит файл параметров(values) `example-app.yaml`, это yaml файл который можно править соответственно с [go json структурой](https://a.yandex-team.ru/arc_vcs/market/sre/library/golang/deplate/values/values.go)
   Так же в этом указан основной шаблон(template.root) на который будут наложены как "слои" модифицирующие шаблоны(template.modifiers). В template.tools указывается какие тузлы из tools/ можно импортировать в указанных шаблонах
3. На основе полученых values можно создать стейдж выполнив команду
   ```shell
   cmd/deplate/deplate stage create dev_isonami_deplate-service example-app.yaml --project market-sre -t -l isonami-app-app:2592681523
   ```
   Где:
   * `dev_isonami_deplate-service` имя стейджа, должно быть уникальным
   * `example-app.yaml` имя файла с values
   * `--project` имя проекта в котором создать стейдж, нужны парава овнера или мейнтейнера в этом проекте, можно создать личный проект если нет подходящего
   * `-t/--temporary` создаст стейдж в `tmp` квоте, который надо будет удалить в течении нескольких недель
   * `-l isonami-app-app:2592681523` задает версию сендбокс ресурса с приложением, имя ресурса зависит от заданого в values name. Если запустить без этого аргумента, то в выводе ошибки будет пример как указать нужное имя
4. Обновление стейджа после изменения values делается комадой:
   ```shell
   cmd/deplate/deplate stage update dev_isonami_deplate-service example-app.yaml
   ```
   Она так же принимает параметр `-l` если нужно обновить какой-то ресурс
