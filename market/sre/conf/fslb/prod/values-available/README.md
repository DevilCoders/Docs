Naming conventions
```
{NUM}-{SERVICE}-{ENV}-{SPECIALS}.yml
```

```{NUM}``` Можно выбрать любой свободный номер старше 10 (до 10 - резерв для служебных конфигов). Изначально номера конфигов совпадали с историческими номерами конфигов в nginx.

```{SERVICE}``` Название сервиса (должно быть частью service_name)

```{ENV}``` testing, prestable, production etc

```{SPECIALS}``` Для дополнительных описаний  характеристик или профилей сервиса, например noxsl, plain-http.
