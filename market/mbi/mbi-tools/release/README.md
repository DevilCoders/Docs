## release/make_release.sh

Автомерж тикетов в текущую ветку. 

Есть 2 режима:

1. Задаём переменную ISSUES, в которой перечислены тикеты через пробел.
2. Задаём переменную SESSIONID, содержащую вашу куку Session_id к стартреку. Теперь аргументом нужно указать релизный тикет.

Пример 1
```
export ISSUES="MBI-1 MBI-2 MBI-3 MBI-4"
$MBI_TOOLS_PATH/release/make_release.sh
```

Пример 2
```
export SESSIONID=<ваш Session_id>
$MBI_TOOLS_PATH/release/make_release.sh MBI-20650
```

Во втором режиме скрипт ищет в стартреке fix version у указанного тикета, и вытаскивает по нему список тикетов.
Далее скрипт спрашивает у гита ветки, ещё не смерженные в текущую ветку и содержащие в названии номера тикетов.
После этого по одной мержит найденные feature-ветки и останавливается в случае конфликта, который нужно руками зарезолвить и закоммитить фикс.
После резолва конфликта перезапустить скрипт, он продолжит с того места, на котором остановился (засчет выборки только несмерженных веток).

По желанию, после этого можно выполнить `git rebase master`, чтобы убрать merge коммиты. Правда после этого нельзя будет перезапускать скрипт, т.к. все feature-ветки снова будут несмерженными.
