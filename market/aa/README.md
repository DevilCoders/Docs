#### Как проверить решение кандидата
Для каждой задачи должен быть ProblemNameCandidateTest. Достаточно вставить код кандидата, поправить сигнатуру и ошибки компиляции и запустить его.

#### Что должно быть для задачи
1) отдельный package в сорцах и тестах
2) Интерфейс решения 
3) Несколько написанных решений
4) Базовый класс с тест-кейсами
5) Наследники от базового класса для конкретных решений задачи
6) Задизейбленный тест-класс, куда можно вставить решения кандидата

#### На что стоит обратить внимание
1) Осторожнее с рандомом в тестах и базовых решениях. Очень желательно инициализировать рандомный генератор конкретным seed'ом, чтобы тесты были повторяемыми
2) Для больших параметризованных тестов желательно задать `@ParameterizedTest(name = "{index}")`, по умолчанию в имя теста выводятся все параметры
