## CPA-бекенд { #cpa-backend }

В названиях чатов -- синоним Чекаутера. Почему: так получилось. Никто не знает деталей.

## Агентское вознаграждение { #agents-comission }

Агентское вознаграждение, оно же комиссия за эквайринг.
Плата, которую Маркет взимает с партнера за прием и перечисление денег от покупателя.
См. также [внешнюю документацию](https://yandex.ru/support/marketplace/rates.html).
Не путать с премиями агентствам и fee (плата за размещение товара на Маркете).

## Белый (контур)

В данном случае имеется в виду система маркета, которая была изначально. Работает по принципу сравнений товаров из
разных магазинов. Основной доход с рекламы по модели CPC (cost-per-click) - Маркет получается вознаграждение за каждый переход (клик по товарной карточке) на сайт интернет магазина.

## Биллинг (обиливние)

Процесс, при котором подсчитывается стоимость всех услуг, оказанных Яндексом магазинам. После того
как по всем типам услуг будет посчитана стоимость, результат будет направлен в службу Большого Биллинга Яндекса (Баланс).

## Джоба (она же жоба, она же job-a)

Является по сути расписанием запусков для работы каких-либо бинов классов из проекта.
Посмотреть на активные джобы можно через TMS.

## Дистрибуция

Услуга которую оказывают блогеры, рекламируя товары Маркета, как дополнительный канал рекламы.

## Дроп-офф { #drop-off }

Передача партнером товара на пункт выдачи заказа (ПВЗ), откуда он будет доставлен в сортировочный центр для дальнейшей доставки покупателю.

## Премии агентствам { #agency-reward }

Премии, которые Маркет выплачивает агентствам, если они выполняют цели по важным показателям.
Не путать с агентстким вознаграждением.

## Синий (контур)

Система продаж маркета, которая раньше была Беру. Маркетплейс. Основной смысл в том, что через нее можно приобретать товары.
Поставляемые услуги делятся на несколько типов (FBY, FBY+, DBY...), о которых можно прочитать отдельно в соглашениях. В Синем также есть продвижение (реклама), по модели CPA.

## Тарифница { #tariff-service }

Сервис для создания и хранения тарифов на услуги Маркета

## TMS (Task Management Service)

Менеджер задач для джобов. Через него можно задавать новое расписание джобам, удалять джобы и т.д.

## TPL (Third Party Logistic)

TPL, он же 3PL, он же Биллинг Логистики. Компонент, отвечающий за три биллинга: Курьерский, СЦ и ПВЗ

## ПВЗ
Пункт выдачи заказов

## БПВЗ
Брендированный ПВЗ

## GMV
Gross Merchandise Volume, Общий объем заказов

## ДС

Сокращенно - дополнительное соглашение. Подписывается с партнером для применения каких-то нестандартных условий, которые, например, отличаются от текущей оферты.

## XDOC

## Модели подключения

Также см. официальную документацию https://yandex.ru/support/marketplace/models.html

### DBS (Delivery By Seller)
Магазин самостояетельно доставляет заказы покупателю. Маркет работает только витриной.

### DSBS (DropShip By Seller)
Старое название для FBS модели. Dropship - поставка, имеется ввиду поставка на СЦ (НЕ ПОКУПАТЕЛЮ! Собственная доставка покупателю это - DBS).

### FBS (Fulfillment By Seller)
Магазин сам везет заказ на сортировочный цент (СЦ)

### FBY (Fulfillment By Yandex)
Товары лежат на складах Яндекса Маркета, Яндекс сам везет их на СЦ, групирует в заказы и отвозит покупателю.

### FBY+ (Fulfillment By Yandex+)
Новое название для кроссдок. доставка поставщиком товара на склад к заказу, дальше все аналогично FBY;
