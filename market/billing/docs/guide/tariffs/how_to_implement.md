# Как заводить тариф

{% note alert %}

До непосредственного заведения тарифа **необходимо** чтобы были выполнены **все** остальные шаги [регламента](../../regulations/tariffs/tariffs.md)

{% endnote %}

Для заведения тарифа всегда __строго__ используется именно ДС.  
Сам же ДС может содержать множество услуг одновременно - на каждую комбинацию "услуга + модель работы" нужен отдельный тариф.

В интерфейсе тарифницы необходимо нажать на кнопку "Создать +" на любой странице и в выпадающем меню выбрать "Черновик тарифа".

* ![alt Создать](../../_assets/images/tariffs/head_create.jpeg "Создать")
* ![alt Создать черновик](../../_assets/images/tariffs/create_draft.jpg "Создать черновик")

Появится следующее окно:

* ![alt Редактирование информации](../../_assets/images/tariffs/editing_main_info.jpg "Редактирование информации" )

Сразу можно увидеть что много полей подсвечено красным - все они обязательные.  
В рамках подтверждающего тикета нужно указать только ключ самой задачи. При этом, можно указывать только очередь MARKETBILLING (то есть, тикет должен быть заведен по форме).

Тип услуги для черновика.  
После выбора этого поля появится блок "Мета", в который нужно будет добавлять непосредственные данные услуги.

* ![alt Редактирование меты](../../_assets/images/tariffs/creating_meta.jpg "Редактирование меты")

Тип модели работы партнера.  
Реально из этого поддерживаются:

* FBS (fullfilment_by_seller). Бывший Дропшип.
* FBY (fullfulment_by_yandex). Бывший Фуллфилмент.
* FBY+ (fullfilment_by_yandex_plus).
* DBS (dropship_by_seller).

Дата от (включительно).  
Дата до (не включительно) - не обязательное поле, если тариф "вечный".  
**Важно** В ДС обычно пишется дата включительно. Как следствие нужно указывать на 1 день больше.

Партнер - имеет саджест по идентификатору партнера.

После заполнения этих полей нужно заполнить "Мету" услуги, нажав кнопку "Добавить".  
Каждая услуга может содержать несколько полей в "Мете" (например несколько строчек в ДС) - их нужно указывать __все__.

После всех манипуляций можно проверить тариф, нажав кнопку "Валидировать".  
Если валидация прошла успешно, то нужно нажать кнопку "сохранить".

* ![alt Валидация и сохранение](../../_assets/images/tariffs/validation_and_saving.jpg "Валидация и сохранение")

{% cut "Если появилась ошибка" %}

Например, в правом верхнем углу после валидации/сохранения могут появится такие ошибки:  

* ![alt Отсутствует подтверждающий тикет](../../_assets/images/tariffs/err_ee_approval_ticket.png "Отсутствует подтверждающий тикет")  
* ![alt Есть пересечения с другими тарифами](../../_assets/images/tariffs/err_ee_intersection.png "Есть пересечения с другими тарифами")

Внимательно прочитайте сообщение об ошибке. Проверьте те данные (поля, связанные тарифы и т.д.), которые в нем указаны. Перепроверьте последовательность самого заведения тарифа.  
Если ничего не помогает, то обратитесь к [дежурному](https://abc.yandex-team.ru/services/market_billing/duty/)

{% endcut %}

После успешного сохранения черновика нужно:

1) Перейти во вкладку "Черновики"
2) Скопировать ссылку на сам черновик
3) Приложить эту ссылку в тикет на подтверждение
4) Призвать коллегу, который имеет право подтверждать тарифы (указаны в регламенте).

![alt Ссылка на тариф](../../_assets/images/tariffs/draft_url.png "Ссылка на тариф")

Аппрувер прийдет в тикет по призыву и проверит соответствие указанных в черновике данных относительно самого тикета и подписанного ДС.  
Если все корректно и не вызывает каких-либо дополнительных вопросов, а все остальные действия выполнены по регламенту, то тариф будет подтвержден.
