# Комиссия за доставку товара клиенту (услуга delivery_to_customer)

## В договоре/ДС

ДС могут содержать в себе информацию для разных моделей работы. Тогда нужно заводить тарифы для **каждой из моделей по отдельности**.

* Стандартный вариант для разных моделей работы  
    ![alt Пример договора](../../../_assets/images/tariffs/services/delivery_to_customer/agreement_example_1.png)

     {% note alert %}

     В самом ДС указывается информация именно для __доставки__,
     но применяется она и к услуге "delivery_to_customer_return".
     Таким образом, нужно заводить обе услуги сразу.

     {% endnote %}

## В админке Тарифницы

В рамках Меты нужно заводить отдельные поля в рамках следующего окна:  
![alt Добавление меты](../../../_assets/images/tariffs/services/delivery_to_customer/meta_addition.png)

При этом, обратите внимане на следующие вещи:

* Relative/absolute значение - процент или фикс.
* Ordinal - порядоквый номер меты, по которой будет выбираться нужная ставка. С его помощью можно более корректно распределять ставки от частного к общему случаям.
* Статус заказа - обычно это всегда "delivered", если вдруг в ДС не указано __в явном виде__ иное.
* В рамках поля "dimensionsTo" указывается сумма трех измерений в сантиметрах
* В рамках поля "weightTo" указывается вес __в граммах__
* Поля "minValue" и "maxValue" это минимальная/максимальная ставки для выбранной меты (по порядковому номеру и измерениям)
* Поле "priceTo" обычно не используется, но с его помощью можно огрначить стоимость товара, для которого применяется ставка.
