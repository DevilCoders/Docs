# Онбординг

## Общий порядок

1. Перед выходом нового сотрудника кто-нибудь (**кто?**) заводит тикет `Новый разработчик: <login>` (подробнее про тикет см. ниже)
2. В первый день работы новый сотрудник получает этот тикет с заданием "сделай вот это все"
3. Новенький идет по инструкции, отмечает чеклист, записывает в тикет наблюдения
4. Когда заканчивает чеклист &mdash; закрывает тикет
5. Кто-нибудь (**кто?** предположительно lena-san@, sergeitelnov@ или hmepas@) читает тикет. Если по результатам надо что-то доработать в доке &mdash; записывает в тикет <https://st.yandex-team.ru/MARKETBILLING-61> и планирует доработки
6. Кто-нибудь (**кто?**) проводит встречу "порефлексировать онбординг"
