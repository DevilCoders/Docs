# Монорепозиторий фронта Маркета

## Начальная настройка

Для корректной работы требуется провести [начальную настройку](/arc_vcs/market/front/monomarket/wiki/start.md)

## **Важно! Обязательно к прочтению! Дважды!**

* в монорепе **никогда**, **нигде** и **никак** не используется `npm`. Для всех задач в монорепе нужно использовать **только** `arc` и `yammy`, если явно не указано иное.
* прочтите предыдущий пункт ещё 2 раза
* теперь ещё раз, но на этот раз внимательно
* для запуска **дочерних** скриптов в скриптах в `package.json` нужно использовать `yarn run`. Прочтите первый пункт ещё раз. Этот случай как раз то самое "иное".
* **никогда** не используйте вызов `yarn` напрямую из консоли - прочтите первый пункт ещё раз.
* крайне рекомендуется настроить и использовать автодополнение для `yammy` - это решает бОльшую часть проблем с использованием и пониманием синтаксиса (см. [документацию по начальной настройке](/arc_vcs/market/front/monomarket/wiki/start.md))
* для того чтобы не страдать и не решать один и тот же конфликт 20 раз при ребейзе, не плодите коммиты в ветке, а делайте всю задачу в одном коммите, изменяя его с помощью `arc commit --amend`
* в файлы `*.js.flow` нужно руками вносить изменения, автоматически они не собираются.

## Использование

Для управления репозиторием используется консольная утилита `yammy`. Для получения актуальной справки по доступным
командам используйте `yammy --help`.

Для мёржа пулл-реквестов следует использовать специальный кубик в пайплайне.
