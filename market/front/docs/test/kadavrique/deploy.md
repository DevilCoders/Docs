# Деплой

Кадаврик, как уже говорилось [ранее](https://docs.yandex-team.ru/marketfront/test/kadavrique/how), состоит из нескольких частей, процессы деплоя которых отличаются друг от друга.

## Сервер

Деплой серверной части необходим в случае:

- изменений самого кода node.js приложения
- доработок моков, к которым обращаются автотесты, используя боевые хосты кадавра в качестве прокси сервера (все hermione тесты по дефолту используют как раз их)

Релиз кадавра создается автоматически в [ЦУМе](https://tsum.yandex-team.ru/pipe/projects/market-frontech/delivery-dashboard/market-kadavr-cd-arc) после пул реквеста в транк. Далее релиз выкатится автоматически, сопровождает релиз команда инфраструктура фронты Маркета. В случае возникновения вопросов - их можно задать в [чате](https://t.me/joinchat/C5yyIUy9ythghPB6TfsD7w).

## Плагины

В случае изменения логики работы [плагинов](https://a.yandex-team.ru/arcadia/market/front/apps/kadavrique) - необходимо публкиовать новую версию Кадаврика в npm.yandex-team.ru и обновлять зависимости в основном приложении (например в marketfront). Попросить опубликовать и зарелизить можно в [этом чате](https://t.me/joinchat/C5yyIUy9ythghPB6TfsD7w).

## Локальные моки

В случае использования кадавра для локальных моков (например в [testament](https://docs.yandex-team.ru/marketfront/test/testament) тестах) - поход в боевые хосты не используется. Поэтому требуется релиз, аналогичный релизу при изменении плагинов.
