# Ручки маркетинговых кампаний
Ключ в стейте `vendorMarketingCampaigns`.

**Структура стейта**
```js
    list: {} // объект, описывающий маркетинговые кампании
```

При инициализации стейта подмешиваются дефолтные параметры

```js
    approvedBy: vendorId || 0,
    campaignName: 'Привет твоей маме',
    dateFrom: 1690837200000,
    dateTo: 1693515599999,
    serviceType: 'BANNERS',
    status: 'NEED_APPROVE',
    sumCents: 100500,
```

## Получение кампании маркетинговых услуг
`GET /vendors/<vendorId>/marketing/campaigns/<campaignId>`

Вернет данные из `list.{campaignId}`

## Получение списка кампаний маркетинговых услуг
`GET /vendors/<vendorId>/marketing/campaigns`

Вернет данные из `list`

## Создание кампании маркетинговых услуг
`POST /vendors/<vendorId>/marketing/campaigns`

Добавит в стейт новую кампанию, подмешивая к переданным параметрам дефолтные.

Вернет объект
```js
{
    balanceLeft: 100500, // остаток баланса (значение по-умолчанию)
    campaign: {} // созданная кампания
}
```
При этом в параметры можно докинуть `balanceLeft`. Тогда именно это значение будет возвращено в ответе.

Так же в параметрах можно указать `id` создаваемой кампании.
Если не указать, то `id` будет создан функцией `createId` из `utils`

## Изменение кампании маркетинговых услуг
`PUT /vendors/<vendorId>/marketing/campaigns/<campaignId>`

Обновит параметры кампании в стейте по `list.{campaignId}`.

Вернет объект
```js
{
    balanceLeft: 100500, // остаток баланса (значение по-умолчанию)
    campaign: {} // обновленная кампания
}
```
Так же, как и в предыдущей ручке, можно докинуть параметр `balanceLeft`

## Удаление кампании маркетинговых услуг
`DELETE /vendors/<vendorId>/marketing/campaigns/<campaignId>`

Удалит из стейта кампанию по `list.{campaignId}`.

Вернет объект
```js
{
    balanceLeft: 100500, // остаток баланса (значение по-умолчанию)
    campaign: {} // удаленная кампания
}
```
Так же можно докинуть параметр `balanceLeft`.

В объект `campaign` в ответе допишутся дефолтные параметры

## Подтверждение кампании
`POST /vendors/<vendorId>/marketing/campaigns/<campaignId>/approval`

Меняет статус кампании `list.{campaignId}` на `SCHEDULED`, `FINISHED` или `ACTIVE`
в зависимости от сохраненных параметров `dateFrom` и `dateTo`

Можно докинуть в параметры `status`. Тогда именно это значение будет установлено.

Вернет объект подтвержденной кампании, куда допишутся дефолтные параметры
