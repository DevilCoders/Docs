# NearestShop

Сущность для хранения ближайшего склада и магазина для бизнеса.

##Почему не храним в Shop

1. Ближайший склад и магазин может быть уникален для разных координат.
2. Из за того, что может быть уникален решили не добавлять этот признак на стороне репорта,
а ходим и получаем его с отдельным параметром. При таком похоже количество магазинов в бизнесе
   и складов в магазине сокращается до одного, что вызовет StateDivergenceError, поэтому было решено
   вынести это в связующую сущность.
