Многоуровневое состояние отзывов
======

Реализует каскад состояний страницы, при необходимости независимо изменять/сбрасывать состояния.
Используется на КМ, для реализации уровней фильтров отзывов.

Пример пользовательского сценария: открываем фильтры на КМ в отзывах, активируем N фильтров.
При закрытии страницы фильтров без применения, фильтры должны вернуться в состояние, в котором были до входа на них.
При применении фильтров активированные N фильтров должны сохраниться,
и при следующем открытии/закрытии страницы не сбрасываться.

Используются три уровня состояния:
* CURRENT - отражает изменения активного в данный момент фильтра
* CONFIRMED - выбранные пользователем, но не примененные фильтры
* APPLIED - примененные в данный момент фильтры.

Селекторы делятся на выбирающие один конкретный уровень, либо мержащие уровни вместе.
При мерже значения на более "высоких" уровнях (имеющие меньший индекс в LEVELS_PRIORITY) имеют больший приоритет.
Т.е. каскад стейта мержится сверху вниз.

Можно представить прототипное наследование CURRENT -> CONFIRMED -> APPLIED
(при отсутствии свойства в CURRENT будет попытка найти его в CONFIRMED и, затем, в APPLIED).

Пример:
```javascript
{
    current: {
        filter1: 'a',
    },
    confirmed: {
        filter1: 'b',
        filter2: '123',
    },
    applied: {
        filter1: 'c',
        filter3: '321',
    },
}

/**
* {
*     filter1: 'b',
*     filter2: '123',
*     filter3: '321',
* }
*/
mergeStateLevels(collection, STATE_LEVEL.CONFIRMED)

/**
* {
*     filter1: 'a',
*     filter2: '123',
*     filter3: '321',
* }
*/
mergeStateLevels(collection, STATE_LEVEL.CURRENT)
```
