Нужен для изменения параметров вложенного виджета на строне обработки node сервером фронта,
т.к. в каких то случаях (дырки динам. морды) виджет подставляется темплатором
и нет возможности его доредактировать силами cms (к примеру через вкладку cms схем "Свойства по путям")

Варианты реализации серверной обработки:

Отдельную ветку обработки в CmsLayout под процессинг (менять форматы внутренних узлов)
- узаконенный общий костыль

Заменять на уровне фильтрации (filterWidgets(ctx))
- Из плюсов: там сейчас уже скрытие сделано по hideCriteria
- узаконенный общий костыль на более низком уровне (т.к. логика лейзи не на этом уровне обработки требуется)

Виджет прокидывающий/рендерящий напрямую childrens
- Локальный костыль

Наш случай полагаю локальный,
и тут не нужно вводить общих костылей (костыль - т.к. позволят неявно править логики/данные вложенных виджетов),
поэтому текущим виджетом делаю.
