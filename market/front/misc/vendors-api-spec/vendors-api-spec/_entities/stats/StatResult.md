# Сущность "StatResult"

Результат запроса статистики по вендору

### Поля:
  - **statId** `<String>` - идентификатор типа статистики, по которой сделана выборка (например, `RECOMMENDED_CLICKS`)
  - **yUnit** `<String>` - идентификатор единиц по оси `Y` в графике
  - **filters** `<Map[String, Object]>` - мапа фильтров, использованных для выборки 
  - start [`<DateTime>`] - дата первого элемента в графике (может отсутствовать, если нет данных)
  - end [`<DateTime>`] - дата последнего элемента в графике (может отсутствовать, если нет данных)
  - summary `<Number>` - агрегат всех значений за выбранный период (например, сумма всех трат,
   или среднее значение соотношения кликов). Может отсутствовать, если в графике нет значений.
  - chart [`<Chart>`] - график статистики. Может отсутствовать, если нет данных за выбранный период.
  
### Возможные значения `yUnit`:
- `MONEY` - денежные единицы (любые целочисленные значение, в "копейках")
- `PERCENT` - доля в процентах (целочисленные значения в диапазоне `[0, 10000]`, проценты с 2-мя знаками после запятой)
- `CLICKS` - клики (количество)
- `SALES` - заказы (количество)
- `POSITION` - позиция в рейтинге/каталоге (целочисленные значения в диапазоне `[0, 10000]`)
  
### Возможные ключи в `filters`:
- `VENDOR_ID` - идентификатор вендора
- `CLIENT_ID` - идентификатор клиента
- `MODEL_ID` - идентификатор модели на Маркете
- `CATEGORY_ID` - идентификатор категории на Маркете
  
### Примечания
- По оси `X` в графике пока всегда будет дата-время в миллисекундах,
так что поля `xUnit` нет.
- **Время в графике (значения `X`) всегда присылается в UTC**
- Все проценты присылаются с точностью 2 знака после запятой, но в целочисленном формате,
т.е. значение `950` означает `9.5%`, а значение `1705` означает `17.05%`.
- Айди вендора вынесено в мапу опциональных фильтров, потому что потенциально
может существовать статистика, не зависящая от конкретного вендора
