# Товар у вендора с информацией о ставке и исторической информацией и прогнозу кликов
   
Представляет собой базовую информацию о товаре, историческую информацию и прогноз по показам и кликам

### Поля:
  - **id** `<Number>` - идентификатор товара
  - **name** `<String>` - название товара
  - minPrice `<Number>` - минимальная цена
  - maxPrice `<Number>` - максимальная цена
  - shopsCount `<Number>` - число магазинов продающих товар
  - recommendedShopsCount `<Number>` - число рекомендованных магазинов продающих товар
  - bid `<Number>` - текущая ставка
  - newBid `<Number>` - применяющаяся ставка (не передаётся в случае, когда bidStatus == APPLIED)
  - minBid `<Number>` - минимальное значение ставки
  - bidStatus [`<BidStatus>`](/_entities/modelbids/BidStatus.md) - статус применения ставки
  - bidError [`<BidError>`](/_entities/modelbids/BidError.md) - код ошибки применения ставки (передаётся в случае, когда bidStatus == ERROR)
  - bidForPlace `<Map<Number, Number>>` - необходимая ставка чтобы занять место
  - categoryId `<Number>` - идентификатор листовой категории товара
  - categoryPath `<Array[String]>` - краткий "путь" категории (не более 4-х последних элементов)
  - image [`<Image>`](/_entities/Image.md) - изображение товара
  - historicalInfo `<Array[Object]>` - историческая информация по показам и кликам
    - **periodStart** `<DateTime>` - дата начала периода
    - **periodEnd** `<DateTime>` - дата конца периода
    - clicksCount `<Number>` - количество кликов за период
    - showsCount `<Number>` - количество показов за период
    - ordersCount `<Number>` - количество заказов за период
       
### Примечания
  - `bidForPlace` содержит только достижимые позиции для модели. 
  - Если несколько позиций достижимо с одной и той же ставкой, то возвращается только максимальная позиция
  - `historicalInfo` возвращается отсортированным по полю `periodDate` в естественном порядке
