# Сущность "ModelParameter"

Параметр вендорской модели на Маркете

### Поля:
  - **paramId** `<Integer>` - идентификатор праметра 
  - skuMode [`<SkuMode>`](/_entities/model-editor/sku/SkuMode.md) - тип SKU (присутствует для SKU-параметров)
  - customValue `<Boolean>` - возможно ли свободное заполнение значений параметра (присутствует для SKU-параметров)
  - name `<String>` - имя параметра
  - unit `<String>` - единица измерения параметра
  - hint `<String>` - подсказка по заполнению параметра
  - type `<String>` - тип параметра (boolean | enum | numeric | text) . Отдаём всегда, на вход не принимаем.
  - mandatory `<Boolean>` - является ли параметр обязательным для заполнения
  - editable `<Boolean>` - является ли параметр редактируемым.
  - multivalue `<Boolean>` - является ли параметр мультиполем (возможно ли несколько значений)
  - value `<Boolean|String|Number|Array[Object]>` - значение(я) параметра. В случае multivalue==true, приходит массив значений.
  - minValue `<Number>` - минимальное возможное значение (для числовых параметров)
  - maxValue `<Number>` - максимальное возможное значение (для числовых параметров)
  - picker [`<Array[Image]>`](/_entities/Image.md) - пикеры для значений параметров. В случае multivalue==true - возможно несколько изображений.
   Всегда должно быть **либо** столько же элементов, как в `value`, **либо** пустой список (Если в `value` лежит объект вместо массива - значит тут массив с одним элементом). 
  - pickersAllowed `<Boolean>` - Если тип этого параметра позволяет устанавливать пикеры значениям - `true`.
  Если нет - `false`
  - options `Array[Object]` - возможные значения параметра (только для type == enum) 
    - **id** `Number` - идентификатор значения параметра 
    - **name** `String` - имя значения параметра 
  - tab `<String>` - имя группы параметров
  - block `<String>` - имя подгруппы параметров внутри группы

### Примечания
  - Значения не редактируемых полей перед отправкой заявки в МБО перезатираются актуальными значениями параметров модели.
  - Новые пикеры от юзера должны загружаться в ту же ручку аплоада, что и картинки моделей, и отправляться в заявке в виде файла.
  Уже существующие файлы, полученные от нас в виде УРЛа - обратно посылать тоже в виде УРЛа.
  Тип [`<Image>`](/_entities/Image.md) позволяет отправлять картинки и УРЛом и файлом. 
  - `pickersAllowed` - этот параметр показывает, если у параметра нет **и не может быть** пикеров. Возможны 2 сценария:
    1. Если пришёл параметр и у него `pickersAllowed==true` - значит пикеры можно и добавлять новые и менять текущие.
    2. Если пришёл параметр и у него `pickersAllowed==false` - значит у неё не должно быть пикеров и их нельзя добавлять.
     **Если вдруг с ней пришли пикеры, то это левак и их можно игнорить.** 
  - Каждое значение параметра соответствует одному пикеру - т.е. в поле `picker` всегда должно быть либо столько же значений,
  как в `value`, либо - ни одного. Пикеры и значения соотносятся **по-порядку** (по индексу) - т.е. в случае с полями:
  `{value: [v1, v2, v3], picker: [p1, null, p3]}` - первое и последнее значение параметра имеют пикеры, а второе значение - не имеет.
