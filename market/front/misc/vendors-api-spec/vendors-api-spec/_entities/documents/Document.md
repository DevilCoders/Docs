# Сущность "Document"

Документ, состоящий из набора файлов

### Поля:
  - id `<Number>` - идентификатор документа
  - name `<String>` - имя файла (под которым он был загружен на сервер)
  - createdAt [`<DateTime>`] - дата создания документа на сервере 
  - expireAt [`<DateTime>`] - дата окончания действия документа 
  - files [`<Array[FileMeta]>`](/_entities/documents/FileMeta.md) - набор мета-данных файлов,
   принадлежащих этому документау

### Примечания
  - При создании документа:
    - Если указано поле `id`, то все остальные поля можно не указывать.
    - Если поля `id` нет, то обязательно указать имя и список файлов. Дата протухания - опциональна.
    - Подробнее описано тут: [Правила работы с документами](/_rules/documents/DocumentRules.md#postинг)
  - При получении документа с сервера, гарантированно присутствуют все поля, кроме даты протухания.
    Её может не быть, и тогда документ считается бессрочным.
  - Если это документ в оферте, то **параметр `name` может содержать только
    определённый набор значений**! Подробнее описано тут: 
    [Правила работы с документами в оферте](/_rules/documents/OfferDocumentRules.md)

### Примеры:
```
{
  "id": 42,
  "name": "Some document",
  "createdAt": "2017-05-22T14:12:01Z",
  "expireAt": "2018-01-01T00:00:00Z",
  "files": [
    {
      "uploadedAt": "2017-05-22T13:32:22Z",
      "name": "scan1.png",
      "id": "5c155ed6-b050-4981-b263-9a2cf0c7109b",
      "size": 49333
    },
    {
      "uploadedAt": "2017-05-22T13:44:12Z",
      "name": "scan2.png",
      "id": "c7792fec-8c65-48a3-a63d-1b94fd01cd75",
      "size": 5544
    }
  ]
}
```
