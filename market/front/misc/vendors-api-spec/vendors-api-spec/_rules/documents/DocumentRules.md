## Правила работы с сущностью [`<Document>`]

### POST'инг

Создать документы можно передавая необходимые данные в ручки типа [`POST /entries`] и [`POST /vendors`].
Т.е. ручки устроены так, что в них можно передать хоть уже существующий документ,
и тогда он будет переиспользован; хоть новый, который будет при этом создан.

#### Пример

Допустим у нас уже есть документ №42. Тогда вызвав ручку [`POST /entries`],
мы можем передать её такие данные:
```
{
  ...,
  "documents": [
    {
      "id": 42,
      "name": "SOME_DOCUMENT",
      "files": [
        {"id": "5d5ea327-eb15-4f12-a058-8eb1e8ca9002"},
        {"id": "2d26da8a-c84a-4083-a356-739d3d6b116e"}
      ]
    },
    {
      "name": "SOME_OTHER_DOCUMENT",
      "files": [
        {"id": "fa404dfd-ec91-4805-9115-aa845f674ed9"},
        {"id": "23fcdd54-6efe-4dc5-a81c-04baffe02690"}
      ]
    }
  ]
}
```

Документ с полем "id" будет проверен на существование с ошибкой в случае провала;
документ без поля "id" будет создан, и ему будет присвоен новый индентификатор, и в ответе
он будет содержать поле "id".

#### Важно

Если в документе присутствует поле "id", то **все остальные поля игнорируются**.
А значит вот такие документы воспринимаются идентичными, когда присланы на сервер:
```
{
  "id": 42,
  "name": "SOME_DOCUMENT",
  "files": [
    {"id": "5d5ea327-eb15-4f12-a058-8eb1e8ca9002"},
    {"id": "2d26da8a-c84a-4083-a356-739d3d6b116e"}
  ]
}
```
```
{
  "id": 42
}
```
