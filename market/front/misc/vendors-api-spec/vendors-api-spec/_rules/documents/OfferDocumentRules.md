## Правила работы с сущностью [`<Document>`] для оферты

Документы, передаваемые в ручки [`POST /entries`] и [`POST /vendors`],
являются частью проекта "оферта". На работу с такими документами
накладываются дополнительные ограничения:

### Создание документов
Имя документа в оферте должно совпадать с одним из этих вариантов:
1. `TRADEMARK_CERTIFICATE` (`Свидетельство на торговый знак`)
2. `TRADEMARK_LICENSE` (`Лицензионный договор`)
3. `TRADEMARK_LICENSE_EXCERPT` (`Выписка из лицензионного договора`)

На данный момент проект "оферта" поддерживает только 3 фиксированных
типа документов, и нет смысла давать пользователю возможность ввести
любую строку в качестве имени документа.

Фронт должен давать пользователю выбор из 3-х человекочитаемых
названий документов, и присылать на бекенд одно из 3-х enum-значений.
В документах, полученных **от** бекенда так же всегда будет одно
из перечисленных 3-х enum-значений, которые надо будет заменить
на человекочитаемое, для отображения пользователю.

#### Ошибки

##### Прислан документ с невалидным для оферты именем
  - statusCode: `400`
  - code: `UNEXPECTED_DOCUMENT_NAME`
  - message: `Illegal document name: '{name}'! Expected one of: [TRADEMARK_CERTIFICATE, TRADEMARK_LICENSE, TRADEMARK_LICENSE_EXCERPT]`
