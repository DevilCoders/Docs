# Changelog
Все существенные изменения в компоненте будут задокументированы в этом файле.

Формат файла основан на соглашении [Keep a Changelog](http://keepachangelog.com/en/1.0.0/), а сам проект использует [Семвер](http://semver.org/spec/v2.0.0.html).

##
### Изменено
- Перевод кода на тайпскрипт

## 1.2.4 — 2020-03-27
### Добавлено
- Экспорт функции `transliterate`

## 1.2.3 — 2019-09-18
### Изменено
- Обработка случая, когда у фильтра типа `enum` нет поля `values`.

## 1.2.2 — 2019-09-16
### Изменено
- `babel-plugin-lodash` отключен для cjs сборки – теперь используется только для esm.

## 1.2.1 — 2019-09-10
### Изменено
- `ambar` перенесен в `peerDependencies`
- flow-fix в методе `getSeoFilter`

## 1.2.0 — 2019-09-09
### Изменено
- Удалена завимость от `lodash/fp`, вместо него теперь используется `ambar` [[1]](https://github.yandex-team.ru/market/ambar).

## 1.1.1 — 2019-02-04
### Добавлено
- Указали `module` точку входа в `package.json` для современных версий Node.js [[1]](https://webpack.js.org/guides/author-libraries/#final-steps) [[2]](https://github.com/dherman/defense-of-dot-js/blob/master/proposal.md).

## 1.1.0 — 2019-01-25
### Добавлено
- ESM-версия модуля.

## 1.0.3 — 2018-12-12
### Изменено
- Сделал исключение в транслитерации для имени компании `Яндекс` при генерации слагов в `getSlug(...)`.

## 1.0.2 — 2018-11-19
### Добавлено
- Дефиниции Flow-типов для `lodash`.

## 1.0.1 — 2018-10-16
### Добавлено
- Поддержка `veendor`,
- дефиниции Flow-типов теперь доступны из NPM-пакета в файлах `.js.flow`.

### Изменено
- Обновил `@yandex-market/codestyle` до версии `2.10.3`.

## 1.0.0 — 2018-10-19
### Изменено
- Переименованы функции публичного API: `canonical -> getSlug`, `metaPagerUrls -> getMetaPagerUrls`,
- функция `getSlug` теперь возвращает строки длиной 1000 символов максимум.

## 0.2.6 — 2018-10-15
### Добавлено
- Поддержка диакритических символов в верхнем регистре (забыли поддержать в 0.2.5).

## 0.2.5 — 2018-10-04
### Добавлено
- Поддержка диакритических символов в нижнем регистре, например:

  ```js
  canonical('fließend flüssig'); //=> 'fliessend-flussig'
  ```

## 0.2.4 — 2018-08-17
### Удалено
- Обнаружились проблемы с включением browserify-бандла в webpack-бандл, так что откатили рабочий код до состояния версии `0.2.2`.

## 0.2.3 — 2018-08-14
### Добавлено
- Поддержка браузерной среды исполнения:
  - пакет теперь собирается в цельный бандл,
  - использует `lodash` вместо `lodash/fp`,
  - тестируется Karma'ой.

## 0.2.2 — 2018-07-16
### Изменено
- Фильтр `megasale` внесён в чёрный список ID фильтров, которые никак не влияют на построение SEO-заголовков функцией `compileTitle`.

## 0.2.1 — 2017-12-13
### Добавлено
- Чёрный список ID фильтров, которые никак не влияют на построение SEO-заголовков функцией `compileTitle`. Например: `deliveryincluded`, `local-offers-first`, `onstock`.

## 0.2.0 — 2017-12-01
### Добавлено
- Проверка синтаксиса согласно `@yandex-market/codestyle`.

### Изменено
- Интерфейс модуля, как того требовал кодстайл,
- теперь `compileTitle` возвращает пустую строку, если нет ни рецептов, ни фильтров.

## 0.1.3 — 2017-11-21
### Добавлено
- Опция `options.addShopName` функции `compileTitle`, позволяющая отключить добавление названия магазина к заголовку.

## 0.1.2 — 2017-10-27
### Изменено
- Теперь `metaPagerUrls` правильно хендлит CGI-параметры в аргументе `request`.

## 0.1.1 — 2017-10-26
### Добавлено
- Создан NPM пакет на основе хелперов из market/mobile.

### Изменено
- Код переписан с использованием Flow;
- теперь `compileTitle` «деградирует» третий компонент заголовка, когда есть фильтр без SEO-сведений;
- сигнатура `metaPagerUrls` изменена для более удобного вызова из Десктопа.
