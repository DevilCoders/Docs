# rules

В этом каталоге находятся правила проверки критических изменений в swagger-схемах.

В каждом файле находится название правила, функция получения списка найденных
критичных изменений по diff'у между swagger-схемами и функция получения
человеко-понятного отчёта по списку найденных критичных изменений.

## Тестирование

Тестирование правил происходит в src/test в рамках сравнения двух больших схем.
Таким образом, для проверки каждого правила в схемах есть
отдельные кусочки со своими путями-параметрами. Один большой прогон двух
схем позволяет не только проверить каждое правило, но и проверить взаимодействие
и ложные срабатывания нескольких правил на одной большой схеме, чего нельзя
добиться, если тестировать каждое правило по-отдельности.
