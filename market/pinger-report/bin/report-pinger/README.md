# Разработка и debug REPORT PINGER-а

Чтобы удобано разрабатывать report pinger нужно поднять его локальную копию.

0. Скомпилируйте report-pinger.
1. Создайте файлик ./var/www/yandex/report-pinger/nanny_conf.json и добавьте в него такое содержимое: {} (просто 2 скобки -- пустой json-словарь), иначе следующая команда будет падать.
2. [Получите Nanny OAuth token](https://nanny.yandex-team.ru/ui/#/oauth). И экспортируйте переменную окружения ```export NANNY_TOKEN=AQAD...```.
3. Сгенерируйте файл report.json - файл в котором содержится информация о бэкендах репорта:

```
./report-pinger --cron --prefix . --testing
```

(посмотрите все опции report-pinger-а - там есть возможность прикинуться не только тестингом, но возможно у вас не хватит дырок)

3.
Запустите devserver:
```
./report-pinger --devserver-port 1337
```
Откройте в браузере http://HOSTNAME:1337/

4. Если вы делаете изменения связанные со сбором данных с хостов, то не забывайте повторять п.1, чтобы перегенерировать report.json.
