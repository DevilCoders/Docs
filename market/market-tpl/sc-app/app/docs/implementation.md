# Особенности реализации

## State machine

На многих экранах для управления текущим состоянием используется state machine. В зависимости от
текущего действия вызывается dispatch, и от пары event/state определяется новое состояние экрана.
Все останые же поля просто подписанны на стейт и определяются его текущим состоянием.

Почти для всех экранов со сканированием, state machine не пишется, а используется общая
ScannerStateMachine, переопределяя только необходимые переходы.

## Settings

Ключи для объектов, которые сохраняем в shared preferences храним в SettingsPreferences, для
простого доступа из любой точки приложения
