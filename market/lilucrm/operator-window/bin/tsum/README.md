Скрипты для работы с (фронтовым, за неимением дугого) API ЦУМа.
Затевалось это всё для анализа времени выполнения различных стадий
выкладки наших релизов в тестинг в рамках тикета [OCRM-3829](https://st.yandex-team.ru/OCRM-3829).
Но кто знает, для чего это ещё может пригодиться.

Собственно времена можно выгрузить так:
```bash
./get-active-releases-timings.sh | tee tmp/active-releases-timings.tsv
```
Формат TSV:
* id запуска
* id джобы
* название джобы
* время работы джобы в мс.

А чтобы вывести средние значения по полученным данным,
достаточно вызывать `./summarize-timings.py`.
Установить зависимости можно командой `make init`.
```bash
$ ./summarize-timings.py tmp/active-releases-timings.tsv
                                                                 mean          std
                                                         duration_min duration_min
job                                  title
563e1945-a218-4146-822f-9b45074ad6f0 Сборка пакета          19.440645     5.730665
6f51994b-69e1-4bb5-bb24-f0192d3e6c36 Миграции БД testing     1.507284     0.277625
96b824e1-1502-43da-8658-00cae9d01f22 Деплой в testing       17.527439     7.299421
```
