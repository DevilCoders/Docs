# Экспериментирование с интерфесом операторов

Периодически изменения в интерфейсе пользователя хочется проверить на части операторов и, если
изменения роняют основные метики, быстро их откатить. Для этих целей есть возможность указывать частям интерфейса пользователя фиче-флаги вариантов отображения, а сотруднику
задавать какой вариант интерфейса ему отображать. 

Список возможных вариантов интерфейса указывается в справочнике "UI эксперимент" (FQN=uiExperiment). Эксперимент, к которому относится сотрудник,
указывается в атрибуте сотрудника "UI эксперимент" (uiExperiment). Одновременно сотрудник может попадать только в один экспеимент.

У каждого контента есть возможность указать эксперименты для которых он отображается, а так же при каких экспериментах
не отображается. Пример:

```xml
<u:content xsi:type="u:tabBar">
    <u:tab>
        <u:caption>
            <c:value>Основные параметры</c:value>
        </u:caption>
        <u:experiments>
            <u:include>EXP_1</u:include>
        </u:experiments>
        <!-- ... -->
    </u:tab>
</u:content>
```

В данном примере таб "Основные параметры" будет отображаться только если пользователь попал в эксперимент EXP_1.

```xml
<u:content xsi:type="u:tabBar">
    <u:tab>
        <u:caption>
            <c:value>Основные параметры</c:value>
        </u:caption>
        <u:experiments>
            <u:exclude>EXP_1</u:exclude>
        </u:experiments>
        <!-- ... -->
    </u:tab>
</u:content>
```
В данном примере таб "Основные параметры" НЕ будет отображаться если пользователь попал в эксперимент EXP_1.

Список экспериментов так же отдается на фронтенд вместе с карточкой объекта.

**Важно**: эксперимент - это временная проверка гипотезы, и после ее подтверждения или опровержения весь код связанный с
экспериментом должен либо удаляться, либо становиться основным (использоваться вне эксперимента.)

## Технические детали

Логика определения эксперементов в которые попадает пользователей определяется в реализациях интерфейса
UiExperimentsStrategy и может быть расширена через определения новой стратегии, в частности, попадание в эксперимент
пользователя может вычисляться на основании отображаемого объекта.
