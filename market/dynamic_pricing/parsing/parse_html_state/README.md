# Процесс регулярного перепарсинга стейта html.
На входе:
    Стейт HTML
    Предыдущая таблица с распаршенными результатами
На выходе описание оффера конкурента с информацией на каком урле этот оффер размещается в заданном окне по времени прокачки.

1. Берем стейт html
2. Парсим его
3. Расхлопываем мультиофферные страницы в отдельные оффера
4. Фильтруем урлы страницы которых не выглядят как продуктовые страницы
5. Конкатим с предыдущим распарщенным стейтом.
6. Группируем по урлу и оставляем последнюю скачанную версию.
7. Обрезаем по тайстемпу слишком старые данные


Испольдуется в пайплайнах матчинга, трендов и новинок, выгрузок с ассортиментом конкурентов
