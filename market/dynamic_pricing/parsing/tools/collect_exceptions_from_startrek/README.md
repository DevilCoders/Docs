# Парсер тикетов
Получает на вход директорию с блеклистом и очередь Стартрека, которую надо распарсить.

Идет в стартрек и достает все тикеты из очереди, которые подходят под условия:
    1. В статусе OPEN
    2. С заголовком "Добавить матчинг" или "Удалить матчинг"
    3. Есть экселевский файл

Если первые два пункта выполнены, а третий - нет, переводит тикет в Need info и пишет комментарий с призывом автора с просьбой приложить эксель и переоткрыть тикет

Если все три пункта ок, то скачивает и парсит экселевский файл.
Есть проверка на то, чтобы файл удовлетворял требованиям:
    1. Имел три столбца с заголовками market_sku url comment для блэклиста или два стобца market_sku и url для вайтлиста
    2. Имел длину не более 100 строк

В случае ошибки в каком-то из пунктов, пишет комментарий в тикет и призывает автора, переводит тикет в Need info

Если эксель спаршен корректно, то добавляет связки url-msku из него к текущему блеклисту или вайтлисту,

После сбора данных из всех открытых тикетов, публикует на YT и переводи тикет в статус "Загружено". Так же проверяет таблицу API на предмет отсутствия в ней связок из блеклиста. Если находит - удаляет.
