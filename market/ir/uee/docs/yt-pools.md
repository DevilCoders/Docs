# Кастомные yt pool

Вся работа по обогащению смартматчером и значительная по обогащению УК выполняется в yt. По дефолту, все задачи выполняются в одном и том же пуле - нашем. Ресурсов в нем не очень много, поэтому с ростом количества потребителей и задач увеличивается время обогащения. Однако, потребители, у которых есть свои CPU ресурсы в yt, могут ускорить за счет них обогащение. Для этого сделана возможность указать собственный yt pool, чтобы задачи в yt выполнялись в нем.

Как сделать, чтобы UEE для ваших задач использовала ваш пул:
0) Если у вас еще нет аккаунта в UEE, попросить нас его создать в чатике поддержки и выдать доступ к нему вашим сотрудникам и роботам
1) Дать доступ к пулу нашим роботам **robot-mrt-ir-sm-prod** и **robot-mrk-uee**
2) Попросить нас дать разрешение на использования вашего пула из вашего аккаунта в чатике поддержки
3) При запуске UEE через API указываете ваш пул в поле ytPool json-конфига запуска. Пример: https://nirvana.yandex-team.ru/flow/fd912da3-858b-45b2-9c16-7eb9dbe5d04a/11710237-eeb3-447b-b522-8253f5ed85c0/graph/FlowchartBlockOperation/code/Single_Option_To_Json_Output
(не забудьте указать правильный аккаунт, которому дано разрешение на запуск в вашем пуле).
--**При запуске из нирваны нужно обновить Arcadia Revision (опция кубика запуска, см пример) до 9246944**
