# Как вносить изменения в документацию

На каждой странице можно нажать на карандаш и провалиться к нужному файлу в аркадии
![image](https://jing.yandex-team.ru/files/eandreyf/2022-04-14_18-19-56.png)

После редактирования, нажать `Commit` -> `Send to review`. Изменения автоматически смержатся в trunk и обновление выкатится на прод.

## Описание синтаксиса
https://ydocs.tech/ru/

## Редактирование через интерфейс аркадии
1. открываешь файл, карандашик, редактируешь, выбираешь ветку trunk, создается PR
2. открываешь этот PR, копируешь название ветки вверху
3. открываешь второй файл, карандашик, редактируешь, вставляешь название ветки из предыдущего шага, комитишь
4. повторяешь N раз

В ПР автоматом призовётся команда разработки, которая проверит, что всё корректно, а на почту придёт отбивка об успешном создании ПР и прохождении проверок.

## Легковестный редактор кода от разработчиков аркадии

## Добавление нового файла
Ссылка на новый файл обязательна должна быть в оглавлении, иначе ссылки на него не будут работать
