# Дополнительные возможности конфигурирования и работы с акцией

## Автоматическое формирование ссылки на лендинг и на правила
Если в акции не указана ссылка на лендинг или ссылка на правила, то недостающие ссылки формируются автоматически.
Примеры проще всего найти у текущих акций в проде

## Режим скрытой акции
Для акции можно выставить скрытый режим. В этом случае она не будет видна на фронте.
Включить её показ можно будет через дополнительный query параметр `?rearr-factors=promo_enable_by_shop_promo_id=id_акции`.
Поле в protobuf: `disabled_by_default`

## Скрытие акции за перками, экспериментами
Для акции может быть выставлено, что она показывается только для пользователей с определёнными перками или в определённом эксперименте.
Например, только для плюсовиков в эксперименте `blue_exp`.
Поля в protobuf: `perks, experiment_rearr_flags`.
Для перков, при запросе в репорт нужно передать перки в параметре `perks`.
Так же можно завести эксперимент через параметр в репорте `exp_perks`, который будет транслироваться в ограничения из поля `perks`.
Можно комбинировать условия И/ИЛИ

## Проверка акции с флагом будущего
Для проверки работы акции, можно завести акцию со стартом в будущем, например, через неделю или просто заранее, до старта акции.
Дальше с помощью query параметра установить текущее время для акции
`?rearr-factors=market_promo_datetime=20200728T152044%2B03` - 2020-07-28 15:20:44

## Принудительное отключение акции
В случае необходимости, акцию можно принудительно отключить:
1. Проставить ей скрытый режим
2. Перевести в статус выключена. Поле в protobuf: `force_disabled`
