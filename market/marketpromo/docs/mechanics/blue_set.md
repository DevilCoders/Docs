### Места отображения
Карточка/корзина

### Настройки акции
1. Можно опционально указывать размер скидки на каждый дополнительный товар или на все товары сразу
2. Можно конфигурировать показ промо блока на карточке. Только на карточке основного товара или на дополнительных тоже
3. Комплект можно задать из 2х или 3х товаров
4. Можно включить возможность выбора дополнительного товара

### Ограничения
1. Основной товар не может пересекаться ни с другими основными, ни с комплектными. Комплекты могут быть одинаковые у разных основных товаров.
2. Основной товар и комплектный к нему должны быть разными. (Купи телефон X и получи такой же с 20% скидкой НЕЛЬЗЯ) Почему? Потому что подарочные комплекты собираются в корзине в жесткий комплект. Корзина настроена таким образом, что ей будет непонятно почему один то же товар является и подарком и основным.)
3. Комплектный товар в одной связке (связанный комплект=ДА) НЕ может быть основным товаром в другой связке

### Как выставлять процент скидки за комплект?
1. Скидку можно выставить, как на все товары из комплекта, так и на каждый по отдельности:
- Допустим, Вы хотите выдать скидку 5% при покупке товаров Х + Y вместе, тогда у каждого товара ставите нужный % скидки
- Допустим, Вы хотите выдать скидку 10% на товар Х при покупке его вместе с товаром Y, тогда только для товара Х ставите нужный % скидки
2. Если скидка не указана, то комплект соберется без скидки

### Можно ли указать в одной акции товары с разных складов?
Нет, нельзя.

## Информация для разработчиков { #dev }
**Название:** `blue-set`

На подарки репортом вешается акция `blue-set-secondary`

Генерится во время парсинга промо `blue-set` для дополнительных товаров комплекта `blue-set-additional-item`

### Report
Для проверки наличия дополнительных товаров используется [кеш вторичных оферов](../components/report/index.md#secondary-cache).
