# Среда для тестирования qparser
## MARKETINDEXER-41192

Тест **MARKETINDEXER-41192** предназначен для мануальной отладки валидации изменения цен в qparser.
Для того, чтобы обращаться к тестовому контуру вместо разворачиваемого YT-контура на *localhost*, нужно локально установить флажок **isRealDataTest** в самом тесте и прописать путь к токену YT в конфигурационном файле.

*test_MARKETINDEXER-41192.cpp*
```cpp
const bool isRealDataTest = true;
```
*common-debugging.json*
```json
"token_path": "/home/username/.yt/token"
```

{% note info %}

После окончания отладки нужно не забыть вернуть значения обратно.
Это безопасно: если потом случайно попытаться закоммитить изменения, то тесты не пройдут.
Необходимо также обратить  внимание перед коммитом, что удалён путь к своему токену в *common-debugging.json*.

{% endnote %}

## Отладка прочих компонентов qparser
При необходимости отладки прочих компонентов qparser, можно добавить новые файлы в эту папку и переиспользовать уже готовый *yt_env.h* для разворачивания локального YT-сервера.
В папку *data* можно добавлять новые входные файлы для qparser отладки и брать файлы из *MARKETINDEXER-41192-test-feed* в качестве основы.
