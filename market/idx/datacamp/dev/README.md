# Dev тулзы для ручного управления офферным храниилщем

## generate_offers

Тулза для генерации оферов

Так как в тестинге реальных пуш магазинов на порядки меньше, чем в проде, то для того,
чтобы проверять, что наши программы справляются в нагрузкой, мы генерим для одного тестового поставщика столько оферов
сколько хотим иметь в тестовом хранилище для базовой проверки работоспособности хранилища в целом

## local_run

локальный запуск хранилища

## push_offer_operations

Тулзы для генерации разлчных запросов для пуш оферов

## push_update_task

Тулза для пуша задания на парсинг

## push_update_tasks

Тулза для пуша множества заданий на парсинг по набору магазинов в shopsdat

## repush_all_update_tasks

Перепушить все фиды за определенное время с тем же, таймстемпом. Все задания читаются по таблицам логфелера

## update_push_offer

Тулза для модификации пуш офера по стейту из хранилища.

В случае, если есть понимание, какие данные нужно запушить(зная текущий стейт в хранилище) можно воспользоваться тулзой, чтобы изменить вручную данные в хранилище.

!!Использовать с осторожностью!!

## update_logfeller_protos.sh

Скрипт для обновления дескрипш прото файлов для логфеллера. Для того, чтобы новое поле из протобуфки началось парсится в логфеллере, необходимо запустить этот скрипт и закомитить полученные файлы

## testing_priority_partners.json
