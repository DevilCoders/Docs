Тулза для работы с фильтром дропа обработки оффера, способ запуска: 
- вывод текущего фильтра: `./drop_filter_tool --get` 
- установка нового фильтра (который позволит обрабатывать синтетические оффера): `./drop_filter_tool --set -c beautiful_color --no-drop --filter-inside -b 229 -s 229`
- почистить поля конфига: `./drop_filter_tool --clear-all`

Для работы должна быть задана переменная окружения `YT_TOKEN_PATH`

Аргументы: 
- `--prod` - !!! ВАЖНО !!! - для тестинга и продовых данных работа осуществляется на разных ydb-нодах. По умолчанию все функции работают для тестинговой ноды. Чтобы взаимодействовать с продовой, нужно взвести этот флаг 
- `--no-drop` - по умолчанию дропаем синтетические оффера
- `--filter-inside` - по умолчанию не анализируем внутреннее состояние батча офферов. При взведении - смотрим внутрь и дропаем синтетические в составе батча
- `--clear-all` - очистить все поля конфига в дефолтное состояние
- `-c new_color`
- `-b new_business_id` - добавить новый business_id в фильтр
- `-s new_shop_id` - добавить новый shop_id в фильтр
