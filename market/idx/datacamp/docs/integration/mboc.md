Легенда:
- ЕОХ - единое офферное хранилище
- КИ - категорийный интерфейс
- АГ - автогенерация (компоненты контура Прием контента, AG API и GG, есть еще PPP, но он здесь не участвует)
- ПД - прием данных (контур Прием контента)

Цепочка, по которой обрабатывается оффер (или его данные):

ЕОХ -> КИ -> АГ (ПД) -> МБО

Из КИ оффер попадает в AG API (агапи), далее его берет в обработку GG (гутгин).

После обработки оффера АГ передает данные в КИ, а КИ далее кладет их в ЕОХ (результаты обработки: маппинг, ошибки и прочее).

Если где-то в этой цепочке потерялись данные, есть [статья](https://docs.yandex-team.ru/market-datacamp/support/mboc_integration?searchQuery=траблшутинг#trablshuting-integracii-eoh-less-greater-mboc) по траблшутинг на дистанции ЕОХ <-> MBOC
