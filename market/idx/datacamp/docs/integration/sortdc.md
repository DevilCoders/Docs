SortDC - сервис хранящий некоторый набор информации про офферы в интернете. Ключом в стейте SortDC является урл, по которому в т.ч. агрегируются статистики пользовательской активности, считается популярность. Сам сервис имея огромную базу (десятки млрд документов) может их ранжировать и использовать эту информацию в других сервисах (например Товарной Вертикали)

На данный момент SortDC в интеграции с DataCamp-ом выполняет две функции:
  - через него (и другие соседние компоненты) работает пайплайн создания "диких" офферов ТВ (т.е. тех, которые были собраны умным парсером с просторов интернета)
  - через него Товарная Вертикаль приносит в DCMP информацию о том, какие из существующих офферов (маркета / директа / etc) она хочет добавить / убрать в своем индексе

Т.к. ключом в SortDC является url, он не только должен присутствовать для всех офферов, но и по возможности должен быть персистентен (чтобы можно было агрегировать статистику кликов по конкретному офферу и т.п.). Это представляет проблему в разрезе маркетных CPA офферов - их урл генерируется из `ware_md5`, который известен не сразу, а только после майнинга оффера + может меняться. Поэтому отдельно для этого класса офферов используются фейковые урлы, генерируемые из идентификаторов оффера. Также в рамках событийной поставки в SortDC отправляется информация о реальных урлах оффера (подписка на url и ware-md5)
