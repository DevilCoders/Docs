# Событийная модель

DataCamp предоставляет пользователям возможность подписаться на конкретные части оффера и получать данные через logbroker при обновлении этих полей. Подробнее про настройку подписки можно почитать [здесь](../for-developers/new-subscriber.md)

# Поставка данных в DataCamp

Есть несколько способов поставки данных в офферное хранилище:
- топик в logbroker. **предпочтительный вариант** для большой нагрузки. про настройку [написано тут](../for-developers/bringing-data.md)
- статические таблицы на YT. **предпочтительный вариант**, когда ваши данные - это какой-то снапшотный дифф, который надо накатить. про настройку [написано тут](../for-developers/bringing-data.md)
- фиды. есть возможность расширить парсер необходимыми вам форматами фидов
- http интерфейс. в основном используется партнерским кабинетом маркета. под большие нагрузки не расчитан. [подробнее](../components/stroller.md)
