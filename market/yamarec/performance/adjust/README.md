#### Отправка событий в Adjust

* purchase_5k_first.yql
  - произошел первый заказ на сумму **от 5 тыс. рублей** (и до 100 тыс рублей)
  - заказ должен произойти в течение 56 дней с момента установки
  - события отправляются **по вчерашним заказам** (т.е. установки могли быть в любой день в течение последних 56)

* purchase_after_install.yql
  - первый заказ произошел **в течение первых трех дней** с момента установки
  - события отправляются **по вчерашним заказам** (т.е. установки могли быть в любой день в течение последних 56)

* hot_user.yql
  - пользователь, совершивший установку, попадал в горячие сегменты **хотя бы раз** в течение прошедшей недели
  - события отправляются **по вчерашним установкам** (все вчерашние установки проверяем на горячесть и отправляем)

* hot_user_market.yql
  - то же самое, но для приложения Белого
 
* white_event.yql
  - пользователь, совершивший установку Х дней назад (изначально 3), попал в топ 10% пользователей по cpc-revenue
  - отфильтровываются пользователи, не прошедшие фильтры антифрода
  - сейчас для каждого пользователя отправляется только факт вхождения в топ, в будущем можем начать отправлять value события
  - события отправляются **устанокам Х дней назад**
