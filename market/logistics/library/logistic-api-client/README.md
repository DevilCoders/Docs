# logistic-api-client
Клиент внешнего api логистики

### Правила добавления/доработки dto в api
- DTO запросов/ответов и вложенных в них DTO делаем иммутабельными.
- Явно прописываем ```@JsonProperty``` с именем полей для сереализации/десериализации (даже если совпадаем с именем полей)
- В случае если все поля DTO обязательные и нет списков - инициализируем класс конструктором
- В случае если есть хотя бы одно списочное поле и/или есть необязательные поля - инициализируем класс конструктором + билдером. В этом случае у пользователя не должно быть возможности вызвать конструктор со всеми полями, поэтому необходимо сделать конструктор объекта со всеми полями приватным и инициализировать поля следующим образом: обязательные поля заполняются в конструкторе билдера, необязательные в сеттерах билдера, соответственно для инициализации объекта в методе билдера вызывается приватный конструктор объекта со всеми полями.

При реализации билдера всегда предоставляем констуктор без аргументов. Если это не матчится с наличием обязательных полей делаем билдер без аргументов приватным.
