# Настройки для разных окружений
## Указание настроек для определенных сред
Для любых полей есть возможность указания значений для разных сред. Для этого в yaml-файле на том же уровне, что и требуемое поле, указывается словарь env. 
- Ключами являются значения среды
- Значением - объект с полями, которые нужно переопределить для текущей среды.

Например:
```yaml
tvm:
  id: 123
  env:
    testing:
      id: 234
    prestable:
      id: 666
    production:
      id: 345
```
Среды можно называть как угодно, главное потом эти названия передавать при запуске.
```yaml
tvm:
  id: 123
  env:
    vasya:
      id: 234
```
Тогда в конфигурации запуска в идее надо добавить параметр `-Denvironment=vasya`.

Пример:
![](https://jing.yandex-team.ru/files/sid-hugo/image%20%287%29.png)

А чтобы задать окружение для выполнения в CI, надо в `ya.make` добавить строку `ENV(ENVIRONMENT=vasya)`. Обычно это нужно для запуска тестов в определенном окружении.
Пример:
