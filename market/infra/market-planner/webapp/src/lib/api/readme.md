# Ресурсы

Данные могут приходить с не только с бэка Планнера, но в большинстве случаев стоит использовать его даже как прокси.

Данные приходят в удобном для БД состоянии, затем нормализуются в страничных контроллерах или реакциях.

Ресурсы описывают поход за данными, содержат моки, декларации запросов и ответов. При этом ресурсы ничего не должны знать о сущностях в entities и доступны только из ApiClient.

Однако в реальности ресурсы импортируют для параметров фронтовые сущности, но возвращают всегда `DenormalizedEntity`. Это сделано с расчетом на то, что в будущем ресурсы могут стать сложнее, поэтому сейчас они умеют готовить данные.

Разумеется, при обращении к тому или иному ресурсу не стоит импортировать его типы, они должны быть продублированы во вне. Кишки ресурсов должны быть изолированы.
