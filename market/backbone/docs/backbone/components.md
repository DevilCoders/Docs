# Контрагенты
## [BackboneApiProxy](../components/backbone-apiproxy.md)
Сервис-интегратор API. Предоставляет по ручке для каждого виджета (или объединения виджетов).

## [Prices](../components/prices.md)
Отдает цены офферов с учетом скидок и стратегий по переданному подмножеству офферов: принимает идентификаторы офферов, региональность пользователя, возвращает пользовательские цены для каждого переданного оффера.

## [Offers](../components/offers.md)
Отдает офферы по идентификатору оффера/модели/мску. Оффер содержит все известные Маркету невычислимые  атрибуты оффера (из Datacamp).

## [Backbone Buybox](../components/backbone-buybox.md)
Отдает по модели/мску оффер (цену, доставку, промо) для отображения на кнопке Купить. Имеет упрощенную логику выбора по мин. цене. Возможно стоит совместить с Prices (ручка /v1/offers/min-price)

## Shops
DataApi-сервис, отдающий информацию о магазины по его идентификатору.

## [JumpTable](../components/jump-table.md)
Принимает на вход исходную msku + выбранные параметр и значение, возвращает целевую msku + статистику по значениям ее параметров (информация о переходах при нажатии)

## [ModelApi](../components/model-api.md)
DataApi-сервис про модельные/мску параметры: принимает на вход идентификаторы модели/мску, возвращает атрибуты модели/мску, в т.ч. параметры в виде идентификаторов (для не-gl – имена).

Также включает DataApi-сервис про GL-параметры:
- листит параметры по категории
- листит значения для параметра
- резолвит идентификаторы в human-readable имена

## [MskuStat](../components/msku-stat.md)
Предоставляет статистику доступности по мску по региону.

## [Сервис Скрытий](../components/hider.md)
Отвечает на вопрос: скрыт ли оффер, и если скрыт, то по какой причине.

## [ShowClick](../components/showclick.md)
Генерит clickurl-ссылку и пишет show-log по переданному контексту.

## Картер
Возвращает содержимое корзины для пользователя.

## Комбинатор
Определяет сроки и типы доставки товаров пользователю.

## AuthProxy
Выполняет аутентификацию пользователя, меняет session_id2-куку на user_ticket, который и ходит дальше по всем сервисам.
На текущий момент точная реализация неизвестна, есть мнение, что нужно ходить с балансера в сайдкар.
