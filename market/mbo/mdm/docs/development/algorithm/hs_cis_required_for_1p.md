# Не требовать сравнения КИЗ для 1Р

{% note tip %}

Это исключительно служебный атрибут. Сам по себе он интереса не представляет, но участвует в варке атрибута *SSKU* [CisHandleMode]((hs_cis_handle_mode.md))

{% endnote %}

## Как хранится

Это параметр уровня *MSKU*

### В конструкторе

- [Не требовать сравнения КИЗ для 1P](https://mdm.market.yandex-team.ru/constructor/mdm-entity-type/215906716/4761292148)

### В БД

- MSKU по версии МДМ ([Yt](https://yt.yandex-team.ru/hahn/navigation?offsetMode=key&path=//home/market/production/mdm/storage/msku_gold), [пример в YQL](https://yql.yandex-team.ru/Operations/Ys7XYQVK8LDTvEgM1hn8gFbX6ZylXpkIbTBzxuFWvQY=))

### В докторе и UI

В Докторе параметр можно найти на вкладке `MDM` по следующим `xslName`, равным `cisNotRequired`

## Откуда приходит

Ниоткуда не приходит, рассчитывается на основании уже имеющихся знаний о категории *MSKU*.

## От чего зависит

Зависит от:
- знаний о категории, которые берутся из [категорийных настроек](https://mdm.market.yandex-team.ru/mdm/category-param-values)
- [префикса ТН ВЭД](hs_ccc_prefix.md)
- [дерева ТН ВЭД](https://mdm.market.yandex-team.ru/mdm/tnved-categories)

В категорийных настройках хранятся знания о префиксе ТН ВЭД. Чтобы понять, какие следует ожидать значения для параметра, достаточно посмотреть в графу __*Честный знак. Не требовать сравнения КИЗ для 1Р*__, корня с сответствующим префиксом.

## На что влияет

- [CisHandleMode](hs_cis_handle_mode.md)

## Как вычисляется
Вычисление параметра происходит при пост процессинге золотых блоков мску. Алгоритм примерно следующий:
1) Из категорийных настроек вычитывается префикс ТН ВЭД
2) По префиксу определяется узел дерева ТН ВЭД, который ему соотвествует. Из дерева вычитывются все характеристики узла
3) Данные о разметке уже хранятся на узле, надо их просто взять. Мы и берем - так получается новое золотое значение
4) Новое вычисленное золотое значение вступает в борьбу с уже имющимся. Борьба может идти в 3 этапа:
    1) в первом выживает тот, кто был установлен вручную
    2) если нет победителя, во втором выживает тот, приоритет чьего источника выше в мдм
    3) если  до сих пор нет победителя, приоритет отдается самому свежему значению

## Валидации

Валидации на этот параметр не предусмотрены

## Куда отдаётся

Никуда, используется только в мдм
