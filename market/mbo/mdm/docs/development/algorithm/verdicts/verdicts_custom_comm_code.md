# Правила расчета вердиктов на префикс ТН ВЭД

## Базовые правила
- Код должен состоять только из цифр
- Код должен иметь либо 14, либо 10 символов
- Код не должен быть одной повторяющейся цифрой(например, код 1111111111 - плохой код)
- Код не должен начинаться с двух нулей и более(например, код 0011111111 - плохой код)

## Код ошибки
```mboc.error.md-customs-commodity-wrong-format```

## Уровень ошибки в зависимости от Feature при отправке В ЕОХ
- FULFILLMENT - ```WARN```
- CPA - ```WARN```
- DBS - ```WARN```

## Особые случаи
### 1P
Для 1p код префикса дополнительно проверяется на наличия префикса в актуальном справочнике ТН ВЭД. Посмотреть его можно [тут](https://yt.yandex-team.ru/hahn/navigation?path=//home/market/production/mstat/dictionaries/mdm/customs_comm_code)
При этом возможны 3 варианта работы, которые могут переключены в зависимости от положения Feature Toggle ```cccValidationMode```:
- ```NO_OP(0)``` - дополнительная валидация не производится(установлен сейчас, на 03.06.22)
- ```GENERATE_WARNING(1)``` - дополнительная валидация производится, ошибка маркируется как ```WARN```
- ```GENERATE_ERROR(2)``` - дополнительная валидация производится, ошибка маркируется как ```ERROR```
В  ```skv``` следует задавать число
