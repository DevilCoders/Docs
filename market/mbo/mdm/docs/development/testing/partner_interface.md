# Тестирование с Партнёрским Интерфейсом (ПИ)

В тестинге Маркета есть техническая возможность заводить тестовый ассортимент так, как это делал бы партнёр, прямо через ПИ. Это позволяет максимально аутентично проверить сценарии приёма серебра SSKU от поставщиков через EOX и дальнейшую обработку таких данных.

**Инструкция**

1. Сперва запрашиваем доступ в тестинг ПИ. Идём в [IDM](https://idm.yandex-team.ru) и заказываем себе роль `Маркет Тестинг` -> `MBI` -> `YA_SUPPORT`. С вас потребуют дополнительный параметр — это внешнеянексовый логин. Выберите любой из предложенных.

    {% note info %}

    Переиспользовать существующий внешний логин не получится. В свою очередь, новый логин можно не привязывать на staff (см. ниже).

    {% endnote %}
2. После аппрува роли на почту должно прийти письмо с завершением регистрации внешнего логина. Выполните предписанные там действия.
3. Зайдите на главную Яндекса и перелогиньтесь под свежесозданного юзера.
4. Теперь вы можете управлять ассортиментом от лица партнёра в [кабинете](https://partner.market.fslb.yandex.ru/).

{% note warning %}

При переходе по страницам ПИ всегда следите, чтобы в URL была приставка `fslb`. Это гарантирует, что вы находитесь в тестовом окружении. К сожалению, некоторые редиректы изредка могут приводить на продакшен-страницы.

{% endnote %}

**Интерфейс**

Открыв ПИ, вы увидите список бизнесов в вашем аккаунте. В нормальной ситуации он должен быть общим для всех.

![Список бизнесов](../../_images/partner_interface_1.png)

Выберите любой, у которого есть модель размещения "Продажи". Так вы попадёте в каталог наиболее интересных для нас синих FB* офферов.

На странице магазина в левой панельке выберите "Товары", а затем "Каталог".

![Каталог](../../_images/partner_interface_2.png)

{% note info %}

Если вы увидели красную плашку с предупреждением, что данный магазин используется для интеграционных тестов, то лучше вернуться на домашнюю и выбрать другой магазин с "Продажами". В целом, даже в каталоге без плашки просьба внимательно относиться к тестовым данным. Не меняйте офферы, у которых в названии что-то в духе `autotest` или `проверка` - так вы можете помешать чьей-то работе. Лучше просто создать свои офферы.

{% endnote %}

Вы увидите список товаров (если есть хотя бы один), кнопку создания новых и переключалку моделей размещения (сервисов). Можете выбрать существующий оффер или создать новый, и почти сразу после сохранения он поедет через ЕОХ в МДМ.
