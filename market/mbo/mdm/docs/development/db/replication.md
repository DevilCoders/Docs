# Типы реплик в PostgreSQL

Существует два вида репликаций.
- Синхронная. Такая репликация участвует в кворуме и может стать мастером.
- Асинхронная каскадная. Такие реплики доступны исключительно для read, не могут автоматически стать мастерами и могут делать реплику с реплики. Каскадные реплики - это подстраховка от варианта отключения ДЦ, чтобы не свалить мастер чтением, которое было со сломанной реплики. Каскадную реплику можно потом руками сделать активной. Отличить каскадную реплику от некаскадной в Yandex Cloud можно по наличию значения "источник" в разделе хосты кластера.
