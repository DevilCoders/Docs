# Как удалить серебро

Процесс проходит для случая, когда удалить серебряный оффер, которого нет в ЕОХ.

# Процесс удаления
- Собираем список `supplierId`, `shopSku` на удаление.
- Идем в доктор и смотрим текущую `datacammMdVersion` для каждого оффера. Он понадобится для очереди разметки на удаление.
Версия нужна для того, чтобы гарантировать, что удаляемый оффер не был прислан из ЕОХ еще раз после добавления оффера в очередь на удаление
- Делаем insert офферов в таблицу `mdm.markup_offer_for_delete_queue`
Пример:
```sql
INSERT INTO mdm.markup_offer_for_delete_queue (id, supplier_id, shop_sku, datacamp_md_version, added_timestamp,
                                               is_processed, processed_timestamp, discarded)
VALUES (1, 4647550, '5845856', 1778566252067879700, '2022-07-07 15:03:26.000000', DEFAULT, null, DEFAULT);
```
Если хотим оперативно удалить данные офферов, нужно задать минимальные id.

- После обработки офферов из очереди они пометятся на удаление и перестанут участвовать в расчете золота.
Дополнительно они отправятся в очередь на удаление и фактически удалятся через 90 дней
- Если же хочется удалить данные по офферам прямо сейчас, то нужно найти их по бизнес-ключам в очереди `mdm.ssku_to_delete` и снизить дату добавления на 3 месяца
