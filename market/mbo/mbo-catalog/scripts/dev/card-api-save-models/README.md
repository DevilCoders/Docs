Как работает:
1. Сохраняем запрос в локальный файл (запрос берём из тикета или грепаем по `logs/mbo-card-api/mbo-card-api-proto-errors.log` для свежих трассировок там будут запросы).
2. Запускаем `python3 send_save_request.py path-to-request.json`.
3. По умолчанию оно шлёт на card-api, поднятый на localhost.
4. PROFIT (дебажим там, что надо).

Как работает скрипт:
- копирует модельки с прода в тестинг (в локалхост по умолчанию)
- хранит файлик с маппингом карточек, которые он притащил, старые id на новые
- перед запросом получает modification_date для новых id с тестинга
- патчит запрос, меняя id-шники c прода на тест (id, relations, transitions) и обновляя modification_date
