Модуль для тестирования реально поднятого компонента
====================================================

Основные принципы написания тестов
-----------------------------------
- Эти тесты задумывались просто как последний рубеж проверки поднятого компонента. Не стоит тут проверять бизнес логику.
  Для этого подходят обычные юнит или интеграционные тесты. Дёрнули ручку, убедились, что вернулось что-то правдоподобное,
  закончили.
- Тесты будут запускаться на деве/тестинге/проде. Поэтому они не должны ничего ломать! Если генерируете какие-то данные,
  то добавляйте методы очистки.
- Тесты могут запускаться параллельно.

Локальный запуск тестов из ya.make
----------------------------------
```
ya make -ttt --jvm-args="-Denvironment={Указать окружение}"
```
Если не передавать окружение, по умолчанию будет использоваться testing

Локальный запуск тестов из Идеи
----------------------------------
В рядом лежащий ya.make добавляем в самый конец файла
```
RECURSE_FOR_TESTS(
    src/integration-test
)
```
Затем рефрешим проект ya ide. В ямейке это не прописано по дефолту, чтобы тяжёлые тесты не запускались из кубика со сборкой в релизе, для этого есть отдельный кубик в пайплайнах.
