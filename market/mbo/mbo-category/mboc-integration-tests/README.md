Модуль для тестирования реально поднятого компонента
====================================================

Основные приниципы написания тестов
-----------------------------------
1. Эти тесты задумывались просто как последний рубеж проверки поднятого компонента. Не стоит тут проверять бизнес логику. Для этого подходят юнит тесты или интеграционные тесты в модуле mboc-common.
2. Тесты будут запускаться на деве/тестинге/проде. Поэтому они не должны ничего ломать!
3. Тесты могут запускаться параллельно.
4. Так как у нас нет возможности в тестах ходить в базы (PGaaS, YT) напрямую, из-за отсутствия возможности пробросить
 датасорсы на тимсити машинки, поэтому можно писать дополнительные ручки у компонента для фабрикации данных.

Локальный запуск тестов из ya.make
----------------------------------
ya make -ttt --jvm-args="-Denvironment={Указать окружение}"
Если не передавать окружение, по умолчанию будет использоваться testing 

Особенности
-----------
Все ручки по-умолчанию требуют авторизации. Поэтому для интеграционных тестов выборочно мы убираем проверки в SecurityConfig.
