Зачем так сложно?
================= 
Почему нельзя использовать просто `@Secured(ROLES)` и всё?

Основная задача - показывать внятные (и автоматические) сообщений об ошибках пользователю: пойди туда и запроси такую роль.

В исходнике `@Secured` делегирует в сложные цепочки проверки и там идёт просто исключение без деталей (FIXME: если кто-то найдёт, как хорошо подлезть туда).

Соответственно что тут:
- `@SecuredRoles`/`@SecuredRolesIgnore` - аннотации для разметки. `@SecuredRolesIgnore` - потому что есть Unit-тест, проверяющий, что все контроллеры размечены так или иначе. Чтобы случайно не забывали анотации.
- `SecuredRolesAdvice` - проверяет роли.
- `SecuredRolesForbiddenException` - спец. исключение, которое сохраняет, какие роли нужны.
