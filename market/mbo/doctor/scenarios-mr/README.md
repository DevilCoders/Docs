Reduce процесс для процессинга сценарием всех оферов в хранилище.

Выделен отдельной библиотекой, чтобы бы:
1. Собрать отдельной прогой с UBERJAR(), это позволяет собрать всё в 1 jar-ник и резко ускоряет операции в YT-е 
   (когда сотни jar-ников - много job-ов виснет на подготовке окружения).
2. Был минимум зависимостей, чтобы был более лёгкий jar-ник, который заливается в YT.

Запуск: `ya make && jdk/bin/java -cp doctor-scenario-mr.jar ru.yandex.market.doctor.scenariosmr.MrMainKt`

Если отлаживаетесь, есть ряд опций для пропуска операций (см. `--help`), например, `--skip-mboc --skip-picrobot --skip-partners` - будет запускать только редьюсер и потом yql, используя данные от предыдущего запуска.
