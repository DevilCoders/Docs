Косяки MobX-а, с которыми столкнулся на практике:

1. 
```js
let rule = somethingObservable[key];
if (!rule) {
  rule = {...};
  somethingObservable[key] = rule;
}
rule.change = 'it';
```

Такой паттерн не работает, потому что при присвоении в `somethingObservable` объект копируется и заменяется на прокси, изменение простого объекта далее не отслеживается.

Решается просто:
a) можно делать `rule = observable({...})`
б) можно делать 
```js
if (!somethingObservable[key]) {
  somethingObservable[key] = {...};
}
const rule = somethingObservable[key];
rule.change = 'it';
```

2. Возвращение сложных данных в разные места, которые сами ожидают иммутабельность данных и сравнивают по ссылке, например, RDG. Решается мутабельностью соответствующих мест.

3. Очевидным образом получил проблемы с react-data-grid, который внутри себя умный и рендерит данные не трогая внешний компонент, как следствие получилось:
- с т.з. observer компонента данные не трогаются
- потом мы их меняем коллбэком изменений
- и ничего не происходит, т.к. для RDG эти данные тоже не видны

Выводы тут:
- в целом - mobx предсказуемее, если он живёт глубоко (т.е. всё использует его)
- также норм, если всё, что отрисовываем - трогаем в рендере родительского компонента
- в принципе, такие кейсы можно детектировать и довольно просто обрабатывать
