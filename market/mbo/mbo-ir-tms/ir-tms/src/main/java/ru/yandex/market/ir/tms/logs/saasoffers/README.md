Работа с копированием сессий в SaaS:
------------------------------------

Сам по себе процесс описан в `CopySessionToSaasRunner.run`, общий принцип, как это работает:
- есть два сервиса, по одному ищем, другой обновляем, за это отвечает SaasZooKeeperInfo (соответственно инфа в зукипере)
- выбираем сессию ir-а, которую будем копировать
- одна MR таска смерживает mbo_offers_mr и params_mr и готовит на последнем шаге документ для SaaS-а в их protobuf-ку
- это скармливается standalone_indexer-у SaaS-а, который в YT-е варит индекс
- после этого standalone_indexer ещё раз запускается с командой --publish, которая инициирует процесс разливки индекса по шардам SaaS-а
- дальше ждём, пока разливка закончится (дёргая ручки SaaS-а на этот счёт)
- и переключаем сервис в ZooKeeper-е

С одним сервисом вместо двух всё так же будет работать, но:
- в процессе загрузки новых копий индексов на шарды будет несинхронизированное переключение
- поиск всегда будет работать, но какие-то шарды/реплики - по новым данным, а другие - по старым
