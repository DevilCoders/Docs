# Переезд с Zora Online

## Общие различия

GoZora это достаточно простой и лёгкий прокси сервер, сделанный для минимизации накладных расходов. Поэтому, в отличие от Zora Online, в нём отсутствует:
* постпроцессинг (определение языка, кодировки и т. д.);
* переходы по редиректам (это надо делать самостоятельно или дождаться решения [задачи](https://st.yandex-team.ru/ZORA-1349)).

{% note info %}

Важный момент: User-Agent передаётся пользовательский (из запроса пользователя). В будущем это может измениться.

{% endnote %}

## Порядок перехода
1. Найдите свой источник в дочерних [тикетах на переезд](https://st.yandex-team.ru/ZORA-1404) (если тикета нет, то напишите izetag@);
2. Заполните необходимые поля в тикете;
3. Дождитесь выкатки свежей конфигурации GoZora с вашими id-шниками;
3. Адаптируйте свой код к GoZora. Пример [тут](https://a.yandex-team.ru/arc/trunk/arcadia/robot/zora/gozora/samples/python);
4. Проверьте, что всё работает корректно на [Клиентском дашборде GoZora](https://solomon.yandex-team.ru/?project=zora&cluster=production&dashboard=gozora-client-health&l.host=cluster&l.client=gozora).

## Для пользователей Zora Online HTTP API

Zora Online предоставляла обрезанный интерфейс http-прокси, в частности, не поддерживала метод //CONNECT//.
GoZora поддерживает CONNECT, но при этом использует самоподписанные корневые сертификаты.

## Для пользователей Zora Onlie MessageBus API
MessageBus API далее поддерживать не планируется. Поэтому всем клиентам MessageBus API рекомендуем перейти на HTTP API.
Если с этим есть сложности, то приходите в [ZORA-1404](https://st.yandex-team.ru/ZORA-1404) для обсуждения.
