# How it works: Yandex.Cloud, pssh and so on

Важно различать доступ по SSH во всем Яндексе и в Яндекс.Облаке. Несмотря на то, что они оба работают на схожих технологиях, дьявол кроется в мелочах ([short-lived vs long-lived certificates](https://clubs.at.yandex-team.ru/security/14877#short-livedcertificates)) и они не взаимозаменяемы. Поэтому:
  - для походов по SSH в контур Яндекс.Облака вы должны использовать pssh и его производные
  - для походов по SSH в остальную Яндексовую инфраструктуру - skotty

При этом, вы можете использовать один и тот же Yubikey, т.к. `yc-yubikey-cli`/`pssh`/`pssh-agent` и `skotty` специально не пересекаются по занимаемым слотам и не должны мешать друг другу. Поэтому если вам одновременно нужен доступ и в контур Яндекс.Облака и в Яндекс, то вы:
  - энролитесь с помощью `yc-yubikey-cli`
  - и [настраиваете skotty](https://docs.yandex-team.ru/skotty/quick-start-guide), он должен спросить у вас текущий PIN

По идее, вы можете делать это в любом порядке. Т.е. сначала заэнролить Yubikey с помощью `yc-yubikey-cli`, а потом настроить `skotty` или наоборот.
